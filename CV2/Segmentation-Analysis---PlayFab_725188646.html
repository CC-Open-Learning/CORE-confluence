Overview

Segmentation allows groups of users to be categorized based on specific properties or behaviours, which are actionable categories that provide valuable insight for decision-making. For DLX projects, these categories may include UX improvements and learning outcomes. Once the segment is defined, you can monitor the total number of users in the segment and run custom actions to discover opportunities for improvement.

In PlayFab, the PlayStream events are processed in real time and based on the configuration, will drive the new information on these users into or out of the defined segments. By having an event tie to a ‘thing of interest’ you can use these events as a differentiating factor in the decision-making process by making a segment for it.

An example of user segmentation is the time of completion of an event. Each user will be placed within a specific segment based on the time of completion, then each segment will be evaluated. In PlayFab, this can be tracked by using a statistic called TotalTime, and when the condition for TotalTime changes, then the user will enter a new segment based on the specific configuration.

Access PlayFab Segments

In the PlayFab Dashboard, to access the player segments:

  1. Navigate to Players in the menu

  2. Select the Segments tab

Segments

The Segments page displays the segments that you currently have defined, and offers basic management capabilities.

  1. Use the NEW SEGMENT button to create a new segment.

  2. The segment ID uniquely identifies the segment.

  3. The segment Name identifies the user(s).

  4. The Player count area displays the total number of players matching the segment filter.

  5. Using the checkboxes to the left of the screen, you can select certain segments, and X Delete or > Run Task… custom actions on the chosen segments.

Player Segment Configuration

The segment configuration page allows you to configure new or existing segments. You can:

  • Adjust the name

  • Define the filters

  • Assign tasks

Each segment allows you to define useful or interesting groups of players, and perform exclusive actions on that group.

How to create or access a segment

To access segment configurations:

  1. On the left side select the Players area.

  2. Select the Segments tab. From there you can then opt to:

    1. Us the New Segment button to create and configure a new Segment

    2. Locate an existing Segment, and select its Name label to configure the Segment.

How to configure a segment

Prerequisites

Some things must already exist for you to configure a segment. There must already be:

  • Players with and without a distinctive characteristic defined in PlayFab

  • A defined CloudScript PlayStream hook.

Note: a CloudScript PlayStream hook is a prerequisite because PlayStream is a real-time data pipeline that allows the capture, process, and routing of user data in real-time. Therefore, the hook is required as a prerequisite for configuring a segment in PlayFab because it is the mechanism by which you can customize the processing of PlayStream events related to the segment.

For more information about CloudScripting, please refer to the PlayFab Docs or the CloudScripting section.


Configuring

When a segment is defined, you have a variety of action options to run when a player enters or leaves the segment.

Cloudscript is by far the most flexible action, granting you full control of the player and segment information at the time of segment transition.

Use the second parameter - context - in your CloudScript handler to identify the player and segment transition. Afterwards, you can perform any action you wish for the player, such as user data, or statistics.

The segment configuration requires 4 simple steps:

  1. Assign an appropriate Segment name

  2. In the Player area, assign any conditions that the Player must meet to enter the segment.

  3. In the Type area, add an Execute Cloudscript action for the entered segment trigger. Configure the action to run the function.

  4. Use the Save Segment button to commit your activities.

Advanced Segment Filtering

Segment filtering allows you to define what players are included in the segment. An in-depth analysis of funneling can be found here.

Each filter has a unique configuration and purpose. Here are the following filters available:

  • All Players - this filter has no configuration and is unique because it allows you to create a segment of all players. This comes in handy when you want to run automatic operations for every player.

  • First login (date) - allows you to filter based on the first login datetime.

  • First login (timespan) - allows you to filter based on the first login timespan relative to the current datetime.

  • Last login (date) - allows you to filter based on the last login datetime.

  • Last login (timespan) - allows you to filter based on the last login timespan relative to the current datetime.

  • Linked user account - allows you to filter based on the users' linked accounts (ex. student emails)

  • Location (country/region) - allows you to filter based on the user’s country/region

  • Push notifications enabled - allows you to filter based on player push notifications settings and capabilities (ex. players that have Google push notifications enabled)

  • Statistics value - allows you to filter based on your own custom statistic attributes (ex. players that complete a quiz with an 80%+ score)

How to manage actions

Each action needs a trigger in order to run. Some actions are run manually and triggered right from the page. But sometimes, you have to select a trigger - an event that causes an action to run. A trigger defines the context of the action.

For example, the Player Entered Segment Trigger passes you the context containing a reference to the player.

The general workflow for the Actions panel is the following:

  1. Select and event condition (the trigger)

  2. Select the Type of action

  3. Configure the action (specific for each action type)

  4. Remove an action from the trigger

  5. Use Add Action to add an action to the trigger.

How to inspect players segments

It is possible to look up what segments a player belongs to by means of the Player Segments page.

To access Players Segments:

  1. Select the Players section

  2. Select the Players tab

  3. Click on the ID label for the player you want to inspect

  4. Select Segments in the Players toolbar

On the Player Segments page, you will see the following:

  • The Player ID label identifies the player that you are currently inspecting.

  • The Segment table contains a list of all segments that a player belongs to

  • Select the Segment name label to configure the segment.
