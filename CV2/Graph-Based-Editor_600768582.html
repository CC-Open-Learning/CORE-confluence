CORE Framework : Graph-Based Editor

Created by David Inglis, last modified on Aug 17, 2023

This section will describe:

  • the use of the NodeCanvas library in DLX projects to provide a visual graph editor,

  • the CORE package providing additional tools to extend NodeCanvas functionality, and

  • an example of NodeCanvas being used in the CORE Sandbox repository

Installing NodeCanvas in a DLX Project

NodeCanvas is a third-party asset developed by Paradox Notion. VARLab maintains and internally distributes NodeCanvas as a UPM-compatible package through a Bitbucket repository.

In the Packages/manifest.json file of a DLX Unity project, add the following line to dependencies:

"com.paradoxnotion.nodecanvas": "ssh://git@bitbucket.org/VARLab/nodecanvas.git#3.2.6"

This will install the specific stable version of NodeCanvas as a UPM package into the Unity project.

Installing the NodeCanvas Tools CORE package

NodeCanvas-Tools is dependent on the base NodeCanvas package. Until a scoped registry for CORE packages has been deployed, this dependency must be managed manually.

In the Packages/manifest.json file of a DLX Unity project, add the following line to dependencies:

"com.varlab.nodecanvas.tools": "ssh://git@bitbucket.org/VARLab/nodecanvas-tools.git#1.0.2"

Optionally, the numbered version tag (ie. 1.0.0) can be replaced with upm or a specific development branch name in order to target in-development versions of the package.

If your PC is not yet configured to access VARLab repositories via SSH, follow the instructions on Setting Up Bitbucket & Sourcetree before continuing.
