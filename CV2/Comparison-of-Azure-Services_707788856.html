CORE Framework : Comparison of Azure Services

Created by Lisa Nguyen, last modified on May 26, 2023

This article will outline basic information about the many analytic services that Azure provides.

Service Name

Source(s)

Notes

Azure PlayFab

PlayFab | Microsoft Azure

Pricing (playfab.com)

Azure PlayFab Multiplayer Services – Servers for Gaming | Microsoft Azure

  • Event driven

    • Uses raw JSON data

  • Cloudscripting

    • Can be written in both C# or JavaScript

  • Supports multiplayer using their own multiplayer service

  • Data can be sent externally via webhooks

  • Supports A/B (split) testing

Azure Synapse Analytics

(probably not the best choice for DLX)

Azure Synapse Analytics | Microsoft Azure

  • SQL based- an enhancement to Azure SQL Data Warehouse

  • Uses Apache Spark and Azure Data Explorer

  • Seems to be best used for organizations with large databases and in combination with multiple Azure services

Azure Analysis Services

(also not the best choice for DLX, more for data visualization than gathering)

Analysis Services | Microsoft Azure

  • For use with large amounts of data

  • Not an analytics tracker- essentially a data merger/a place to combine and analyze datasets

  • Data modeling/visualization

Azure PlayFab

CloudScripting

PlayFab CloudScript using Azure Functions Quickstart Guide - PlayFab | Microsoft Learn

Description

Cloud Scripting allows client code to request for server-side code to be executed.

Http Trigger

Functions that are invoked by HTTP requests to build serverless APIs and respond to webhooks. The function result will return in the HTTP response.

Queue Trigger

Requests are queued to Azure Function then processed in order. The function result will not be returned.

Playstream

Description

The event processing system used in PlayFab to take the entire data flow and put into single event stream.

(Temp source: Getting Started for Business Intelligence - PlayFab | Microsoft Learn)

Playstream Events

https://learn.microsoft.com/en-us/gaming/playfab/features/analytics/metrics/playstream-events

  1. Automatic Events: “Out of the Box” PlayFab events, ie. Player logs in with valid email/pass. 

  2. Custom events: Creating a custom event specific to your game., fully reflected through real time analytic tools available. These are classified by entity bound to a player, character, or title. An event reference is specified using 2 fields EntityType and EntityID.
