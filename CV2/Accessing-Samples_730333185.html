CORE Framework : Accessing Samples

Created by estronge, last modified on Jun 15, 2023

This page is primarily for documentation of the current state regarding samples and could be modified in the future

Currently, the ability to access samples in each package through the package manager has been changed due to some odd behavior with the Verdaccio registry (They aren’t appearing for download in the package manager window). Until a workaround is implemented, here is the easiest method to access samples without too much extra hassle.

After importing a package into Unity, navigate to the packages “Samples” folder in the Project Window. For this example, I will be using the CloudSave package.

Click “Example” in the the right hand display and drag it into either a Samples folder in your Assets folder, or just into the Assets folder directly.

With this done, the sample is imported, and the only issue now may appear when you open the scene. For some reason, the materials attached to each gameobject is lost when the sample is imported and will look something like this:

UPDATE: This was because I was working in a project that didn’t have URP installed, but I will leave this in case anyone makes the same mistake

Therefore if the visuals are important, you will need to create a new material by right clicking in the project window and navigating to Create > Material. Finally, drag it onto any meshes you would like a material on and the scene should be ready to go.
