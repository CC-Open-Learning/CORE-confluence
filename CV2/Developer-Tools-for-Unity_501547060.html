This section will cover the use of the https://bitbucket.org/VARLab/unity-developer-tools package, which provides tools for Unity such as a runtime profiler and command prompt for executing debug commands.

Installation

The Developer Tools can be found in the CORE UPM Registry as com.varlab.developertools.

Navigate to the Package Manager window in the Unity Editor and install the package under the "My Registries" sub-menu.

Legacy Installation

In the Packages/manifest.json file of a DLX Unity project, add the following line to dependencies:

"com.varlab.developertools": "ssh://git@bitbucket.org/VARLab/unity-developer-tools.git#upm"

If your PC is not yet configured to access VARLab repositories via SSH, follow the instructions on Setting Up Bitbucket & Sourcetree before continuing.

Features

Runtime Profiler

The Runtime Profiler provides a floating, movable GUI displaying runtime statistics such as:

  • Frames-per-second

  • Frame time (ms)

  • Screen resolution

  • Draw calls

  • Triangles rendered

The profiler can be toggled using a hotkey (default: Backslash \). To avoid unnecessary overhead, statistics are only calculated while the profiler window is active,

Command Interpreter

The Command Interpreter, also known as a command prompt or developer console, provides a prompt for developers to trigger debug commands while the application is running. The console window can be toggled using a hotkey (default: Tilde ~).

With the default configuration, the console is only accessible in the Unity Editor and in compiled builds where the Development Build flag has been set. The configuration can be customized to allow/block any particular runtime environment or disable the console altogether.

Some simple commands have been provided with the Command Interpreter:

  • help - displays help text for all available commands

  • log - prints a message to the Unity debug console

  • time - modifies the timescale at runtime

  • fps - modifies the current target FPS at runtime

  • reload - unloads and reloads the current scene

  • quality - read and edit quality settings at runtime

DLX developers can configure additional commands to suit the project's specific needs.

Configurations

When the package is installed, both the Profiler and Command Interpreter will be automatically enabled and accessible in the Unity Editor and in Development builds.
