Package Name

Description

Provides the ability to easily convert custom Unity components into a serialized format and deserialize data to restore components to their previous state.

Provides the ability to store and retrieve serialized data from local storage or a cloud storage system such as Azure Blob Storage.

Installing Cloud Save for DLX

Cloud Save can be found in the CORE UPM Registry as com.varlab.cloudsave.

Navigate to the Package Manager window in the Unity Editor and install the package under the "My Registries" sub-menu.

Legacy Installation

In the Packages/manifest.json file of a DLX Unity project, add the following line to dependencies:

"com.varlab.cloudsave": "ssh://git@bitbucket.org/VARLab/cloudsave.git#1.0.0"

If your PC is not yet configured to access VARLab repositories via SSH, follow the instructions on Setting Up Bitbucket & Sourcetree before continuing.

Cloud Save API Endpoint

Production

The current production API endpoint provided by version 1.0.0 of the Cloud Save package:

https://varlabcloudsave.azurewebsites.net/core/

Previous Versions

Click here to expand...

The following endpoint was previously used for Cloud Save API:

https://saveloadapi.azurewebsites.net/varlabs/

There is a known issue with the DELETE action in this API, which has been resolved in the current production endpoint. The update is not backward-compatible.

This endpoint has been deprecated and is no longer maintained. The service is running as a free resource in Azure to continue providing Cloud Save for Trucking Release III.
