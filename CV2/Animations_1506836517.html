Animations

This page is intended to give a quick overview of some features and tips of animations in UI Toolkit. Unity documentation will be linked for further reading. Animations in VELCRO are primarily used for entry/exit and hover transitions.

Transition Examples

This example will go over how the entry transition for Notifications works. This is a simpler transition that doesn’t need to target any child elements.

Button Hover Transition

Transition properties are not always needed. If you only want a simple USS property change on “:hover” applied instantly, then all that is needed are pseudo states “:hover:enabled” with the desired USS changes. This is how all VELCRO buttons handle transitions

.button-primary {
    border-width: 0;
    background-color: var(--colour-button-primary-default);
    color: var(--colour-text-primary-default);
}

/* This changes the border width, border, background, and text colour on hover. 
All of these are applied instantly since no timing function was set */
.button-primary:hover:enabled {
    border-width: 4px;
    border-color: var(--colour-button-primary-border-hover);
    background-color: var(--colour-button-primary-hover);
    color: var(--colour-text-primary-hover);
}

The “:hover:enabled” is required in Unity versions 6+ for all transitions. If “:enabled” is not added, then the element will still trigger transitions if the element is disabled

Notification Entry Transition

For more control, you can specify the properties, timing, duration, and delay. In the example below, the opacity change and translation direction/magnitude is set through code as they can be modified from serializedFields. This is triggered when the notification is shown. When this happens, the notification will translate, and change opacity over the course of 0.75s using the timing function:

.notification {
    min-width: 200px;
    min-height: 80px;
    border-radius: 40px;
    color: var(--general-13);
    opacity: 0;
    transition-duration: 0.75s;
    transition-property: opacity, translate;
    transition-timing-function: ease-in-out-back;
}

Child Element Transition

If the elements you need to adjust on a :hover is not the same element you hover over, you can target other elements using Id (“#”) selectors and child (“>”) selectors. A common example of this are buttons with images. VELCRO’s icon only buttons have this issue. The button is 60px x 60px, but the child icon is 36px x 36px.

Before Unity 6, icons on buttons are not supported and you must add a child VisualElement to contain the image.

Below will go over steps needed to achieve the desired hover state from the default state.

Default State:

Desired Hover State:

  1. Have a USS class on the parent element. In this example, it is “icon-large”, and “icon-square”

.icon-large {
    width: 60px;
    height: 60px;
}

.icon-square {
    background-color: var(--icon-default-background-colour);
    border-color: var(--icon-square-default-border-colour);
}

  1. Target the child or Id of the element you want to affect when the given pseudo state triggers. In this example, we want the #Image element image tint to change from “#FFFFFF” to “#000000” on “:hover"

/* This will target the child element by Id #Image when the element that has the 
USS class ".icon-square" is hovered and enabled */
.icon-square:hover:enabled > #Image {
    -unity-background-image-tint-color: var(--icon-square-hover-focus-image-colour);
}

.icon-square:hover:enabled {
    background-color: var(--icon-square-hover-focus-background-colour);
    border-color: var(--icon-hover-border-colour);
}

The “:hover:enabled” is required in Unity versions 6+ for all transitions. If “:enabled” is not added, then the element will still trigger transitions if the element is disabled
