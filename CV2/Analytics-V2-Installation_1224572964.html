Installing the Analytics Package

Overview

With access to the VARLab SSH key and the Analytic Package pathway from Bitbucket, the installation can be done using two different methods:

  1. Package Manager

  2. Manifest.json

Package Manager

Using this method will allow the package to be installed using the package manager on Unity. There are two steps to complete this installation process.

Step 1 - BitBucket

To find the git URL, go to the Analytics Package on Bitbucket --> Core-Analytics

Protocol:

ssh://

Git Repo Location:

git@bitbucket.org/VARLab/core-analytics.git

Branch:

#upm

git URL for package installation:

"ssh://git@bitbucket.org/VARLab/core-analytics.git#upm"

Step 2 - Package Manager

  • In Unity, select the Assets tab

  • Navigate to the “View in Package Manager”

  • In the Package Manger, select the + symbol

  • Select “Add package from git URL…”

  • Paste the Analytic package git URL

  • Select “Add”

Congrats! The Analytics Package is now installed on your Unity project!

Manifest.json

Using this method allows the package to be installed directly into the Unity project, so the program can handle the package as a resource for the project under the packages in the project tab. There are two steps to complete this installation process.

Step 1 - Bitbucket

Here is the link to the Analytic Package on Bitbucket --> Core Analytics

Package name:

“com.varlab.analytics":

Protocol:

ssh://

Git Repo Location:

git@bitbucket.org/VARLab/core-analytics.git

Branch:

#upm

Pathway string for package installation:

"com.varlab.analytics": "ssh://git@bitbucket.org/VARLab/core-analytics.git#upm"

Step 2 - Manifest.json file

With the pathway string copied, let’s open up the Unity project folder.

  • Open the Packages folder

  • Open the manifest file

  • Add a comma to the last pathway

  • Paste the pathway for the Analytics package in a new line under the last pathway

    • As seen in the photo

Congrats! The Analytics Package is now installed on your Unity project!

Updating an Existing Installation

Once the package has been installed, periodic updates may occur. To ensure you have the latest version, click on Window → Package Manager.

In the Package Manager window, look for CORE Analytics.

If there is a white circle with an up arrow on the right side of the row where CORE Analytics is listed, it means there is an update! Click on the row with CORE Analytics listed and you will see a window with the package details appear on the right side of the window. Click on the Update button to update the package! Note: when updating from a pre-release version, you may have to update more than once to get the latest version.

If the Analytics Package is currently undergoing new development, you will need to expand the CORE Analytics line and choose the latest full version (only shows the version number - 2.0.17 in the screenshot below) from the list prior to clicking the Update button.

Congrats, you now have the latest version of the analytics package!

PlayFab Account & Dependencies

For production build DLX’s there is an official VAR Lab PlayFab account, to get these account details please contact VAR Lab IT. For development/testing you can create a free PlayFab account here: PlayFab

PlayFab is a dependency and is included when installing the Analytics Package. If the Editor Extensions tab isn’t automatically available, click on Tools → PlayFab → Editor Extensions to open it.

Once available, Log into PlayFab in the Unity Editor. You will need a free PlayFab account (for testing) which can be created here: PlayFab For Release DLX use the VAR Lab Account (mentioned above) in your build.

Once logged in, select your title (if already created).

The Title needs to be setup on the PlayFab website, using your testing account create a title for testing. For release candidates use the title in the official VAR Lab account. The dropdown below shows instructions on creating a new title.

Adding a new title on PlayFab

Once you are logged in to PlayFab from the game manager you will create a new title as shown below.

You select new title, give it a name, and from there it should show up in Unity under the title IDs for the account you are logged in as. You may need to refresh the studio which is located here.

Now PlayFab is set up in Unity. Note: The DLX (if newly created) will have to be moved to the VARLab DLX section of PlayFab as below. Only full-time VARLab developers have access to complete this step. If required, reach out to your full-time team leader to have this completed.

Congrats! Everything is installed and you can start implementing analytics into your project!
