About the Inventory

The inventory is a unity package that implements a dynamically created inventory that can include containers of items or sub-containers that the developer can modify to include the title, image, or game object/prefab of the item or container that is able to be nested as many times as they would like and is. created on runtime. This works with inputted UI and data into their respective component classes that will be found when implementing the system.

About Inventory UI

Inventory UI is a component class that is responsible for modifying and storing the UI UXML assets for the button to get into the UI and the Inventory UI itself. This is also responsible for keeping track of all of the Visual Element and Button names used for functionality in the Inventory System. Here are all of the modifiable inspector values that will be used with the system:

UI Assets

  • Panel Settings - An inputted Unity Panel Settings asset that will be used with the built in UI document

  • Base UI - An inputted UXML asset file that is used to access the Inventory UI when trying to open or re-open the system in game

  • Inventory UI - An inputted UXML asset file that is used for the entire Inventory UI, which should have all of the Visual Elements inside named correctly to match the following names so that the Inventory functionality works correctly

UI Component Visual Element Names

  • Item Row - An inputted string that is related to the name of the Visual Element parent that will hold all of the dynamically created items/containers on runtime

  • Object Viewer - An inputted string that is related to the name of the Visual Element that will hold the parent for the built in Object Viewer functionality

UI Component Button Names

  • Open Inventory Button - An inputted string that is related to the name of the button for opening the Inventory UI, this Button should be created in the “Base UI” UXML asset file, which was stated above

  • Item Button - An inputted string that is related to the name of the button that will be used as a template for every other item/container that is created dynamically at runtime

  • Back Button - An inputted string that is related to the name of the button that closes the Object Viewer functionality, this will only show when the object viewer is active (when an item is clicked)

  • Close Container - An inputted string that is related to the name of the button that closes the current container that is opened and goes back one to the previous container

  • Inventory Close Button - An inputted string that is related to the name of the button that closes the entire Inventory

Unity Events

  • Open Inventory UI - A Unity event that is called when the Inventory UI is opened

  • Close Inventory UI - A Unity event that is called when the Inventory UI is closed

About Inventory Data

Inventory Data is a component class that is responsible for storing the data for all of the items/containers that is inputted in the inspector. These values that are inputted can be nested as many times as the developer wants and can include titles, images, and game object/prefabs of items or containers. Here are all of the modifiable inspector values that will be used in this system;

Type

<null> - This type is the default value when adding a new item to the Inventory data component, this should not be used and is just used as a default

Item - This type is used for any items that are created to be singular objects that will use the Object Viewer functionality

Container - This type is used as a storage for any other items or containers that you want to be categorized under

Title

Title - The title of the item or container that is being added, this will show in the correct object UI that it is inputted under

Image

Image - The image of the item or container that is being added, this will show inside the correct object UI that it is inputted

Object

Object - The object that can either be a game object or prefab that will be used with the item types so that they will be shown in the Object Viewer when clicked

About the built in Object Viewer

The Inventory System is using the VARLab package “Object Viewer” for its built in 3D object viewer functionality. This will work automatically and a version of this package will be installed alongside the installation of this package.

More information about this package and how to setup the values can be found here: Object Viewer

Known Issues

  • Arrows on Inventory Data sometimes don’t work correctly

    • To avoid this, click the label instead of the arrow if you encounter this problem

  • Inventory Data inspector values can be too close to each other to ready correctly

    • To fix this, enlarge the inspector window so that the inspector values can show correctly
