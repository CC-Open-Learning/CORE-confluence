CORE Framework : NodeCanvas Nested Graphs in DLX Projects

Created by Jonah Pearce on Mar 01, 2023

Nested Graphs will be vital to many DLX projects, as it is a lot more effective than having one super graph for a Unity scene.

Collaboration and Conflicts

Collaboration is greatly improved when using nested graphs, as all graphs except the starting graph will not be bound or active in the scene but instead stored as assets in the project directory. From this, we get the same benefit with prefabs, where one person editing one graph handling a portion of the project will not conflict with someone editing another graph, as they are saved as separate files. With one graph in the scene, there would be many merge conflicts over time from multiple users making edits.

Multiple Graph Types

Using nested graphs allows for, depending on what that graph will be handling, the most use of the different graph types, as dialogue trees can be worked in where communication with the user is the focus or if its complex functionality like an AI, for example, a behavior tree graph can be used.

Organization

Having subgraphs stored inside of assets will keep all the graphs more simple and clear, making working on task-driven DLX projects much easier.
