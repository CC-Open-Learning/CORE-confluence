The classes documented on this page are all contained in the velcro-structure.uss style sheet in the VELCRO UI package. Class functional categories include margin, padding, flex and other alignment options, width, height, and visual modifiers. Their primary purpose is for use in the structural makeup of UI templates in the VELCRO library, and for making it easy to swap out structural components by simply adding and removing classes at will in the UI Builder.

Spacing

Classes for margin and padding are named using the format [property][direction]-[value]

Where properties include:

  • m for margin

  • p for padding

Where direction includes:

  • t for top

  • r for right

  • b for bottom

  • l for left

Where value includes:

  • # value of the pixel count for that margin/padding property

Layout

All of the layout properties are affected by the element’s direction value. The values in the tables show the default values/behaviour. Justification is affected by the direction of the element’s primary axis, while alignment is affects by the direction of the element’s secondary axis.

Direction

Class Name

Property(s)

Description

Example

.flex-column

flex-direction: column

All child elements will be ordered in a column from top to bottom

.flex-column-reverse

flex-direction: column-reverse

All child elements will be ordered in a column from bottom to top

.flex-row

flex-direction: row

All child elements will be ordered in a row from left to right

.flex-row-reverse

flex-direction: row-reverse

All child elements will be ordered in a row from right to left

Justification

Justification determines the child element vertical positioning. This will be reversed if the direction is set to a column-reverse or row-reverse, and swapped to determining the child element horizontal positioning if the direction is a row instead of a column.

Where:

  • column - flex-start is the top

  • column-reverse, flex-start is the bottom

  • row, flex-start is the left

  • row-reverse, flex-start is the right

Property(s)

Description

.justify-start

justify-content: flex-start

All child elements will be positioned (Vertically by default) at the start position of the this element’s primary axis

.justify-center

justify-content: center

All child elements will be positioned (Vertically by default) at the center position of the this element’s primary axis

.justify-end

justify-content: flex-end

All child elements will be positioned (Vertically by default) at the end position of the this element’s primary axis

.justify-space-between

justify-content: space-between

All child elements will be distributed along this element’s primary axis with an equal amount of space between each other. The first and last element will be flush against flex-start and flex-end

.justify-space-around

justify-content: space-around

All child elements will be distributed along this element’s primary axis with an equal amount of space between each other. The first and last element are spaced half the amount from flex-start and flex-end

Alignment

Alignment determines the child element horizontal positioning. This will be reversed if the direction is set to a column-reverse or row-reverse, and swapped to determining the child element vertical positioning if the direction is a row instead of a column.

Where:

  • column - flex-start is the the top

  • column-reverse, flex-start is the bottom

  • row, flex-start is the left

  • row-reverse, flex-start is the right

Background

Class Name

Increment Amount

Property(s)

Description

Example

.background-stretch

-unity-background-scale-mode: stretch-to-fill

stretch-and-fill will stretch the background image, and contain it within the parent’s bounds. This breaks the background image’s aspect ratio

.background-scale-crop

-unity-background-scale-mode: scale-and-crop

scale-and-crop will stretch the background image, but not contain it within the parent’s bounds. It will scale the image until entire parent’s bounds are covered, and crop the overflow

.background-scale-fit

-unity-background-scale-mode: scale-to-fit

scale-to-fit will stretch until either the vertical or horizontal reaches the parent’s bounds. Then it will stop, and maintain the background image’s aspect ratio

.background-size-10 to .background-size-100

10

background-size

This element’s background-image width/height will be set to [10%-100%] of it’s defined/inherited size

.background-x-left

background-position-x

This element’s background-image will be set to the start of it’s X axis. This will ignore the main/secondary axis flip from using flex-column, or a reverse layout.

.background-x-right

background-position-x

This element’s background-image will be set to the end of it’s X axis. This will ignore the main/secondary axis flip from using flex-column, or a reverse layout.

.background-y-left

background-position-y

This element’s background-image will be set to the start of it’s Y axis. This will ignore the main/secondary axis flip from using flex-column, or a reverse layout.

.background-y-right

background-position-y

This element’s background-image will be set to the end of it’s Y axis. This will ignore the main/secondary axis flip from using flex-column, or a reverse layout.

Position

Class Name

Property(s)

Description

Example

.relative

position: relative

This element’s positioning will depend on the parent element’s bounds and other sibling elements that are set to relative

.absolute

position: absolute

This element’s positioning will depend solely on the parent element’s bounds.

.grow

flex-grow: 1

Determines how the item will grow relative to the rest of the flexible items inside the same container. Flex-grow will make the element take up all available space in the parent container and share it evenly with other elements that have the same grow value

.shrink

flex-shrink: 0

Determines how the item will shrink relative to the rest of the flexible items inside the same container

The flex-grow/shrink property can be set to any non-negative decimal value. Depending on the value, an item can have more (or less) grow priority than others which means they will take up more (or less) space.

Size

All size classes for width and height use % since UI Toolkit doesn’t support em and rem units. However, that is less of an issue since the PanelSettings asset handles the majority of scaling for us. Pixel count can be used in DLX specific USS classes or inlined.
