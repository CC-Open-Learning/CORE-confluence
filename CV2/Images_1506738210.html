Images

This page is intended to give a quick overview of image use in VELCRO and UI Toolkit. Unity documentation will be linked for further reading.

Images in VELCRO

Most images in VELCRO are plain white versions of the icons used in the Figma designs. This allows us to only have 1 version of the image in the project files, and use the -unity-background-image-tint-color property for customization/theme switching.

All image references in VELCRO are in Dark.tss, Light.tss or referenced in SerializedFields for some prefabs. Images can be overridden exactly like colours and font seen in Overriding VELCRO Themes for DLX.

All images are located in the “Assets/VELCRO UI/Sprites/X” folder.

How to use Images in UI Toolkit

Using sprites that cannot be plain white images

  1. If an image requires multiple colours, outlines, etc. such as the location pin from notifications, then it should be exported from Figma with a light and dark version.

  1. Create USS variables in your project’s [Name]Light.tss and [Name]Dark.tss to hold the file paths to the sprite using url() or resource().

Light.tss

--notification-image: url("../Sprites/Icons/Pin_Light_Sprite.png");

Dark.tss

--notification-image: url("../Sprites/Icons/Notification/Pin_Dark_Sprite.png");

  1. Use these USS variables in your USS classes, and apply the required classes to your VisualElement. The sprite will automatically switch when the theme used is switched in your project’s panel settings asset.

velcro-notification.uss

.notification-icon {
    background-image: var(--notification-image)
}

Using sprites that can be plain white images

  1. If an image does not require multiple colours, outlines, etc. such as the header bar that is used for various modals, then a white version should be exported instead. This is best if you find the matching icon on SVG Repo. If not, then you can alter the exported image from Figma. The colour of the icon can be controlled through the -unity-background-image-tint-color property instead.

  1. Create USS variables in your project’s [Name]Light.tss and [Name]Dark.tss to hold the file paths to the sprite using url() or resource().

  2. Create USS variables to hold the background colour.

Light.tss

--header-start-background-image: url("../Sprites/UI/HeaderTopLeft_Sprite.png");
--header-tint-colour: #AFBFCA;

Dark.tss

--header-start-background-image: url("../Sprites/UI/HeaderTopLeft_Sprite.png");
--header-tint-colour: #636F6E;

  1. Use these USS variables in your USS classes, and apply the required classes to your VisualElements. The sprite/tint colour will automatically switch when the theme used is switched in your project’s panel settings asset.

velcro-components.uss

.header-start {
    background-image: var(--header-start-background-image);
    -unity-background-image-tint-color: var(--header-tint-colour);
}
