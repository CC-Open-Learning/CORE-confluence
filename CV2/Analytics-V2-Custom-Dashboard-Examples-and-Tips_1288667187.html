Getting our Data

The first step in making a Power BI dashboard is to get what data we want

Once the Get Data tab opens, type “Azure Data” into the search, and then choose the “Azure Data Explorer (Kusto)” option

Then head to Microsoft Azure, go to our Azure Data Explorer Cluster, and Copy the URL for Power BI

Then add that URL to the Cluster field in Power BI. For now, the database we are using is just the one, which is named “playful-event-db “. So we will be adding that to the database field in Power BI. Once we have more databases (If ever), we will have to figure out which one holds the data we want. To find the database we want, in playfab-data-explorer (Our cluster), on the left side there will be a drop-down data, and then there will be a database option

So in Power BI, our Azure Data Explorer (Kusto) window will look like this;

Usually, when working with mass amounts of data we will want to create a query that will filter only for the rows we will need. However, right now we don’t have much data, so we can skip that. Before we hit ok, we want to scroll down and choose the Direct Query option

The big difference between Import and Direct Query is that Import will store the data on the Power BI application, giving it weight, but with Direct Query, it will not store the data locally.

After clicking ok, it will (probably) prompt us with a login menu, just click Sign in, and use our work Conestoga credentials, and we shouldn’t have any issues.

After logging in, a Navigator window should pop up with all tables in our database, or just the ones we queried for.

For this example, it’s how to make an executive dashboard with multiple tables, so we’ll select ABM and MPC, so we will select those tables by checking the checkboxes on the left side. And then click load

Once it loads, we will see a data tab on the left side of the Power BI App.

DAX measures

From there, combining data through joins and such things can be tricky, but it has the same premise as with SQL joins, like inner, outer, etc. However, for us, we can use DAX measures to get the data we want. Richard made a nice DAX measure that will get unique users.

First I like to make an empty table to hold all of our measures, complex dashboards can have a lot of measures. to do this, on the top of the application, if we’re on the home tab, we are going to want to click the Enter data option.

A Create Table window will open, we don’t need to enter any data, we just want to change the name of the table. I like to usually just call it “Measure”. I would call it Measures, but Power BI doesn’t like that. After renaming it, we can click load.

Once that loads, in the data tab, we will be able to see our new table, and from there, we will want to make our first DAX measure. Right-click on our Measure table, and then go to the New Measure option.

When we click it, a little text window at the top of Power BI will open. Here we will write our DAX measure. Something to note, if there is nothing in the measure and we click off of the little box, it will delete the measure. This is fine, we’ll just have to right-click and do the new measure again.

in the measure, for this example, we’re going to put this snippet of code (DAX) in;

UniqueUsersABM = CALCULATE(
    DISTINCTCOUNT('paramedic-ambulance'[Entity_Id]),
    'paramedic-ambulance'[FullName_Name] = "Development_Login_Successful"
)

It should look like this in Power BI;

Then we can click the checkmark on the left, or we can just hit enter. After this, our measure will be created, and now we can delete that pesky “column” that’s in our measure table (This is just for cleanliness).

To quickly display our measure, there is a visualization tab on the right side beside data, and we’re going to want to find the “Card”, and then we’re going to click it.

Data Displays

Once we click the card visualization, it comes up on our report. It won’t have any data displayed, so we want to add our measure to it. For this, it’s very simple, first, we can click the card to have it selected, then click the checkbox beside our measure, and it should display the measure.

Then we’re going to do the same thing, but for MPC, so in a new measure, replace the 'paramedic-ambulance' with 'motive-power-circuits'. Our DAX measure should look like this for MPC.

UniqueUsersMPC = CALCULATE(
    DISTINCTCOUNT('motive-power-circuits'[Entity_Id]),
    'motive-power-circuits'[FullName_Name] = "Development_Login_Successful"
)

Once that measure is created, we’ll want to make a new card for this data. Be sure to click off the old measure (make sure it’s not selected) or it will replace the current data in it with the new measure we just made. After that, add the measure to the card like we did previously, it should end up looking like this.

MPC is blank because we haven’t implemented the new Analytics into it yet.

Lets make our final measure just like we’ve been doing, and use this snippet;

Total Logins ABM & MPC = [UniqueUsersABM] + [UniqueUsersMPC]

And now we have a measure that says how many unique logins have happened over both sims.

And there you go! You have successful made a dashboard with data from two different projects, and made measures to show specific data!
