CORE Framework : Analytics V2 Unity Implementation

Adding Required Assemblies

  1. Click on the assembly file for your project. It will likely be named similar to VARLab.<name of project>. In the screenshot below it is called VARLab.ambulance.

  1. In the Unity Inspector, find the Assembly Definition References section. Note: The assemblies shown in the screenshot may not match those in your project depending on which packages are being used.

  1. Two references will need to be added in order for the analytics package to function correctly. The first is com.varlab.analytics and the second is PlayFab.

  1. To add com.varlab.analytics click on the at the bottom of the Assembly Definition References section. You’ll see a missing reference is added to the list.

  1. Click on and you’ll be presented with a list of all the available assembly files. You can search for analytics in the top bar to make finding the correct file easier.

  1. Double-click on the com.varlab.analytics file to add it to the Assembly Definition References.

  1. Repeat this process for the PlayFab file (searching for PlayFab to find the file. Note: You will find two files, we only need the first file and the PlayFab.Editor file can be ignored.

  1. Once you have both references added, save your Assembly Definition Asset (Ctrl-S or by clicking on another asset and selecting save when prompted before switching to the clicked on file).

Congrats! The assembly definition references are now set up and the package is now ready to be used.

Adding the Analytics Service Prefab

  1. In your Unity project window, double-click the Packages folder at the bottom of the list. Note: you may also have a Packages folder in your assets folder. If so, ignore this folder as the one at the bottom (outside your assets) is the one we need.

  1. The list of installed packages should appear. If not, click on the icon at the top right of the Project window to un-hide the packages.

  1. Double-click the CORE Analytics folder and then the Prefabs folder from the expanded CORE Analytics folder.

  1. Drag the Analytics Service prefab from the folder into your Scene.

  1. Once the analytics service prefab is in your scene, save the scene.

The Analytics Service prefab is now in place and ready to be configured!

Creating the Analytics Configuration File

The Analytics Configuration Scriptable Object included with the analytics package is for reference only. Before using the Analytics package, a configuration file must be created and a reference added to the Analytics Service prefab.

  1. Create the configuration file by clicking on Assets in the top Unity menu, then Create, CORE, Analytics and finally Analytics Configuration.

  1. A new analytics configuration file will be created in your Assets folder. This can be renamed to fit your project. Refer to Using the Analytics Configuration File document for information on how to use this file.

  1. In your Scene, click on the Analytics Service prefab.

  1. Drag the analytics configuration file into the Analytics Manager → Analytics Configuration section in the Analytics Service Inspector window.

  1. Save the Scene to ensure the analytics configuration is ready to use.

Congrats! Everything is now installed and configured and you are ready to begin sending Analytics Events!
