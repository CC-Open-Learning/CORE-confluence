CORE Framework : Data Processing and Modeling

Created by Hayden Auterhoff, last modified by Jacky Lam on Jun 14, 2023

This article will go over using PlayFab Data Explorer advanced mode to discover insights in your event data using Kusto Query Language (KQL). KQL can only perform read-only requests to process data and return results.

The events.all Table

All event received is stored in the events.all table by default.

Generate Visualizations with KQL

KQL can generate graphs and charts along side with query result using the render operator keyword.

The following keywords can be used to generate : areachart, barchart, columnchart, scatterchart, piechart. With additional kind property to generate variations of the charts, those properties can be find in https://learn.microsoft.com/en-us/azure/data-explorer/kusto/query/renderoperator?pivots=azuredataexplorer.

Limitations

There is no customization with PlayFab Dashboard.

Data Explorer (advanced) does not have the feature to save query in the web portal to be reuse later. It has tabs, but once you close the tab, the query is gone. There is a high risk of accidentally closing a query tab, so it is a bad method to store query statement to be reused.

A better way to have store and reuse queries is to use https://dataexplorer.azure.com/.

Check out the article about Azure Data Explorer.

For more powerful tool, consider looking into Microsoft Power BI.
