CORE Framework : Filtering Code Coverage Results

Introduction

This page will provide instructions on how to filter your code coverage results. This will prevent library files from being included in your code coverage. This is important to set up because external libraries should already be tested, and are not our responsibility to test. This will also drastically lower our code coverage percentage, because it will take into account lines from all scripts and libraries.

Text Instructions

  1. To begin, open the Code Coverage window.

  2. Click on the plus sign under “Included Paths” to select the path/folder you wish to focus on for code coverage.

  3. Navigate and select the folder you wish to test (make sure this encompasses all the scripts/files the project needs to keep track of).

  4. Click on the plus under “Excluded Paths” for any files/folders that the project doesn’t need to test that might be included in your previous “Included Paths”. No need to exclude folders/files that were not in your included paths.

  5. Navigate and select the folder you wish to exclude from the tests.

  6. Now your Code Coverage Report should be filtered to the folder/files you added to “Included Paths” and exclude the folders/files you specified in your “Excluded Paths”.

Next Steps

Now that you have filtering setup, you can finally run some tests and get your results. Find out how by following the next guide on Running Unity Tests.
