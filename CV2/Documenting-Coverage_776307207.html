CORE Framework : Documenting Coverage

Introduction

Now that you have your code coverage results, you must document the results. Find the Code Coverage Report page under your project’s Confluence space, which contains three tables.

When creating a pull request, add your code coverage to the first line of your merge message, then fill out the code coverage table while waiting for your pull request to be reviewed.

Filling out the Code Coverage Report

  • Copy the link to your pull request commit and paste it into the Commit ID section.

  • Copy the link to the ticket and paste it into the “Linked Ticket” section.

  • Enter the date of when the test was done under “Date of Test.”

  • Enter the line coverage percentage from your HTML report under the “Code Coverage” section.

  • In the “Untestable lines added” section, enter whether you added untestable lines using Y for yes and N for no.

Afterwards

If your pull request is accepted, update the two tables for line coverage and untestable code tables.

  • Copy and paste the coverage table (highlight the entire table) from the HTML document over the old table, and reformat it to look like the image below. NOTE: you may need to try different browsers to get the copy & paste to work. Google Chrome has been giving us grief. Firefox works.


  • In the Untestable Code table, add one column per group of untestable lines of code. Document which file the code is in, which lines are untestable, and the reasoning for why it’s untestable.

  • If your pull request has been denied, fix any underlying issues, add tests if necessary, re-do your coverage tests, update the coverage table and have it reviewed again.
