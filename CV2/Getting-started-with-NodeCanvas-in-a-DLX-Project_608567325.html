CORE Framework : Getting started with NodeCanvas in a DLX Project

Created by Jonah Pearce on Mar 01, 2023

Creating a new Graph

You can create a new graph by right-clicking in the Unity project window and selecting "Assets" ->"Create" -> "ParadoxNotion" ->"Node Canvas.‚Äù. You can choose the type of graph from the menu, creating a new asset file with the extension .ncgraph that you can open in the NodeCanvas editor.

The NodeCanvas editor is a powerful tool that allows you to create and manipulate the behavior of your graph visually using a flowchart-like system of nodes. The editor provides a wide variety of tools and options that you can use to create custom behavior for your Unity projects.

A lot of functionality is possible with this editor and its built-in actions without writing any code.

Adding Nodes to the Graph

To add nodes to your graph, right-click in the NodeCanvas editor, and you will be prompted with many options depending on the graph type. This is a list of available node types that you can use to build your graph, including:

  • Action nodes represent a specific action, from modifying a game object, such as moving, rotating, or playing an animation, to changing a variable. One node can perform multiple actions.

  • Dialogue nodes, when using a Dialogue Tree, can also be used on some other graph types as well, provided text or a string that will be used on whatever actor you specify from the scene.

  • Other Nodes: Special nodes are available that trigger not in the flow or order off of your graph but off of graph details like on graph update or any state nodes which trigger no matter the current state of the graph.

  • SubGraphs: Subgraphs can be created and added, which will be an entirely new graph saved to its asset file that gets triggered once the main graph you are using reaches that point, which is how we created nested graphs.

Each node type has unique properties and options you can set to customize its behavior based on the actions you give it to perform. For example, an action node that moves a game object might have properties for the speed and direction of the movement.

Connecting Nodes

To connect nodes, you can click and drag from one node's output to another's input. This creates a connection between the two nodes, allowing the output of one node to be used as the input for another. By default, the graph will follow that flow to the node you connected once the previous node is completed, but if you are using a state machine, a condition can be added to it. For example, you might want the user to click a button before moving to the next node.

Editing Properties

Each node and connection has properties you can set to customize its behavior. You can access these properties by selecting and looking at the inspector window. The properties will vary depending on the type of node. For example, an action node might have properties for the speed and direction of the movement, while a connection might have properties for the button or variable it checks.

Running the Graph

To run the graph or series of charts you have made and connected to your scene, you need a graph controller for whatever graph type is your first graph that triggers on a game object in that scene, you must attach it to a GameObject in your Unity scene.

Once the graph is attached and bound to that owner in your scene when you run it, it will trigger based on the behavior you set in the owner.

Using the Graph Editor

The graph editor of NodeCanvas is extremely powerful. It comes with an intuitive and clear interface. A visual node canvas graph editor is highly flexible, allowing users to easily make changes to the design of a system or process. Nodes can be moved, resized, and connected in many ways, making it easy to experiment with different designs and configurations.

Visual Debugging

The graph editor is extremely visual when a program is running, showing what node the graph is currently on, what conditions its waiting for, and what is currently active. It creates a very easy way to debug and understand your graph as you run the scene because it's very clear what is happening with the graph. It updates in real-time as its runs.
