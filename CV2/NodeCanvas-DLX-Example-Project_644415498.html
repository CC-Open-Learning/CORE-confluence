CORE Framework : NodeCanvas DLX Example Project

Created by Jonah Pearce on Mar 01, 2023

The NodeCanvas DLX Example Project is a DLX example of a basic computer inspection process. It uses a series of graphs, state machines, and dialogue trees. The game starts by running a state machine graph that uses a new task called randomizing active objects, essentially randomizing damaged computer hardware on our desktop. The user then has to identify each part, and by having a dialogue graph for each computer part, the user has to answer whether the part is damaged or not using the prompt in the dialogue system. Based on the original randomization, the graphs check if it's correct.

Nest Graphs

There is an example of nested graphs because the main graph that starts when the scene loads run in a game loop to make it replayable, and it has all of the hardware dialogue graphs as nodes and the randomization graph, which runs throughout the game loopâ€”serving as an example of how we can have nested graphs instead of one graph.

Graph Types

State Machines, Dialogue Trees, and the blackboard are all used to make these multigraphs function together and create all the necessary functionality to make this inspection game work. The Dialogue trees would likely be used often to simulate instructions in DLX projects and the state machines for task flows.

Custom Tasks

The randomization of hardware is way beyond the built-in tasks, so this project also demonstrates custom tasks by having a randomized action task our randomization graph uses to make the game different each time it's played randomly.

Automated Testing

The project also demonstrates automated testing of graphs through its test cases that test its subgraphs to ensure they perform as expected with simulated input.
