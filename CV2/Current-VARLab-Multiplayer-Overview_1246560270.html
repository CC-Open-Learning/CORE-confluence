CORE Framework : Current VARLab Multiplayer Overview

Created by Talon Ernst, last modified on Aug 01, 2024

Welcome to the VARLab Multiplayer Overview page. This page is just the landing page and holds all of the multiplayer documentation. If you are interested in checking out anything multiplayer, please visit the CORE Sandbox Repository as that is where everything is being kept for the time being.

The current goal with multiplayer is to allow for a DLX to be able to be played in a multiplayer setting, in either flatscreen or VR, utilizing Unity Lobby for a seamless lobbying system and using Unity Relay to let Unity handle hosting our DLX, allowing for a peer-to-peer architecture.

In the future, we hope to have the option to utilize a dedicated server architecture where we host a DLX on our own personal Azure server. This would change a few things, such as us having to handle encryption ourselves, but would give us more control over basically everything since we are hosting it.

The following is a list of notes about the packages mentioned, more can be added if necessary

  • ParrelSync and Multiplayer Play Mode essentially do the same thing, but both were mentioned in case there is a preference, try out both and pick which one you like more

  • There was an issue that Unity Lobby saw the ParrelSync clones as the same client. In order to fix this, the arguments of the ParrelSync clones need to be different. Unsure if Multiplayer Play Mode also faces this issue

  • Unity Relay handles encryption for us. However, if a dedicated server model is ever desired, encryption will have to be handled manually.
