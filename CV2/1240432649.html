Progress Indicators V1 for UI Elements

Progress indicators are a sophisticated piece of specialized UI that allows a user to keep track of the progression tasks within individual categories.

Key Terms

Category

A category is an element of the progress indicator that contains tasks. Categories consist of individual child tasks and a progress bar that indicates how many of those child tasks have been completed.

Task

A task is an element that is contained inside a parent category. Each task has its own individual progression and when completed, it adds up to the progression of its parent category.

Setting up your Scene

To get started with implementing progress indicators, you want to place the pre-made progress indicator prefab into your scene. Go to Packages → Velcro UI → Prefabs and drag and drop the prefab called Progress Indicator V1 in the scene view. Progress Indicators only have one field that the user can edit to customize their UI. The rest are required fields that should not be touched or unity events.

Field

Description

Category Asset

The reference to the category UI asset.

Do not change this field unless you know what you are doing.

Task Asset

The reference to the task UI asset.

Do not change this field unless you know what you are doing.

Categories

The categories and tasks in the progress indicator.

Event

Description

On Show

Invoked when the progress indicator is shown.

On Hide

Invoked when the progress indicator is hidden

On Progress Updated

Invoked when the progress of a task has been updated.

On Category Added

Invoked when a category is added to the progress indicator.

On Task Added

Invoked when a task is added to a category.

On Category Removed

Invoked when a category has been removed from the progress indicator.

On Task Removed

Invoked when a task has been removed from a category

Scripting

To begin using the progress indicator, you will need to create a script to control the progress indicator. For this example, I will name my script ProgressController.cs. Add this script to an empty game object in your scene.

Inside the script we will need to import the VELCRO namespace.

using VARLab.Velcro;

Now we will need a reference to the progress indicator inside the controller script. Once we have the reference to the progress indicator, we will have access to of all its public properties and methods.

[SerializeField]
private ProgressIndicatorV1 progressIndicator;

Now in you scene, assign the progress indicator object to the reference in your controller script.

Displaying the Progress Indicator

By default the progress indicator is hidden and must be enabled using the HandleDisplayUI() method.

Parameter

Description

title

The title text that shows at the top of the progress indicator. 

progressIndicator.HandleDisplayUI("Title Text");

You can use the Show()/Hide() methods to further control the display of the progress indicator.

progressIndicator.Show();   // Shows the progress indicator.
progressIndicator.Hide();   // Hides the progress indicator.

Adding Categories and Tasks

There are two ways to add both categories and tasks to your progress indicator. You can choose to add them before runtime using the inspector or during runtime through code.

Inspector

In the inspector of the progress indicator object you will see a list field called Categories. This field lets you add categories and in each of those categories is another list that contains the tasks.

This way of adding categories and tasks is helpful for those who prefer engine work or are not comfortable with scripting.

Code

Adding a category or task to the progress indicator is as simple as calling a single method.

// Adds a new category
progressIndicator.AddCategory("Category Name");

// Adds a new task to the category at the specified index
// The third parameter is the maximum amount of progress needed to complete the task
progressIndicator.AddTask(0, "Task Name", 5);

// If any index is invalid, an error will be logged to the console

Removing Categories and Tasks

Similar to adding, removing a category or task can be done by calling a single method.

// Removes the category at the specified index.
progressIndicator.RemoveCategory(0);

// Removes the task at the specified category and task indices.
progressIndicator.RemoveTask(0, 0);

// If any indices are invalid, an error will be logged to the console.

Adding/Removing Progress

Both categories and tasks have their own progress values but have different ways of obtaining progress.

The progress of a category is based on the number of its tasks that are complete. This means a category has its progress automatically updated.

A task must be given progress manually through code.

// Adds progress to the task at the specified category and task indices.
progressIndicator.AddProgressToTask(0, 0);  // Adds 1 progress
progressIndicator.AddProgressToTask(0, 0, 5); // Adds 5 progress

// Removes progress from the task at the specified category and task indices.
progressIndicator.RemoveProgressFromTask(0, 0);  // Removes 1 progress
progressIndicator.RemoveProgressFromTask(0, 0, 5); // Removes 5 progress

// If any indices are invalid, an error will be logged to the console.

Parameterized Events

Some progress indicator events send parameters that represent specific values related to the UI.

OnProgressUpdated

This event sends two integer values.

The first value represents the index of the category that contains the task that had its progress updated.

The second value represents the index of the task inside the category.

progressIndicator.OnProgressUpdated.AddListener(ProgressUpdated);

private void ProgressUpdated(int categoryIndex, int taskIndex)
{
    Debug.Log($"Progress updated at category {categoryIndex} in task {taskIndex}.");
}

OnCategoryAdded

This event sends one integer value that represents the index of the newly added category.

progressIndicator.OnCategoryAdded.AddListener(NewCategoryAdded);

private void NewCategoryAdded(int categoryIndex)
{
    Debug.Log($"New category added at index {categoryIndex}.");
}

OnTaskAdded

This event sends one integer value that represents the index category that contained the task that was just removed.

progressIndicator.OnTaskAdded.AddListener(NewTaskAdded);

private void NewTaskAdded(int categoryIndex, int taskIndex)
{
    Debug.Log($"New task added at index {taskIndex} in category {categoryIndex}.");
}

OnTaskRemoved

This event sends two integer values.

The first value represents the index of the category that contains the task that was added.

The second value represents the index of the new task inside the category.

progressIndicator.OnTaskRemoved.AddListener(TaskRemoved);

private void TaskRemoved(int categoryIndex)
{
    Debug.Log($"Task removed in category {categoryIndex}.");
}
