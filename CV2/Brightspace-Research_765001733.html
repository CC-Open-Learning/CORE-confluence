Objectives for Brightspace

  • Saving completion history (currently overwrites)

  • track progress (objectives and comments)

    • From LMS into the simulation, or vice versa, that provides information to the instructor about the progress

  • Results Export

    • Directly to instructor and econestoga

comments

adds a comment attached to a ep object

Release Conditions

releases resources to users that complete conditions - might work for making sure users complete the DLX before proceeding with the other course material

Authorization

Brightspace API has 2 authorization methods: Their legacy method and OAuth.

The legacy system uses an app id and app key to create a signature and verify the request.

OAuth uses an OAuth registered account to have OAuth verify the request.

Applications can register for OAuth to switch from the default legacy method.

As of now, it is unknown what method eConestoga is currently using.

Integration with Unity

Brightspace can use a .net controller to make calls to the api. Unity can implement .net if the library is imported(Currently running into issue where .net is outdated in unity and I cannot replace with the newer update files).

This implementation would require a completely new build of a .net controller.
