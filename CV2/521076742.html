CORE Framework : Building for WebGL + SCORM

Created by David Inglis, last modified by Jordan Partridge on Jul 31, 2024

This article outlines the steps necessary to create a working SCORM object for a WebGL build. Ensure the steps for installing the SCORM package have been completed before continuing.

The https://bitbucket.org/VARLab/scorm/src/main/ package for CORE now supports optionally creating a SCORM archive after building the WebGL executable, as of version 0.2.3. Follow the updated steps below to ensure the project is properly configured for SCORM.

Configuring SCORM Properties

Before building for WebGL, the SCORM Properties should be configured so that the WebGL + SCORM archive can be properly packaged.

The SCORM Properties are stored as PlayerPrefs and are local to each developer’s machine. These properties must be verified and updated each time a build is created.

In the Editor, select Tools → SCORM → Edit SCORM Properties to open the SCORM Properties window.

Identifier

The CORE build number including project code, sprint info, and date. This Identifier is used as the output file name.

Title

Name of the project.

Description

(Optional) Brief description of the project.

Module Title

(Optional) Name of the current module.

Launch Data

(Optional) Parameters to be provided to the LMS. Currently not in use.

Build Type

The type of build to be built for SCORM. If both of these are left blank, a SCORM package will not be created.

To enable automatic building of SCORM packages, check the boxes Development and/or Release at the bottom. Otherwise, they will be built as WebGL and will require you to manually export it.

Generating a SCORM-Wrapped WebGL Build

After ensuring the SCORM Properties are configured, open the File → Build Settings menu in the Unity Editor and make sure your Platform is set to WebGL, or switch to the WebGL platform if it is not currently targetted.

If this build is intended to be a release that real end-users have access to, make sure that Development Build is left unchecked.

Once the platform has been switched to WebGL, select Player Settings… in the lower left corner.

In the Player Settings window, select one of the available SCORM WebGL Templates under the Resolution and Presentation dropdown.

If the only available options are Default and Minimal SCORM WebGL Template, import the SCORM Templates by navigating to Tools → SCORM → Import WebGL Templates, then close and re-open the Player Settings.

These steps are outlined in the SCORM package documentation.

SCORM Fixed

Provides a fixed-size player window which defaults to 960x600.

Fullscreen can be enabled from the bottom-right of the player window.

SCORM Resize

The player window automatically resizes to fill all available space in the browser. May impact performance if window is frequently being resized.

Fullscreen can be enabled from the bottom-right of the player window

Next, ensure that the Compression Format has been set to Disabled (not Brotli or Gzip) in the Publishing Settings dropdown.

After closing the Player Settings window, build the executable by selecting the Build button in the Build Settings window. When the build completes successfully, a pop-up will appear indicating that “the SCORM Package has been published to {Output Folder}/{Identifier}.zip."

The zip archive will be generated in the build folder with the Identifier name specified in SCORM Properties. This archive is the SCORM-wrapped WebGL executable and is ready to be uploaded to eConestoga.

For a list of common issues that may arise while creating a WebGL build, check out the documentation on Known Issues With WebGL.

Manually Exporting a SCORM Package

This step is now only necessary if a new SCORM archive needs to be generated from the same WebGL build. Otherwise, a SCORM archive is automatically generated from the build steps described in the previous section Generating a SCORM-Wrapped WebGL Build.

After successfully building the WebGL project using the SCORM WebGL template, the project can now be exported as a SCORM package for use on eConestoga.

In the Unity Editor, select Tools → SCORM → Export SCORM Package.

In the SCORM Export Tool window, set the Folder Location to the directory of the WebGL build. Edit the remaining SCORM Properties as noted in the section Configuring SCORM Properties above.

When you are happy with your export settings, click Publish and save the exported .zip file to your computer.

Deployment to eConestoga

For information on how to deploy this SCORM package to eConestoga, check out the article Publishing a SCORM Package to eConestoga.
