CORE Framework : UI/UX guidelines and best practices

Created by Germán Gaussmann, last modified by Sana Javeed on Oct 06, 2023

Visual language

Containers

Containers should have solid soft dark colours.

#383840 100% > #000000 %80

Round corners and a slight drop shadow will help your design pop up from the background.

 DO:

 DON'T:

  • People with photophobia will thank you for using a dark theme by default. Make sure to contrast your text by using whites so that it is always readable.

  • Solid backgrounds help with readability so your content does not blend with what is behind it

Sizing & spacing

Make sure your components are not too close to the edge of your container and each other. Determine a margin for your container and keep it consistent.

Balance your elements so they feel centered in the container

 DO:

 DON'T:

  • Establish a “safe area” for your containers and make sure all the content is added inside those borders

  • Make sure that elements with the same hierarchy are always spaced in the same way.

  • Follow the Gestalt principles (similarity, continuation, closure, proximity, figure/ground, and symmetry & order) to help users identify which elements are related to each other.

Icons & illustrations

Always label your icons and use them sparingly. If this is not possible, use icons that your intended audience will be able to recognize immediately, being able to tell what their purpose is at a glance.

 DO:

 DON'T:

  • Icons can be useful when you need to save space but should be used sparingly (i.e. don’t fill a container with twenty different icons).

  • If an icon needs a tooltip for people to know what its function is, you should consider using another icon.

Relative Position: Menus/ Tools Controls

Position of Menus/ Tools Controls

Place the menus, controls, titles on the left or right edges of the display screen.

 DO:

 DON'T:

  • Users/students prefer to play DLX in the eConestoga playback area instead of full screen mode. This results in a narrow rectangle that is much wider than it is tall. Hence the icons on the top or bottom block the screen.
