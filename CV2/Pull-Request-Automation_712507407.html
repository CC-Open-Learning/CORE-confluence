Deliverable/Final Workflow

C# static code analysis

  • Testing within Visual Studio and VS Code to ensure code follows and confroms to VARLab style guide

  • Display conflicts as errors within the IDE

Docker Testing Environemnt

  • Docker environment that contains:

    • Visual Studio with Code Style configured to VARLab style guide

    • Unity Editor using version Unity 2021.3.24f1 as of 08 May 2023

  • This environment will automatically build and run the current branch after a Pull Request is submitted

Bitbucket/Automation Integration

Automate pull request fail conditions based on test results (represented as flags provided to Bitbucket)

Step 1: Utilize Visual Studio’s built in Code Style Tool, configured to enforce and adhere to VARLab style guide

Step 2: Test Unity build - does it compile without errors or warnings?

  • When submitting a pull request the following happens automatically:

    • the branch is cloned into the test enviroment

    • It is built using Visual Studio and Unity within the environemt, and the exectuable is run.

    • The results of this test will be passed back to Bitbucket

Step 3: TDD and Unit Test integration/automation

Step 4: Based on the results of the testing, the Pull Request is automatically approved or denied
