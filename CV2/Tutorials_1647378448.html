Tutorials

Tutorials are simple modals used to provide learners with information on how to navigate and interact with the DLX. Tutorials have four labels (Name label located at the top left of the dialog, and the 3 button labels) that do not change from one tutorial section to another. The header/description labels, and the image will change in each tutorial section. The tutorial also has an optional dimmed background.

https://www.figma.com/design/om9GwMhkfeJpcADbKz4ov8/CVRI-Design-System?node-id=8087-20371&t=a1xskNDZtPSloqUi-4

Setting up your scene

To get started with implementing a tutorial in your scene

  1. Drag in the tutorial prefab into the scene hierarchy at “/Packages/Velcro UI/Prefabs”

  2. Replace the PanelSettings reference from VELCRO to your project’s PanelSettings asset

  3. Create a TutorialSO, and reference it in the prefab. Right Click/Assets > Create > ScriptableObjects > TutorialSO

  4. Hook up UnityEvents as required

Root VisualElement of the UI

HandleDisplayUI()

Definition:

public void HandleDisplayUI()

 

Comments:

  •  This method is intended to be a single access method to populate the tutorial with content, and display it

  • Invokes OnTutorialShown since Show() is called after content is populated

SetContent()

Definition:

public void SetContent()

 

Comments:

  • Populates all content of the tutorial. The [Name] label, and button text is populated, then the carousel indicators

  • Section content is populated for the first index in the TutorialSections list

  • These methods are called

    • ChangePreviousButtonState()

    • PopulateCarouselIndicators()

    • PopulateTutorialSection()

PopulateTutorialSection()

Definition:

public void PopulateTutorialSection(int sectionIndex)

 

Comments:

  • Populates the TutorialSection with new content

  • This includes the [Header]/[Description] labels and [Image] VisualElement 

 PopulateCarouselIndicators()

Definition:

public void PopulateCarouselIndicators()

 

Comments:

  •  Populates the clickable carousel indicators under the image. The template is cloned List<TutorialSection>.Count times and index matches up with the list in the SO

  • The first carousel indicator is set to active by default

  • The buttons call JumpToIndex(index) on click

    • The index is stored in the button’s tooltip field

 ChangeActiveCarouselIndicators()

Definition:

public void ChangeActiveCarouselIndicators(int sectionIndex)

 

Comments:

  • Sets the current and previous active carousel indicators so the newly clicked one has the active styling

  • Changes the active carousel indicator styling

 ChangePreviousButtonState()

Definition:

public void ChangePreviousButtonState()

 

Comments:

  • Sets the display style of the previous button based on the currentIndex. If the index is > 0, then the previous button will be shown, otherwise it will be hidden

JumpToIndex()

Definition:

public void JumpToIndex(int sectionIndex)

 

Comments:

  • Moves the tutorial to the desired page index without needing to go one by one forwards, or backwards

  • This re-populates the TutorialSection content, and changes the active carousel indicator.

  • These methods are called

    • ChangePreviousButtonState()

    • ChangeActiveCarouselIndicators()

    • PopulateTutorialSection()

  • Invokes OnTutorialJump<int>

AdvanceTutorial()

Definition:

public void AdvanceTutorial()

 

Comments:

  • Advances the tutorial to the next page index

  • This re-populates the TutorialSection content, and changes the active carousel indicator

  • These methods are called

    • ChangePreviousButtonState()

    • ChangeActiveCarouselIndicators()

    • PopulateTutorialSection()

  • Invokes OnTutorialForward

  • Invokes OnTutorialComplete and hides the root of the tutorial if progressing past the last page of the tutorial

ReverseTutorial()

Definition:

public void ReverseTutorial()

 

Comments:

  • Reverses the tutorial to the previous page index

  • This re-populates the TutorialSection content, and changes the active carousel indicator

  • These methods are called

    • ChangePreviousButtonState()

    • ChangeActiveCarouselIndicators()

    • PopulateTutorialSection()

  • Invokes OnTutorialBackward

Show()

Hide()

public void Hide()

Unity Events

Event

Description

OnTutorialShown

Invoked when the tutorial is shown to the screen from

  • HandleDisplayUI()

  • Show()

OnTutorialHidden

Invoked when the tutorial is hidden from the screen from

  • Hide()

OnTutorialForward

Invoked when the tutorial is advanced 1 page index from

  • AdvanceTutorial()

OnTutorialBackward

Invoked when the tutorial is reversed 1 page index from

  • ReverseTutorial()

OnTutorialComplete

Invoked when the tutorial index is advanced past the last index from

  • AdvanceTutorial()

  • Skip Button

OnTutorialJump<int>

Invoked when the tutorial index is changed using the carousel indicators from

  • JumpToIndex()

  • Right Click/Assets > Create > ScriptableObjects > TutorialSO

The text of the secondary button (Previous button in the bottom right)

The text of the tertiary button (Skip button in the bottom left)

Contains the image, header, and description labels for each section:

  • string Header - The [Header] label of the tutorial

  • string Description - The [Description] label of the tutorial

bool IsBackgroundDimmed

Whether the background behind the UI is dimmed or not
