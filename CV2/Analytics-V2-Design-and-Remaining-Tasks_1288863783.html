CORE Framework : Analytics V2 Design and Remaining Tasks

Created by Nilupul Vithanage, last modified by Richard Colbourne on Aug 15, 2024

Overview

The Analytics V2 system is a redesigned and enhanced framework intended to streamline and optimize analytics handling within our application. It encompasses essential components such as core analytics functionalities, network management, time tracking, and SDK integration. This system is crafted to efficiently log user interactions, monitor time-based events, and manage analytics data under various network conditions.

Remaining Tasks

Send Network Infomation.

  • In NetworkHandler.cs, a method named PingServer() is used to measure the network response time. Upon retrieving this information, we need to ensure that a network information event is triggered and sent to log the network status and response details.

Browser Close Handling

  • We can handle “unexpected shutdown handling” from the browser side. There is a way to communicate with the browser and Unity application. We can send a javascript call(beforeunload) from the browser to unity application using browser scripting. This requirement is an optional one.

Analytics for external simulations.

  • We can add analytics to external simulations by modifying the build files. Note that the amount you can track is greatly reduced and their is still a potential issues with gathering the learner ID. A proof of concept has been made for how to implement analytics into external sims.
