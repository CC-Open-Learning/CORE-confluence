CORE Framework : Testing NodeCanvas Graphs in DLX Projects

Created by Jonah Pearce on Mar 01, 2023

Testing is essential to any software development, and we need to test our graphs in our DLX projects. This guide will go over how we can test graphs.

Standard Testing

Create test cases: Create test cases that cover all possible flows and situations that could occur with the graph or graphs of your scene. The test cases should include edge cases and unexpected scenarios to ensure the graph is solid.

Run the test cases: Run the test cases in the Unity Editor by playing the scene and interacting with the graph and its useful visuals. You can pause and modify global graph settings during runtime.

Automated Testing

A more efficient method when dealing with graphs is having completely automatic tests that we can run.

  1. First, include a testing framework in your Unity project to write test cases in C# that will cover all the possible flows and situations your graphs may have.

  2. Write test scripts that automate the testing process. Test scripts should include code that interacts with the NodeCanvas graph, simulates user input, and records the results of each test case.

  3. Run the automated tests using the testing framework. The testing framework will automatically run the test scripts and record the results of each test case.

This is an example of a simple graph test case that finds a graph in the scene and checks that is a valid graph and not corrupted.

using UnityEngine;

public class GraphTests {

    [Test]
    public void TestExistingGraph() {
        // Get a reference to an existing Graph instance
        Graph graph = GameObject.FindObjectOfType<Graph>();

        // Verify that the Graph is not null
        Assert.IsNotNull(graph);

        // Verify that the Graph is valid
        Assert.IsTrue(graph.IsValid());
    }
}
