Overview

The Point & Click Navigation package lets developers easily set up a navigation system with custom waypoints. This system is mainly built on top of Unity’s AI Navigation and Cinemachine packages. Additionally, it uses some components of our in-house CORE Cinema package.

How it Works

The Point & Click package consists of three main components: the player, camera and waypoints.

The Player

Unity’s AI Navigation package mostly drives the player. Features such as obstacle avoidance are inherited from the AI Navigation package.

The Camera

There are actually two cameras included in this package, which are the Main Camera and the POV Camera. The Main Camera is basically the default Unity Main Camera. The POV Camera, on the other hand, is a Cinemachine Virtual Camera. It is named “POV Camera” because the Aim section of its CinemachineVirtualCamera component is set to POV.

The Cinemachine package controls the POV Camera, and the POV Camera drives the Main Camera. POV Camera and Main Camera will always have the same rotation values. It’s important to note that the camera rotations are independent of player rotations.

Waypoints

These are the points of interest (POIs) that the player can be walked into. Once the user clicks on one of these waypoints, the player will navigate into it while looking straight.

How to Setup

The setup is pretty much straightforward. There is one prerequisite, which is setting up the floor/ground. We will need a NavMesh Surface on the ground GameObject for the player to walk.

Create a NavMesh | AI Navigation | 1.1.5 (unity3d.com)

Creating the Main Handler

Once we have NavMeshSurface attached to the ground GameObject, we can go ahead and create the main “handler/controller” and waypoints in the scene for point-and-click navigation.

Go to GameObject → VARLab → Point Click Navigation Handler to create the Point Click Navigation GameObject on the scene. Alternatively, we can right-click on the Unity Hierarchy to access this menu.

You should have only a single Point Click Navigation Handler in the scene.

[Optional] Setting a Look-at Target:

You can set a target to look at once the player reaches a destination/waypoint.

This can be done by serializing a GameObject into the Look At After Walking field. If we leave this empty, the player will keep looking straight after walking.

Creating Waypoints

As you may have already figured out, you can use the same context menu shown above to instantiate waypoints by clicking on Navigation Waypoint. You can have multiple waypoints in the scene and put them under any GameObject.

And that’s it! Now you have a functional Point & Click navigation system in your project.
