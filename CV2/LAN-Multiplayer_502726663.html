CORE Framework : LAN Multiplayer

Created by Lisa Nguyen, last modified on Jul 25, 2022

This article will go over how to host and connect to the same game instance over LAN using the Network Discovery add-on.


LAN Network Discover Component: https://github.com/Abdelfattah-Radwan/Fish-Networking-Discovery

Setting Up LAN

Download the zip from Github. Extract the files and place it in the ‘Plugins’ folder under:

…\ProjectName\Assets\FishNet\Plugins.

In your Network Manager add ‘NetworkDiscoveryHUD’ and ‘NetworkDiscovery’ components. Set your secret, port (make sure the port you use is different than your transport’s port) and discovery interval fields.

Now when you enter play mode or run a build you should see something like this:

The NetworkDiscoveryHUD adds the top right buttons to your UI.

To allow for clients to see an active server, start your server then click ‘start’ under ‘advertising’. Your server will be discoverable to clients that are searching on the same network.

To connect to a server click ‘start’ under ‘searching’ and click on the ip address of the server you want to connect to.

Using Code

To use these code snippets you will need to have a Network Manager with a NetworkDiscovery Component. The HUD is not needed if you’re planning on using the code to customize your client/server start.

Starting Server + Advertising

// Start server and start advertising
InstanceFinder.ServerManager.StartConnection();
FindObjectOfType<NetworkDiscovery>().StartAdvertisingServer();

// Stop advertising the server
FindObjectOfType<NetworkDiscovery>().StopAdvertisingServer();

Search for Servers

// Start searching for servers
FindObjectOfType<NetworkDiscovery>().StartSearchingForServers();

// Stop searching for servers
FindObjectOfType<NetworkDiscovery>().StopSearchingForServers();
