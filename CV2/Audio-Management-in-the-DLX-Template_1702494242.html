CORE Framework : Audio Management in the DLX Template

Created by David Inglis, last modified on Mar 31, 2025

Summary

Audio Management for DLX leans heavily on the existing Unity Audio system, leveraging audio Sources and Mixer Groups in order to route and control playback for in-game dialogue and sound effects.

Instead of a CORE Framework package for audio, a default audio management configuration is provided within the DLX Template, which downstream projects are free to modify as it suits the needs of their specific learning experience.

Audio Mixer

The Audio Mixer and its corresponding Audio Mixer Groups handle the output for all audio in a digital learning experience.

The default mixer is located at Assets/Audio/Main Audio Mixer.mixer and has the following configuration:

  • Global Mixer

    • Dialogue Mixer Group

    • Sound Effects Mixer Group

The Global mixer controls both child mixer groups attached to it, so any adjustments made to this mixer, such as muting or lowering the volume, will be applied to the children as well.

The Global mixer is used to mute all sounds in the sim through the Audio Manager Volume API.

Dialogue and Sound Effects mixer groups, also referred to in code as Channels, are used to separate audio within a simulation so that they can be adjusted separately. Additional audio effects can be applied to one or more mixer groups if it suits the project’s needs.

Play Sounds through the Audio Manager

The Audio Manager provides a straightforward API for playing audio clips during a simulation, which can be called from other classes by referencing an AudioManager object in the scene, or invoked through serialized event handlers.

Channels

As noted previously, the Audio Mixer Groups are also referred to as Channels. The Channel enumerator is used throughout the various APIs to easily refer to a given mixer group, as these mixer groups are themselves not exposed through code. The available channels are as follows:

None

A null case enumerator used to indicate that audio should not be played

Global

The global audio channel, used exclusively for muting or adjusting audio volume for the entire DLX. Audio clips are typically not played through the Global channel.

Dialogue

All dialogue or voice-over clips should be routed to this channel. A volume control for Dialogue audio is exposed through the VELCRO UI settings menu.

Effects

The channel used for any audio that is not dialogue/voice-over. Most playback functions default to this channel. A volume control for Effects audio is exposed through the VELCRO UI settings menu.

Playback Options

Options can be provided to the Audio Manager during playback to control the volume and output of the audio clip provided. Each playback function in the API in the following section takes an optional PlaybackOptions object, which is made up of the following fields:

Channel

Specifies the audio channel to use, typically either Effects or Dialogue. Defaults to Effects

Volume

Adjusts the volume of a clip by this percentage. Default is 1 for 100%

Loop

Indicates whether the audio clip should loop until it is stopped

Interrupt

Indicates whether the played clip should interrupt audio currently playing on the same channel.

Audio Manager Playback API

Play(AudioResource resource, PlaybackOptions options = null)

Plays an audio resource such as an AudioClip or AudioRandomContainer with optional playback settings, using default values if none are provided

Stop(PlaybackOptions options = null)

Stops audio on the channel specified in the PlaybackOptions, defaulting to the Effects channel if no custom options are provided.

Pause(PlaybackOptions options = null)

Pauses playback on the channel specified in the PlaybackOptions

UnPause(PlaybackOptions options = null)

Attempts to un-pause playback of a cached audio resource on the channel specified in the PlaybackOptions

Playing Sounds through Audio Sources

Playing any sound within Unity requires at least an Audio Listener, typically attached to the Main Camera, and an Audio Source component, attached to any object in the scene to provide either 2D (stereo) or 3D (spatial) audio.

The DLX Template provides two Audio Sources by default: one mapped to the Sound Effects mixer group and the other to the Dialogue mixer groups. These Audio Sources are used by the Audio Manager to enable sound for a DLX using the provided Audio Manager Playback API.

To take advantage of spatial audio, additional Audio Sources can be placed around a scene with the Spatial Blend value set to 1. Playing spatial audio can be triggered from events or other scripts which reference these additional Audio Sources.

Ensure any new Audio Sources added to a scene have their Output field set to either Dialogue or Sound Effects.

Volume Control

The Volume API described below uses a decibel (dB) scale to indicate volume for a given Audio Mixer Group. As mixer groups are not exposed publicly through code, adjusting their values requires “exposed parameters” to be configured. The volume values of all three mixer groups described previously are exposed using the following self-explanatory parameters:

GlobalVolumeParameter, DialogueVolumeParameter, SoundEffectsVolumeParameter

Audio Manager Volume API

SetAudioEnabled(bool enabled)

Disables (mutes) or enables (unmutes) audio globally across the DLX, using the Global mixer.

SetVolume(string param, float decibels)

SetVolume(Channel channel, float decibels)

Directly sets the decibel volume value of a mixer group for the corresponding Channel value or the name of an exposed mixer parameter. Expects a value between -80dB and 0dB, though values outside this range technically work.

SetVolumePercent(string param, float percent)

SetVolumePercent(Channel channel, float percent)

Sets the volume value of a mixer group for the corresponding Channel value or the name of an exposed mixer parameter to a specified percentage by converting percent to the appropriate decibel value between the range of -80dB and 0dB.

A percentage value of 0 is undefined and should not be provided to this function. Values should be within the range of 0.0001 to 100, though this is not dictated by the function itself.

float GetVolume(string param)

float GetVolume(Channel channel)

Returns the decibel volume value of a mixer group for the given Channel or exposed mixer parameter.

float GetVolumePercent(string param)

float GetVolumePercent(Channel channel)

Returns a value between 0 and 1 corresponding to the decibel volume value between -80dB and 0dB of the mixer group for the given the Channel or exposed mixer parameter.

Volume Control through VELCRO UI Settings

Simple

The Simple settings menu template from VELCRO provides a simplified configuration interface with one volume slider. This slider should be mapped to the Global audio mixer so that all audio in the sim will reflect these changes.

Any Audio Sources in the scene should still specify one of the Dialogue or Sound Effects mixer groups, as this keeps sound effects organized if a project expands its audio profile in the future and then requires more complex volume controls, as described below.

Complex

If a simulation has both dialogue and sound effects, the Complex settings menu from VELCRO will likely be preferred. Learners are provided with two volume sliders, one for each of the available mixer groups, and a “mute” toggle switch which will toggle audio through the global mixer.
