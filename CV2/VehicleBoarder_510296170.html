CORE Framework : VehicleBoarder

Created by Geyang Liu on Aug 18, 2022

The VehicleBoarder class handles the logic of boarding and unboarding players from vehicles, using the public methods Board and Unboard. Here are the necessary steps to step it up:

Attach the script to the player

The script can be attached to any gameobject, but it is recommended to attach it to the player tracker offset. There are three public fields:

For the player body field, assign the AutoHandPlayer script. This component will be disabled when the player is boarded on a vehicle.

For the Original parent field, assign the player root gameobject. The playerbody will be renested here when it unboards a vehicle.

For the Vehicle Input field, leave it blank for VR players. For PC players, attach a new player input component here and assign it to the field. The vehicle will accept input using this component after boarding.

Setup boarding event

For each vehicle, you need to manually setup an event to trigger the boarding function. In the jetbike model, the boarding event is binded to the left handle. When the handle is grabbed, it will board the player onto the vehicle.

Note: This solution will not work correctly if there are multiple player instances in the scene. A new solution is needed.

For PC mode, the boarding method is called in the OnInteract method in the vehicle base class.

Setup unboarding event

For each player, you need to manually setup an event to trigger the unboarding method for VR mode. Currently the event is binded to the secondary button of the left hand controller.

For PC mode, the unboarding method will be triggered by by the player input asset.
