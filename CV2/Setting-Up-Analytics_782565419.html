CORE Framework : Setting Up Analytics

Created by Joshua Moore, last modified on Aug 16, 2023

This page assume PlayFab has been installed, as well as the CORE Analytics package. If these are not installed please see the following page.

https://varlab-dev.atlassian.net/l/cp/NR07KCmq

Analytics Prefab

Once everything has been installed you will need to drag a prefab from the Analytics Package into your first/start up scene if it is a multi scene DLX, or if the DLX only has one scene you will drag it into that scene.

Prefab Location

This is the location of the Prefab to be placed in your first/initial scene. This is because the object will persist through scenes. If your DLX has one scene you can just place it in that scene.

Integrating with SCORM

There is a known edge case, if you have a project that only has 1 scene that also has a restart functionality please expand the dropdown box below.

Singleton replacement for single scene

Due to the behaviour of the implementation of the singleton in the Analytics object. There is a known issue where if you have a single scene in your DLX and there is the ability to restart the experience which will reload the scene breaking a reference to the login in the new scene. to fix this check this box on the CORE Analytics script.

This will cause the new object to replace the original analytics object in the scene maintaining references.

With all these steps complete you are now able to start using the system! For how to send events with the analytics system please see https://varlab-dev.atlassian.net/l/cp/NjN0NFJX
