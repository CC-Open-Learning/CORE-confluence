Overview

Telemetry

Time

In Azure PlayFab, measuring and analyzing the ‘time spent within the project’ will use the telemetry feature. By subtracting the exit time from the entry time of a user, the time spent in the game can be measured. Additionality, specific parts of a simulation can be measured to identify which areas are more engaging and which areas may need improvement.

Telemetry allows you to collect user data in real-time, including information about how long users spend in the simulation. You can use this information to analyze user behaviour, identify patterns, and make data-driven decisions to improve the experience of the simulation.

Conversion Rate

In Azure PlayFab, measuring and analyzing the ‘conversion rate’ will use the telemetry feature to track user interactions within the simulation.

For example, if the goal of the simulation is to complete a specific task, you can track the number of users who attempt the task and the number of users who successfully complete the task. The conversion rate can then be calculated by the number of successful completions by the total number of attempts.

A/B testing is another way to measure conversion rate, as this involves creating two versions of a specific feature or task within the simulation and measuring the conversion rate for each version. This can help identify which version is more effective at achieving the desired learning outcomes and make data-driven decisions to optimize the simulation.

A/B Testing

With A/B testing, you create a series of 'buckets” into which you partition your users randomly, with weights assigned to each bucket. Each bucket is assigned a different value. Users can see the value assigned to their bucket, but not others. Afterward, you compare each of the buckets and the results to determine the most efficient bucket based on the assigned value.

Note: PlayFab can A/B Test in-game economy.
