CORE Framework : Using the Analytics Configuration File

Created by Richard Colbourne, last modified on Aug 15, 2024

Overview

Properties of the configuration file

Making a Build

At this point, you should have already created the scriptable object for the analytic configuration. If you don’t have this file, click here to see how to create one. This configuration file allows you to change how to package works within your DLX. With this you can set the DLX environment, how often you send out network data, how frequently you check if the user has a connection to the internet, and set the build number of the DLX into the package.

Overview

Properties

The Environment Type property has 3 options and allows us to filter the data in the backend.

  • Development is to be set during the development process.

  • Production is to be set on the build that will be released to students.

  • Test is to be set on builds that are released internally for play testing.

We recommend that this is left on development on your main branch and only changed for the Friday play test (Test), or the student release (Development) builds.

The Network Events property has 3 options and is used to gather network information.

  • None, will not track any information.

  • Startup, will do a ping check at the beginning of the sim.

  • Continuous, will do a ping check continuously throughout the sim in the time interval set in the next property.

Note: This feature currently doesn’t send any network data to PlayFab. However, it should be set so that it will already ready for when the update comes with the implementation.

The Network Ping Interval is used to determine how often we are going to be pinging eConestoga. The value set in this field is represented in seconds.

The Network Check Interval is used to determine how often we are going to for internet connection when we detect the learner has gone offline. This value is represented in seconds.

The Build Version Number is to be set before the build is created and needs to match the build number of the DLX.

Making a Build.

You are going to want to change 2 properties of the config before you build. You’re going to want to change the environment type to the environment you’re building for, and you’re also going to want to change the build version number to match the build number of the DLX. After these 2 fields are updated, you’re good to build your DLX.
