Overview

Attention: To successfully examine the CORE Sandbox implementation, the installation of PlayFab and the VARLab analytic package must have been completed.

If installation is not complete, refer here for more instructions: https://varlab-dev.atlassian.net/l/cp/U7EyYxuJ

This page will provide a simple implementation of the analytics package and how to capture analytic events using UI Elements on Unity. The following example assumes that the VARLab analytic package is installed on your Unity project and that you have installed and signed into the PlayFab SDK.

Note: in the PlayFab SDK select which game title you want to use, or you won’t be raising any events.

Interactable Analytic Tutorial / Example

To find an example of the implementation of the analytic package for interactable elements, check out the VARLab CORE Sandbox Development Environment under the branch CORE-89-Analytics-Tutorial-Updated-Package.

Fork the repository to have a local version of the tutorial, and navigate to the scene called Interactable.

All of the resources that are not in the analytic package will be found in the scene hierarchy and in the Scripts folder. The specific scripts used for this scene will be in the Interact Analytics folder.

Developer Note: Scripts are coupled to Unity in this example

  • More information can be found in the drop down below.

Functional Implementation

The Interactable analytic tutorial uses a functional approach to the logic and implementation with none of the Unity or VARLab Analytics decoupled from the scripts. If you want an example of interfaces and wrappers that decouple the implementation from Unity and VARLab Analytics for potential testing, then check out the UI implementation.

LoginLearner

Note: the analytic package provides a prefab of an Analytics gameObject, more information can be found here: https://varlab-dev.atlassian.net/wiki/spaces/CV2/pages/782565419/Setting+Up+Analytics#Analytics-Prefab

  • The CORE Sandbox examples did not use this prefab, however the implementation is the same as the prefab. For an explanation of the black box implementation of the analytics prefab object, please refer to here: https://varlab-dev.atlassian.net/l/cp/dg1yshzm

In this tutorial, the LoginLearner method from the analytic package will be handled by the Player Controller script on the Player gameObject (Ex. 1). The Player Controller has 1 serialized component, which is the CORE Analytics gameObject.

The LoginLearner method only accepts a string as a parameter for the Learner ID, which in this tutorial is hard-coded in the Start() of the Player Controller script. Therefore, when the program is started, then the LoginLearner information will be sent to PlayFab during runtime. However, this can be implemented in a number of different ways depending on the design of the project.

Note: The LoginLearner method is only captured event that will not be listening for analytic events. The LoginLearner method is the only method that relies on the CORE Analytics class from the analytic package, but will still need to be serialized onto a gameObject due to the mono behaviour.

Developer Note: Based on the Roll-A-Ball tutorial and the LoginHandler needs to be on a gameObject

  • More information can be found in the drop down(s) below.

Roll-A-Ball tutorial

The implementation of interactable analytics is a simple example of ‘how’ analytics can be used in a Unity project. The base for this tutorial was the Roll-a-Ball tutorial by Unity.

LoginHandler

LoginLearner can be applied to any script but does need to be attached to a gameObject since the analytic events class from the analytic package is a mono behaviour. In addition, the logic used for determining the information passed to the LoginLearner method depends on the design of the project. The LoginLearner might be a user input or might need an API call from SCROM to get the LearnerID.

Quiz Components

The quiz component uses the scriptable object called the Analytics Channel. More information can be found by navigating to the “Sending Default Events” section on the page: https://varlab-dev.atlassian.net/l/cp/kyEdsk6X

QuizAnswer Script

With the use of the Analytics Channel, information can be collected and stored in objects. In this example, the quiz component for the correct or incorrect button is a gameObject with the defined quiz properties:

  • isCorrect

  • QuestionName

  • SectionNumber

The components of a quiz object are defined in the editor with the Quiz Answer script (Ex. 2), then extracted in the QuizHandler script to determine if the interactable object has a quiz component.

QuizHandler Script

The QuizHandler script is attached to the Player gameObject and will need the Analytics Event class from the analytics package. The Analytics Event is the same scriptable object from earlier. The Quiz Handler has four serialized components (Ex. 3):

  • Quiz Answer

  • Quiz Complete

  • Num of Sections

  • Analytics Event

The Quiz Answer is left unserialized as this will be populated by the Component of the interactable gameObject passed to the script when the Player gameObject collides with the object. The Quiz Complete is the class that handles the DLXComplete analytic event. The Num of Sections is the tutorial logic to determine how many and how long each section is in the project. In this example, section 1 has 2 questions while section 2 has 1 question. Based on the logic of the tutorial, a user cannot progress to quiz 2 without completing quiz 1.

When the Quiz Handler is passed a collided interactable gameObject that has a component of QuizAnswer (QuizHandlerInvoke()), then a method is called to listen to this analytic event and raise it to PlayFab. After the collision with the interactable gameObject with a QuizAnswer component will raise a ProgressEvent, then the state will check if the Section is complete. If the 2nd question of Quiz 1 is passed to the QuizHandler, then QuizHandler will raise two events; one for the ProgressEvent and the other for the SegementCompleteEvent.

QuizComplete Script

With a SegmentCompleteEvent raise, the method from QuizComplete is called to validate if the program is complete or not. Once the DLX is complete, a DLXComplete event will be raised to PlayFab. The Quiz Complete script is attached to the Analytics gameObject and has the Analytic Event listener as a serialized component, where the Analytics Event is the same scriptable object as before (Ex. 4).

Developer Note: QuizHandler and QuizComplete need to be on a gameObject

  • More information can be found in the drop down below

QuizHandler and QuizComplete

The implementation of interactable analytics is a simple example of ‘how’ analytics can be used in a Unity project. The Quiz Handler and Quiz Complete script can be placed on any gameObject but does need to be on a gameObject since the analytic events class from the analytics package is a mono behavior.

In addition, the logic used for determining when and how a segment is completed is based on the design of the Unity project. The tutorial is really meant to show how to raise these analytic events in a Unity project.

Summary

These analytic tutorials are a tool to help provide a framework or guideline of how to use the CORE analytics and analytic events from the analytic package. The implementation is a general use case but hopes to be beneficial in understanding and easy to reverse engineer.

In the interactable element implementation, there is a functional approach to the logic and implementation with none of the Unity or VARLab Analytics decoupled from the scripts. The UI Implementation provides an example of using interfaces and wrappers to reduce the dependency on these namespaces to provide easier unit testing options. If you need an example of the implementation with interfaces and wrappers, then please refer to this example.
