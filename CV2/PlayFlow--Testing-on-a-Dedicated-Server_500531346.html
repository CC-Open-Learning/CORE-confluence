CORE Framework : PlayFlow- Testing on a Dedicated Server

Created by Lisa Nguyen, last modified on Nov 14, 2022

As of 07/18/22 PlayFlow does not support Multipass. Make sure you are only using one transport when making a WebGL build.

PlayFlow- Server Hosting

PlayFlow can be used to create and host a dedicated server to test WebGL builds using Fish-Net. We will be using the Bayou transport for WebGL builds.

Asset Store: PlayFlow Cloud | Network | Unity Asset Store

Documentation: PlayFlow Cloud - PlayFlow Cloud Documentation

Tutorial Video: PlayFlow - Dedicated Server Tool, Unity Multiplayer with Fish-Networking - YouTube

Setup Steps

  1. Get the PlayFlow asset from the Unity Asset Store and add it to your project.

  2. Open the PlayFlow window and click “Get PlayFlow Token”, you will be redirected to a webpage where you will be prompted to sign in with a Google account.

  3. Copy the app token and paste it into the App Token field in the PlayFlow Cloud window. Copy the port and put it into the port field of the Bayou transport.

  4. Before you can upload your server make sure that your current build platform is Linux Dedicated Server.**

  5. Click ‘Upload Server’, your first upload may take a long time. After the server is uploaded you can switch to another build platform. ‘Use Wss' in the Bayou script inspector and 'Enable SSL’ in the PlayFlow window have to be unchecked when you upload the server.

  6. After uploading, check off the ‘Enable SSL’ box and start the server. When the server starts you will get the ‘server url’ and ‘ssl port’ which you use to update the Bayou transport script. For client builds and editor testing, check off the ‘Use Wss’ box in the Bayou inspector.

  7. From here these fields should not change. You can use ‘Build and Run’ or run it in the editor to test your project before uploading the WebGL build to an external site (ex. Itch.io or GitHub Pages).

**When you upload your server, it has to use the port that was given in the PlayFlow Cloud Dashboard. Once you start your server and get the SSL port, replace the one in Bayou, connecting clients will be using the new port.

The free plan has an active time limit of 60 minutes. This limit can be reset as many times as you want by clicking the ‘Restart Server’ button.
