CORE Framework : VR Multiplayer

Created by Lisa Nguyen, last modified on Aug 09, 2022

This section uses a combination of Fish-Networking and the CoreV2 VR Package.

Getting Started

If you're using a Dell Precision with a USB-C adapter

Restart your device and go into the BIOS menu.

Using your arrow keys go into ‘Display’.

It should be the last option, but the parameter that says ‘Direct Output’, turn that ‘ON’ then save and exit.

The next time you boot up your device the headset should be able to connect and you should be able to run the room setup.

Source: Connecting Cosmos Elite to Dell Precision 5670 - Vive Console Software - Community Forum (htc.com)

See CORE-39 - As a developer I want to have VR work with Fish-Networking Archived for updates.

Import the VR package to your Unity project. See Using Private Bitbucket Repositories on how to use packages. Also take a look at VR for more information about the package itself and installation information.

Add the Fish-Net asset into your project via the Unity Asset Store. Add ParrelSync if needed.

If you’re using the Vive Cosmos make sure the plugin is properly set up. See Vive Cosmos Plugin for instructions.

Player Prefab and AutoHands

When you look into the CoreV2 VR package you’ll see two player prefabs, VRPlayer and PCPlayer. The package is read-only, so copy and paste the prefab that you want to use into the project’s Assets folder.

Go into the prefab and under the parent add a ‘Network Object’ script, leave the settings of the script as default. After adding this you will be able to use this prefab in the Player Spawner script in the Network Manager.
