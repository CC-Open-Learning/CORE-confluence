This page is not meant to be a comprehensive beginner tutorial, however it will cover some basics in the beginning and gradually get less detailed towards the end. This page is intended to teach you how to approach implementing new UI in UI toolkit once you are familiar with the bare minimum. I recommend viewing the page Introduction to UI Toolkit and the resources at the bottom of that page for beginner tutorials.

How to Approach Creating New UI

  1. Create a blank UXML document by Assets/Right Click > Create > UI Toolkit > UI Document

  2. (Optional) Box your UI

    1. The first step to implementing UI is structuring your hierarchy. Styling and functionality can come afterwards. Your hierarchy will usually be full of invisible wrapper classes and parent VisualElement objects that simply exist to shift elements around in your UI. A good way to help visualize these elements is by boxing your UI design. This step is optional. It can help developers who not familiar with UI Toolkit, but is by no means required. What I mean by this, is to literally draw boxes around the elements to see the building blocks of VisualElements/<divs> in your UI.

Default:

Boxed

  • Red: The parent most VisualElement that wraps the entire card. This has the size and alignment rules for children elements

  • Orange and Yellow: Separate the parent into the larger usable blocks such as #Top, and #Bottom that have different justification/alignment

  • Purple: Description label of the card in the #Bottom block

  1. Structure your UXML

  2. Style your UXML through USS classes. These should point to colour variables in the Dark/Light theme files

  3. Write your script

UI Builder Setup

  1. Set your canvas larger than default. I like to set it to 1920x1080. You can also set it to match your editor window.

  1. Link your required USS filles. Most UI will have velcro-structure.uss (margin, positioning, layout classes), velcro-style.uss (colour, border, typography classes), and their specific sheet for this it will be rt-card.uss.

Structuring & Styling UI

I’ll be using USS classes from velcro-structure.uss, and velcro-style.uss to do this. These classes can be applied to elements by dragging them from the USS class selector panel on top of the element in the hierarchy panel or by typing in the class name in the inspector and pressing “Add Style Class to List”.

1. Create the Parent-Most UI Element

We can start with an empty VisualElement container. Here, I named it “Card”. You can do this by dragging the VisualElement in the Library panel into the Hierarchy panel and renaming it.

By default, Unity will set an override inline style of flex-grow: 1. This means the element will expand to take up all available space. We don’t want this behaviour for the card, so we will Right Click > Unset.

We don’t have access to a Figma design for this card, so for the sake of this example, the card will be 330px x 330px. Over in the style sheet, create a USS class to style your base card element. 

.card {
    width: 330px;
    height: 330px;
}

Make sure to save changes in the UI builder before saving changes to the USS/TSS files. Unity will discard all changes if you save the USS/TSS files first. It is a big headache.

Now apply the USS class to the element by either dragging it onto the element, or by adding it in the inspector.

2. Create the #Top and #Bottom Elements

Add two more empty VisualElement containers. Here, I named it “Top” and “Bottom”. Unset the override inline flex-grow: 1 that Unity applies (Right Click > Unset), but add the .grow class back onto the “#Bottom” element. Unsetting the override inline, and adding it back to the element is safer.

Over in the style sheet, create a USS class to style both elements. Don’t forget to apply the USS classes to the new elements.

--card-background-top-colour: #C4C4C4;
--card-background-bottom-colour: #2D2D2D;

.card-top{
    background-color: var(--card-background-top-colour);
    height: 105px;
}

.card-bottom {
    background-color: var(--card-background-bottom-colour);
}

3. Style the #Top and #Bottom Elements

In the previous step, the elements are coloured, but we are missing the border radius. This can be added using velcro-style.uss classes .br-tl-20 and br-tr-20 to the #Top element and .br-bl-20 and br-br-20 to the #Bottom element.

Where properties include:

  • br for border radius

Where direction includes:

  • tl for top left

  • tr for top right

  • bl for bottom left

  • br for bottom right

Where value includes:

  • # value of the border radius for that border property

Read more about style classes at https://varlab-dev.atlassian.net/wiki/spaces/CV2/pages/1125089290/Style+USS+Classes#Border-Radius

4. Create the Separating Bar

Add another empty VisualElement and place it in between Top/Bottom. Here, I named it “Bar”. Unset the override inline flex-grow: 1 that Unity applies (Right Click > Unset).

Over in the style sheet, create a USS class to style the bar element. Don’t forget to apply the USS classes to the new elements.

--card-bar-colour: #FFC009;

.card-bar {
    background-color: var(--card-bar-colour);
    min-height: 2px;
    max-height: 2px;
}

5. Create the Label

Add an empty Label to the “#Bottom” element. Here, I named it “DescriptionLabel”. By default, Unity adds a 4px/2px margin/padding to labels. I remove those by using velcro-structure.uss classes m-0, and p-0. I also added the classes text-wrap, fw-700, and fs-36.

  • m-0 sets margin to 0px

  • p-0 sets padding to 0px

  • text-wrap makes the text wrap inside the #Bottom element

  • fw-700 sets the font to bold and use Lato font

  • fs-36 sets the font size to 36

Read more about font use https://varlab-dev.atlassian.net/wiki/spaces/CV2/pages/1121189929/Project+Structure+Best+Practices+Performance#Using-Font-in-VELCRO-%26-UI-Toolkit and more about typography style classes https://varlab-dev.atlassian.net/wiki/spaces/CV2/pages/1125089290/Style+USS+Classes#Typography

Over in the style sheet, create a USS class to style the label element. Don’t forget to apply the USS classes to the new elements. Additionally, add the p-20 (padding: 20px) class to the “#Bottom” element to space the label away from the edge.

--card-bottom-label-colour: #FFFFFF;

.card-bottom-label {
    color: var(--card-bottom-label-colour);
}
