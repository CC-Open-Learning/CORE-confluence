Overview

Splunk is a data analysis tool that can be used to both monitor and visualize data. It is offered in the formats of a desktop app (Splunk Enterprise) or via the cloud (Splunk Cloud Platform).

Splunk is not limited only to data analysis, its suite of software also covers security, IT and DevOps. It can be integrated with various third-party services, such as Microsoft 365, Azure, Atlassian, AWS, and VMWare.

Tools & Features

Machine Learning Toolkit (MLTK)

  • Users can create ML models that fits historical data and predicts future values

  • Uses Search Processing Language (SPL)

  • Predictive models can be used to discover anomalies in your datasets

Custom Dashboard

  • Users can customize their dashboard UI to include visualizations and move components around as they see fit

  • XML can be used to further customize dashboards and/or user forms

  • Dashboards can be exported in a PDF format for sharing

Real User Monitoring (RUM)

  • Framework agnostic and uses OpenTelemetry based instrumentation

  • User activity capture

    • Tracked by sessions and individual browser-resource interactions

Report Generating

  • Reports can be exported in CSV, PDF, or JSON formats

  • Custom visualizations can be added to reports

Third-Party Plugin/Application/Service Integration

  • Integrates with Atlassian products

Adding Data to Cloud Platform

To get data into the Splunk Cloud Platform you have to use a forwarder or the HTTP Event Collector.

Forwarders are the way to do data input to Splunk Cloud- data collection has to be done outside of Splunk. A forwarder must be installed onto any system that log data is being collected from.

HTTP Event Collectors allow you to forward data to your Splunk Deployment over HTTP/HTTPS.
