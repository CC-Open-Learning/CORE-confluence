CORE Framework : Challenges with PlayFab Accounts and SSO

Created by David Inglis, last modified on Apr 03, 2025

Summary

Traditional authentication (username and password login) for PlayFab has been deprecated, and will be fully disabled on April 30th, 2025. After this date, all accounts accessing PlayFab services will have to use Microsoft SSO.

VARLab currently has an administrator account using the varlabcc@gmail.com email account which is used to generate developer keys for Analytics events in DLX. This account currently has two different sign-in methods for PlayFab, using both the traditional authentication and an associated Microsoft account.

Issue 1: Merging Accounts

We have to merge these two accounts before Apr 30th.

It not yet known if merging these accounts will invalidate any existing developer tokens generated using the username + password authentication procedure.

Either way, the u+p authentication method will no longer exist after Apr 30 so if those keys are going to be invalidated, we'll find out one way or another

Mitigation

We will investigate whether this account merging process will invalidate existing keys

Issue 2: Microsoft SSO for Unity

It seems that when trying to authenticate PlayFab in Unity, Microsoft or Playfab (not really sure who is at fault) does not like that varlabcc@gmail.com is a "personal" Microsoft account, as opposed to a "work or school account"

I'm not really sure what to make of this, as we can sign into the browser using Microsoft SSO with that gmail without issue, and in the past I have had no problem authenticating in the Unity Editor using Microsoft SSO with that email. But it looks like there has been some change in that regard.

Mitigation

It may be possible to copy existing developer keys from previous projects, assuming they are not invalidated during the resolution of Issue 1 above.

Alternatively, a new Microsoft email account created within the Conestoga Active Directory, such as cvri-developer@conestogac.on.ca or similar, would be able to authenticate with PlayFab with least privilege access (certainly preferred to using an administrator account to allow for analytics events to be sent)
