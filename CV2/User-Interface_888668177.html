CORE Framework : User Interface

Created by vtummillo (Unlicensed) on Dec 15, 2023

This page will document the current and future UI elements for Procedurally Generated Worlds

Current Implementation

The current UI is solely functional. It will instantiate the grid for the user to see.

Next Implementation

Clear - this button will clear all input starting from the beginning

Undo - This will undo the last action

Highlight - This will allow the user to highlight a large section of the grid instead of one tile at a time

Scale - This will a slider that will move the camera further or closer, allowing the user to see up close or farther away as they require

Delete - This will change the grid highlighter to red and will delete any placed tiles when the user clicks

Windows/Doors - This will allow the user to click on an existing wall to add a door, window, or stairs

New Floor - This will save the current floor plan, and will display a new grid

Future Implementation

A future goal is to have procedurally generated houses, and the UI would drive this new feature. The user would be presented with a list of configurable options. They will fill in the table, click go, and the house will generate as per their selections:

Number of rooms

Number of Windows per room

Size of floor plan in square feet

Number of floors

Grid size (Small, Medium, Large) Select one

Studs, Drywall, or finished Select one
