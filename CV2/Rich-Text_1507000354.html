Rich Text

This page is intended to give a quick overview with examples of rich text tag use in UI Toolkit. Not every sprite or text customization needs to be in USS, UXML, or script for implementations. These options are especially useful in conversation UI pieces such as AMB’s dispatch conversation or PHI’s manager conversation. They can be used to add in-line sprites, or only style a single word in a label.

Setting up your scene

To get started using rich text in UI Toolkit labels

  1. Create a text settings asset Right Click/Assets > Create > UI Toolkit > Text Settings

  2. Set your desired defaults. The important one is the default path for sprite assets. The path is required to be within the “Resources” folder

  1. Reference the text settings asset in the DLX’s panel settings asset

Rich Text Tag Examples

Below will show two examples of the most likely uses of rich text in DLX. One example will be basic colour changes on text, and another with an in-line sprite. Using rich text in these instances significantly simplified script/USS/UXML implementations.

More complete documentation on rich text and all available supported tags can be read here.

Basic Colour Rich Text

In this example, the conversation entries flip back and forth between the dispatch and paramedic with various style changes. Ignoring the other USS changes on visual elements that cannot be adjusted through rich text,

  • The label for the talker alternates colour

  • The label for the message alternates colour

By using rich text tags, the conversation entry UXML and script was simplified to a single version where the incoming label is set, and no additional code was required.

//When setting text. Use the VELCRO helper method UIHelper.SetElementText()
caption.text = dialogue.Speaker;
description.text = dialogue.Text;

The conversation is populated from a scriptable object where the rich text tags are used

Dispatch:

<color=#FFFFFF>Dispatcher</color>
<color=#000000>Conestoga Paramedics, Code 4 to Highway 401 westbound at Cedar Creek 
for a single vehicle MVC. Reports of a vehicle rollover with entrapment.</color>

Paramedic:

<color=#FFCD29>Paramedic</color>
<color=#FFFFFF>Conestoga Ambulance Unit, 10-4. 401 westbound at Cedar Creek, single 
vehicle MVC with entrapment. We’re 10-8.</color>

In-Line Sprites

In this example, the sprites are not separate VisualElements, but instead, the UXML is a single label. Read more about sprites in text here.

By using rich text tags, the welcome UXML and script was simplified to a single label where the incoming label is set, and no additional code was required.

//When setting text. Use the VELCRO helper method UIHelper.SetElementText()
rules.text = modalInformation.HintAmount

The label is populated from a scriptable object where the rich text tags are used with sprite assets. To create sprite assets, follow the Unity documentation.

<sprite="Timer Icon" Name="Timer Icon"/> No time limit
