About the Object Viewer

The Object Viewer is a unity package that implements a 3D viewer component that be used within UI Toolkit, it includes built-in functionalities such as rotating an object using keyboard inputs or the mouse cursor, resetting object rotation and applying specific animations to objects, host projects can even use the events given by the package or implement their own events and call the functions of the package. The Object Viewer can work with or without a UI and it’s up to the developer to decide which approach to use for their project.

About the Object Viewer Controller

The Object Viewer controller is where most of the functionality for package is located, its the intermediary between the Object Viewer class and the display, it gets fed specific information from the ‘WorldObject’ class which gets stored and called from the controller to the display (UI or Worldspace).

About Object Cloning

Object Cloning is a feature within the Object Viewer controller, it also an object to be inspected without manipulating the transform of the original object by instantiating a clone and manipulating the clone’s transform instead of the original object. This feature can be applied for situations where the original object’s transform cannot be manipulated or if the developer prefers not manipulating the original object’s transform for any other reason.

You can enable this feature by enabling “Clone Game Object” checkbox in the Object Viewer controller script.

About the World Object Renderer

This asset is important to display objects in world space into a UI document, the important features to understand:

  • WorldImage - A UI element that can be added to any UI document, its used to display the World Object Renderer camera on a UI document.

  • Temp Camera - A temp camera is always included when adding the World Object Renderer, the camera of the World Object Renderer follows the same functionality of a regular Unity camera.

  • World Objects List - A list of transforms, the temp camera focuses on the transform in the first index.

  • Id - a string property that's used to connected the World Object Renderer to it’s WorldImage counterpart, the WorldImage element also includes an Id property which needs to be identical to the Id of the World Object Renderer in order for proper display.

In order to get this asset to be imported along side Object Viewer, you must manually import it by going to Tools>Object Viewer>Import/Update Object Viewer Assets.

If you would like to know more about the assets and its other features, you can read this documentation provided by Kamgam here: Link

About Rotating an Object

An object’s rotate functionality is done through the RotateObject script, this script is a required component for the ‘WorldObject’ script, players can rotate an object with WASD, arrow keys, or with the mouse cursor.

Main features of WorldObject:

  • Transform Details - Allows you to set the starting scale and rotation of a certain object, if not set, then the object will be automatically scaled to fit the size of the UI

Main features of RotateObject:

  • Limit Rotation - Allows you to assign a minimum and maximum angle for X, Y, Z.

  • Allow Animations - Allows you to specify how many animation states you want to be included.

  • Animatation States - a list with string inputs that lets you specify which animations you want to play in sequential order (Object must have animator component in order for this to work).

About Keybindings

Keybindings are ScriptableObjects, Keybindings use KeyCode types that allows developers to specify the inputs of four specific commands in the Object Viewer:

  1. Exit Object Viewer - Exits out of the Object Viewer state.

  2. Reset Rotation - Resets the rotation of an object back to its original rotation.

  3. Start Animation - Starts the animation for a specific object.

  4. End Animation - Ends the animation for a specific object.

Adding Preset Objects

There are preset objects that the developer can add to the scene by right-clicking the hierarchy then VARLab->Object Viewer->Intractable Object/Object Viewer Manager/Object Viewer UI, each of these objects have default components and values that the developer can change to their discretion. If you need to change the collider add the new one first and then remove the old collider component.
