CORE Framework : Running Unity Tests

Created by Ryan S (Unlicensed), last modified on Aug 04, 2023

This document assumes you have already installed Unity’s code coverage package. If you have not yet done this, refer to the Code Coverage Unity Setup document located HERE.

Be sure to have a test folder containing your automatic tests. If you do not have a test folder, you can create one by making a new folder in your assets folder called “Tests”. Use the “Create Test Script in Current Folder” button in the test runner to create the test assembly.

NOTE: This file is not the actual test, just the settings. You must add your test to this folder manually.

  • Starting the Test
  • Automated Testing

  • Next Steps

Starting the Test

  • Open Code Coverage and check if Enable Code Coverage is checked and the scripts folder path is in the Included Path.

Automated Testing

If your tests do not show up in the test runner, try switching from “EditMode” to “PlayMode”

  • On the code coverage window, switch to the test runner tab.

  • Select PlayMode or EditMode and then press the Run All button on the left side of the window to run your automatic tests.

  • Your tests should run, this will be indicated by your application running, as well as a code coverage progress bar.

  • If any of your tests fail, investigate the reason behind this.

  • If you have both PlayMode and EditMode tests, run each set of tests before generating the final report.

Gathering Results

  • Once you have run your tests, if you have the Auto Generate Report and the Auto Open Report settings enabled, you can access the report by opening the index.html file in the File Explorer window that will pop-up.

  • If either of those options aren't checked, you may:

    • Generate the report by clicking the Generate Report button on the top left of the window

    • Open the report by navigating in the File Explorer to the folder path selected in the Result Location

  • Open index.html.

  • This will open a browser window with your code coverage report. Congrats!

Next Steps

Now that code coverage reports have been generated, move to the next page to learn about interpreting your results.
