What is it?

Unity Cloud Content Delivery (CCD) is a first-party solution for distributing Asset Bundles to users with automatic download and update functionality. This allows developers to publish only their code and a few shared assets while the rest of their assets are delivered via CCD.

As the assets can be downloaded or updated for all users whenever developers decide, it can be used to distribute specific content to certain users or swap over the theme of assets for an event or holiday.

For CORE Engine we can pack a module’s assets into its own separate bundle and place it on CCD. This allows us to distribute just the engine itself and have users download their module assets as needed. If we make any changes to the assets for a module, users will automatically download those updates when they run the simulation.

This is crucial for WebGL deployment as it reduces the initial download for users significantly and allows us to download the module assets in the background while they navigate the menus.

Creating a Bucket

The first step to setting up CCD is to go to the Unity Dashboard and select Explore under Cloud Content Delivery. From there click the Projects tab and select our project. Once there select Create Bucket, fill out the information needed and click Submit.

With the bucket created, we will need to upload our Asset Bundles and catalog files from when we built our Asset Bundles from the Addressables system. To do so, follow the Uploading Instructions provided on the page, using either the website, API or Command-line interface (CLI) as desired.

Once all of your content is uploaded and ready to go, release the content at the top of the page and your content is now ready to be accessed remotely using the Addressables system. To learn how to use Cloud Content Delivery with the Addressables system, check out Addressables & Cloud Content Delivery .

Using Badges

Badges in Cloud Content Delivery are a way to mark releases with a specific label. This label can be used by the code to access specific releases. The default badge included with CCD is the latest badge, applied to the most recent release at all times. To add more badges, click on the Badges tab in the CCD bucket you would like to create a label in.

Here we can add a new badge, for this example we’ll add in a badge called production so that if our newest content isn’t necessarily production-ready, having users load from the production badge instead lets us hold back that release until we know it’s stable.

Once we click Submit we can now use our badge for any of the releases for this bucket. Switch to the Releases tab and you can now assign your badge to any release.

To learn about how to load content from your Cloud Content Delivery buckets with these badges, check out Addressables & Cloud Content Delivery .

Addressables With Cloud Build

Cloud Build can be configured to build your addressable asset bundles along with the player content and upload the result to Cloud Content Delivery. This saves us the effort of having to manually build and upload our asset bundles. For more information, click here.
