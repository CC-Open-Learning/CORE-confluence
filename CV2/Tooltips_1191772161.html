Tooltips

Tooltips come with 4 different directions, 2 different font sizes, and work for both UI, and 3D game objects.

https://www.figma.com/design/om9GwMhkfeJpcADbKz4ov8/CVRI-Design-System?node-id=1704-6912&t=2wdRKDtdXD7aM4wY-4

Setting Up Your Scene

To get started with implementing tooltips in your scene

  1. Drag in the TooltipUI prefab into the scene hierarchy at “/Packages/Velcro UI/Prefabs”

  2. Add the TooltipObject.cs component to any game object that needs a tooltip

  3. Replace the PanelSettings reference from VELCRO to your project’s PanelSettings asset

  4. Change default SerializedFields if desired. A more complete summary of SerializedFields can be seen below

  5. Hook up UnityEvents as required

    1. You will need to call HandleDisplayUI() on PointerEnterEvent for UI

    2. You will need to call CloseTooltip() on PointerLeaveEvent for UI

SetContent()

Definition:

public void SetContent(TooltipType tooltipType, string message)

Comments:

  • This method changes the label and applies the correct USS class for the tooltip direction. It sets the background image and message

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

ForceSetPosition()

Definition:

public void ForceSetPosition(Vector2 position)

Comments:

  • Moves the tooltip to a new position immediately but does not apply the offset or origin

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

SetPosition()

Definition:

public IEnumerator SetPosition(VisualElement parent, TooltipType tooltipType)

Comments:

  • Moves the tooltip to the new position. The new position will be the edge of the parent element's position, centered and offset from the direction of the tooltip

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

ApplyOffsetAndOrigin()

Definition:

public void ApplyOffsetAndOrigin(TooltipType tooltipType, ref Vector2 tooltipPosition, VisualElement parent = null)

Comments:

  • Adjusts the given position based on the tooltip's correct origin point and position offset

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

CloseTooltip()

Definition:

public void CloseTooltip()

Comments:

  • Closes all tooltips, and hides the tooltip with a fade out animation

  • Invokes OnTooltipHidden since Hide() is called during the FadeOut() method

FadeIn()

Definition:

public IEnumerator FadeIn()

Comments:

  • Tooltips fade in based on the properties serialized in the script. On enter, the tooltip goes from 0 to 1 opacity in fadeDuration time

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

  • Invokes OnTooltipShown since Show() is called during the FadeIn() method

FadeOut()

Definition:

public IEnumerator FadeOut()

Comments:

  • Tooltips fade out based on the properties serialized in the script. On exit, the tooltip goes from 1 to 0 opacity in fadeDuration time

  • Exposed if DLX would like to control individual stages of the UI instead of using CloseTooltip()

  • Invokes OnTooltipHidden since Hide() is called during the FadeOut() method

Show()

Definition:

public void Show()

Comments:

  • Shows the root of the tooltip

  • Invokes OnTooltipShown

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

Hide()

public void Hide()

  • Hides the root of the tooltip

  • Exposed if DLX would like to control individual stages of the UI instead of using CloseTooltip()

ClearClasses()
