Loading Indicators

The loading indicator is intended to be a piece of blocking UI to hide scene initialization. It is active for a few seconds, and can be optionally ended early by either jumping to 100% completion, or failing the load. The loading indicator only has one customization available, which is a dimmed background or an opaque background that switches with light/dark mode.

Setting up your scene

To get started with implementing loading indicators in your scene

  1. Drag in the loading indicator prefab into the scene hierarchy at “/Packages/Velcro UI/Prefabs”

  2. Replace the PanelSettings reference from VELCRO to your project’s PanelSettings asset

  3. Hook up UnityEvents as required

    1. You will need to supply HandleDisplayUI() with a start float otherwise it uses the default 0%

HandleDisplayUI()

Definition:

public void HandleDisplayUI(float startValue = DefaultStartingValue)

Comments:

  • This method is intended to be a single access method to set the loading indicator starting value, start the coroutine that begins the counter, and display it

  • Invokes OnLoadingShown since Show() is called after content is populated

SetContent()

Definition:

public void SetContent(float startValue)

Comments:

  • Sets the load indicator progress, desired canvas style (Dimmed/Opaque), and the loading text to loadingText (SerializedField)

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

StartLoad()

Definition:

public void StartLoad()

Comments:

  • Starts the Load() coroutine of the loading indicator which starts incrementing the progress indicator's progress

  • Calls SetComplete() if progress >= 100

  • Invokes OnLoadingHidden eventually

SetComplete()

Definition:

public void SetComplete()

Comments:

  • Stops the progress indicator from incrementing, and sets progress to 100. Calls the FadeOut() coroutine

  • Sets loading text to loadingCompleteText (SerializedField)

  • Invokes OnLoadingHidden since Hide() is called at the end of FadeOut()

SetFailed()

Definition:

public void SetFailed()

Comments:

  • Stops the progress indicator from incrementing, toggles USS classes on the loading indicator to set the bar red, hides progress to display an error icon, and hides the current progress amount (%)

  • Sets loading text to loadingFailedText (SerializedField)

  • Invokes OnLoadingFailed

ResetLoadingIndicator()

Definition:

public void ResetLoadingIndicator()

Comments:

  • Stops all coroutines, sets current progress to 0, and resets HasFailed/IsLoading

  • Sets opacity to 1, resets the canvas dim/opaque state, and removes loading indicator failed USS classes

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

FadeOut()

Definition:

public IEnumerator FadeOut()

Comments:

  • The loading indicator/canvas will fade out based on the properties serialized in the script. On exit, the loading indicator goes from 1 to 0 opacity in fadeDuration (SerializedField) seconds

  • Invokes OnLoadingHidden since Hide() is called after fade out is complete

Show()

Definition:

public void Show()

Comments:

  • Shows the root of the loading indicator

  • Invokes OnLoadingShown

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

Hide()

Definition:

public void Hide()

Comments:

  • Hides the root of the loading indicator

  • Invokes OnLoadingHidden

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()
