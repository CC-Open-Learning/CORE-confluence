CORE Framework : Content Localization

Created by David Inglis, last modified on Sept 01, 2022

In this section, we’ll cover how to use the Unity Localization package to provide your DLS project with a database of strings and assets that can be interchanged depending on a selected locale.

Installation of Localization package

Unity provides setup steps for installing the Localization package: https://docs.unity3d.com/Packages/com.unity.localization@1.3/manual/Installation.html

The VARLab-developed https://bitbucket.org/VARLab/corev2_localization package provides additional tools and examples for string and asset localization.

The COREv2 Localization package is optional for DLS projects which do not require the ability for users to change the language in the application

Open up the Packages/manifest.json file in your project and add this line to dependencies.

"com.varlab.corev2localization": "ssh://git@bitbucket.org/VARLab/corev2_localization.git#upm"

If your PC is not yet configured to access VARLab repositories via SSH, follow the instructions on Setting Up Bitbucket & Sourcetree before continuing.

After the COREv2 Localization package has been installed, import the App Localization Example content. The sample files will be imported to Assets/Samples/COREv2 Localization Toolset. This will also generate the required Addressable Assets data for the sample files in the Assets/AddressableAssetsData directory.

Lastly, the Localization Settings asset will need to be assigned in the Project Settings window if one is not already being used for the project. Navigate to Edit -> Project Settings -> Localization and assign a Localization Settings asset.

Using Unity’s Localization

The steps outlined in the https://docs.unity3d.com/Packages/com.unity.localization@1.3/manual/QuickStartGuideWithVariants.html provided by Unity cover setting up the initial localization settings asset for a Unity project and creating the string and asset tables necessary for each locale the project will use.

Localization of strings or assets can be performed through components provided by the package or through custom code written for the DLS project.

Using COREv2 Localization Toolkit & Examples

To complement the package provided by Unity, sample assets have been provided to demonstrate Localization setup within a Unity project. These sample assets are being delivered as a COREv2 package.

This package also allows for the delivery of tools that add extra functionality to the Unity Localization package. This toolset provides the basis of functionality needed to allow a user to change the current localization of the application while it is running.
