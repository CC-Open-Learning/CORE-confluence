Summary

PlayFab has some analytics tools available(Experiments, Segmentation, A/B Testing) to create different configurations that you can apply to users based off data or just at random, then use the collected data to compare the configurations(often used to see what type of deals are most profitable). Not important/useful for our needs.

Specifically useful towards our immediate goals/needs is the Metrics part of PlayFab. This is raw data combined to measure things. KPI, include daily/monthly users, screen time, new players, as some examples. These come with “out of the box” automatic events. Custom events, are also supported for our further needs.

PlayFab events as a Data Structure are represented as a JSON Object.

PlayFab includes Automated reports for daily/monthly views.

Overview of PlayFab’s Analytics

What is PlayFab Analytics: A set of features/tools for data tracking, processing, storage and exports.

Segmentation: create groups of players based on their player properties.

Experiments: Create/manage concurrent experiments through game manager, schedule to run at certain times and run across targeted segments.

Metrics: automatically calculates a report daily of industry standard metrics on engagement and revenue.

Experiments

Overview

Experiments are used to identify the best strategies for games. Allowing the running of multiple concurrent experiments. This can then allow the comparison of various game configurations and mechanisms, to determine the best variation for a title.

Capabilities of Experiments

                Creation/Management of multiple concurrent experiments, using the interface on PlayFab Game Manager, and via scalable API’s.

  • Targeting of Desired audience at random, target specific segments or percentages of those segments.

  • Scheduling of experiments.

  • Analysis of Experiments scorecard results.

  • Integrability with other PlayFab services such as Title Data Player Profile PlayStream events, CloudScript and Insights Explorer.

Experiment Configurations using PlayFab’s Experiments Variants

                Compare’s game configurations, configurations can be enabled through a game code mapped to a variant-variable pair or variant-override pair. A “control” experience is mapped to a “control” variant which can then be compared against one or more “treatment” experiences to determine which experience/configuration works better.

Configurations are mappable to variant, which are supported by variables which allow to bundle different game experiences via parameterization.

Metrics

Overview

                Metrics are raw data combined to create a meaningful measure. Industry standard metrics include.

·         New Players

·         Monthly Active Users

·         Session Length

·         Retention

PlayFab automatically calculates these industry standard metrics on a daily basis.

Metrics terminology definitions: Metrics and Terminology - PlayFab | Microsoft Learn

Metrics Quickstart

                2 Primary ways of exploring metric from PlayFab Game Manager, Trends and Reports.

Trends tab has eight key performance indicators

  1. MAU (Monthly Active Users)

  2. DAU (Daily Active Users)

  3. New Players (First time login)

  4. Screen Time

  5. Screen Frequency

  6. Classic Retention

  7. Revenue

  8. Stickiness

Further sections include Filters, Spark Cards, and Graphs. To view the data.

Generating PlayStream events

Generating PlayStream events - PlayFab | Microsoft Learn

Probably most useful for our needs.

  1. Automatic Events: “Out of the Box” PlayFab events, ie. Player logs in with valid email/pass. active screen time tracking, tracks client focus(when they alt tab/click out of the experience).

  2. Custom events: Creating a custom event specific to your game., fully reflected through real time analytic tools available. These are classified by entity bound to a player, character, or title. An event reference is specified using 2 fields

EntityType and EntityID.

Real-time analytics: core concepts

Real-time Analytics Core Concepts - PlayFab | Microsoft Learn

Analytics system gives insight into activities happening inside the game. A PlayFab event as a data structure represent a change in state of the game. Represented as a JSON object, each having 4 parts.

  1. Name (unique Identifier) what happened?

  2. Entity (Player, Title, or Character) who did it?

  3. Timestamp when?

  4. Associated Data (Body) What are the relevant details?

The core of an analytics system is the event flow, a list of all events sorted by time. Will need an analytics system to capture them for later analysis.

Reports

                Pre generated reports by PlayFab under the analytics section of game manager, there are daily monthly 30 day rolling views each report can also be downloaded as a CSM for use in custom applications.

Segmentation

                Segmentation can be used to create targeted groups of players based on their player properties. Playstream events are processed and your configuration puts people into and out of segments that are defined. Example given if a player enters a “high XP” segment it will push a notification to them and grant them bonus gold.

A/B Testing

Running multiple experiments against eachother, from players assignment into “buckets” example given testing discounts in a store to find the highest revenue per user.
