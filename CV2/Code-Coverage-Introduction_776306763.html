CORE Framework : Code Coverage Introduction

Created by Ryan S (Unlicensed), last modified by Talon Ernst (Unlicensed) on Aug 14, 2023

These documents cover gathering testing results, not creating tests. If you have yet to create tests, please refer to the testing documentation first before attempting to capture code coverage.

“Code coverage” refers to the amount of code that has been properly tested. This data can come from either automated tests or manual tests. As developers, we must test our code to ensure no underlying code paths have been overlooked (such as switch cases with 100+ switches).

Manually keeping track of different code paths can be difficult, but Unity’s Code Coverage package helps us avoid keeping track of every tested path. Ensuring that the majority of tests in your project are automated will make this process easier. Manual testing is more time-consuming, while automated tests are quicker and can be left to run. Additionally, automatic tests are more reliable and repeatable than manual tests.

This documentation will provide a guide on setting up, using, and documenting your project’s code coverage. Remember: code coverage isn’t the end goal. It’s a means to an end. It’s more important to employ a risk-based code coverage strategy focusing on covering testable behaviour.

To begin using Code Coverage with Unity, refer to the article on Code Coverage Unity Setup.
