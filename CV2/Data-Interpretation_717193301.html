CORE Framework : Data Interpretation

Created by Hayden Auterhoff, last modified on Jun 07, 2023

Overview

The analytic system by PlayFab provides a powerful backend-as-a-service (BaaS) platform that helps collect and process of collection and process of data. The analytic tools provided by PlayFab are an agonist platform and allow for the implementation of a variety of platforms and game engines, such as Unity.

In this overview, the discussion is about how to interpret and analyze game data using PlayFab in a Unity project. Using the data analytic tools and the Unity game engine, a developer can gain valuable insight into user behaviour, simulation performance, and other critical metrics.

PlayFab Key Components

  1. PlayFab SDK: allows developers to integrate PlayFab services directly into their Unity projects.

  2. PlayFab Game Manager: this is a web-based dashboard that provides a comprehensive overview of your game’s data, including user statistics, simulation performance, and other critical metrics.

  3. Event System: enables developers to track custom events that occur within their game. These events can be used to monitor user behavior, simulation performance, and more.

  4. Analytics: PlayFab offers a suite of analytics tools that help developers identify trends and patterns in their projects. These tools include PlayStream, which provides real-time event monitoring, and Insights, which offers advanced data analysis and visualization.

Data Interpretation Workflow

  1. Define Your Metrics: it’s essential to determine which metrics are most relevant to your project. Common metrics include active user count, retention rate, average session time, and progression.

  2. Set Up Custom Events: use PlayFab’s event system to define and track custom events that can provide valuable insights into player behaviour. These events can be easily configured within the Unity project using PlayFab’s SDK.

  3. Monitor Data in the Game Manager: use the PlayFab Game Manager to monitor your project’s data in real time. This dashboard provides a wealth of information, including user behaviour, and simulation performance.

  4. Analyze Trends and Patterns: use PlayFab’s analytics tools to identify trends and patterns in your project. This information can be used to make data-driven decisions about learning outcomes and ROIs.

  5. Iterate and Improve: Continuously review your project data to identify areas for improvement. Use this information to iterate on your project design, optimize user experience, and maximize user engagement and retention.

Data Analysis and Interpretation

Segmentation Analysis

Segmenting user data can help identify patterns and trends in specific player groups. For example, you can segment users based on their progression and time in the simulation, and analyze how they interact with the project. This information can help you tailor your project to specific groups of users for better engagement and retention.

Let’s say you’ve segmented your user data by time spent and found that users with more time in the simulation had a higher level of engagement than players with lower time spent. This information can help you identify the areas that aren’t engaging for users with lower time spent in the simulation.

Behaviour Analysis

Analyzing user behaviour can help you understand how users interact with your project and identify opportunities for improvement. Here are some examples:

1. Time spent within the project: you can analyze how much time users spend in your game and identify patterns in their behaviour. For example, you may find that users who spend more time in the simulation are more likely to perform better in evaluations than users that spend less time.

2. Conversion rates: you can analyze the conversion rates of various in-simulation information to classroom evaluations. This information can help you optimize the learning outcomes of the simulation and increase SCROM metrics.

3. Player churn: you can analyze user churn rates and identify the reasons why users are exiting the program or being distracted within the simulation. For example, you may find that user engagement can be much lower in certain parts of the simulation. This could indicate that the simulation is too difficult, or not receiving enough feedback. This information can help you make changes to improve user retention and experience.

4. User engagement: you can analyze user engagement levels and identify which features or content are most successful among users. This information can help you create more engaging content and features to keep the users more engaged within the simulation for a longer time or the desired time.

Funnel Analysis

Funnel analysis can help you understand how users move through the simulation and identify where barriers have occurred from users. For example, you can create a funnel that tracks the number of users who start a specific aspect of a simulation and the number who complete this specific aspect of the simulation, This information can help you identify if these certain aspects of the simulation are too difficult or not engaging enough.

If you notice that a high percentage of users are having difficulty in these aspects of the simulation, you may need to provide feedback or modify the engagement of the information or provide more guidance to help users understand how to complete this aspect of the simulation.

Conclusion

PlayFab provides a powerful analytics platform that enables developers to gain valuable insights into user behaviour, simulation performance, and other critical metrics. With the PlayFab SKD and Game Manager, developers can easily integrate and monitor custom events within their Unity projects, and use the suite of analytics tools to identify trends and patterns. By interpreting and analyzing data through segmentation, behaviour, and funnel analysis, developers can optimize their projects for better engagement, retention, and learning outcomes. Continuously iterating and improving based on data-driven decisions can lead to a more successful game project overall.
