CORE Framework : AWS Analytics Service

Created by Jacky Lam, last modified on May 26, 2023

Overview

The analytics solution Amazon offers is called Game Analytics Pipeline. It is a combination of Amazon’s services deployed in one solution.

Pros

More control over the tools

This is because the pipeline is made up of individual amazon services, you get the benefit to control every component based on your needs.

Good documentation and API reference for development

The documentation is solid, it includes text and video tutorial for beginner.

Cons

Initial setup is difficult

It uses Cloud Formation template tool to deploy the stack, however the template provided failed to deploy the stack. There’s also couple additional steps to setup the unity project to be ready.

Slow start for development

The SDK only have connectors to the data streaming service, it doesn’t include common game-related metrics out-of-the-box.

Harder to maintain than other options

Game Analytics Pipeline is just a guidance/framework to set up an data streaming and analytics pipeline, not an integrated solution, which makes it more flexible, but it also add complexity when it comes to maintenance.

No free tier

The data streaming service component (Amazon Kinesis) has no free tier, however this is not a big issue since it uses a pay-as-you-go pricing model, the initial cost is minimal.

Features

Real-time Data Streaming

This is a core component of the pipeline as it collects data from data producer (DLX sims) so we can gain insights from the data.

Dashboard

Real-time data processing with Amazon Kinesis Data Analytics, and monitor real-time metrics on a Dashboard with Amazon CloudWatch.

Data Validation/Transformation

Data cleaning and preprocessing, this feature is not really needed because the data comes from an controlled environment.

Data Storage

Using Amazon Simple Storage Service (Amazon S3).

Data Analytics and Visualizations

This is also a feature that we are interested in. Being able to query and aggregate data to generate reports and visualize the data is very important to our workflow.

Integration Methods

Available SDKs:

.NET SDK

C++ SDK

JavaScript SDK

(and SDK in other languages)

REST API through Amazon API Gateway

Pricing

The cost of running this solution depends on the amount of data being loaded, stored, processed, and analyzed in the solution. Deploying the solution in developer mode (Dev) mode and sending events to the pipeline with the included demo script will cost approximately $10.00 per day.

The demo script generate a batch of 100 events with an average interval of 4 seconds, that translates to approximately 2.16 million events per day.
