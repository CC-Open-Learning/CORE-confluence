CORE Framework : NodeCanvas Overview

Created by Jonah Pearce on Mar 01, 2023

Introduction

NodeCanvas is a feature-filled tool for visual scripting in Unity that allows us to create complex systems and behaviors for projects instead of traditional programming. Some of the key features of using NodeCanvas are:

  • Ability to create and edit visual scripting graphs

  • Has support for a wide variety of node types and connections

  • We can create our custom node types.

  • Conveniently playback and pause graphs

  • Has tools for debugging and logging graphs.

Graph

A graph in NodeCanvas is a collection of nodes you connect to create a specific system or behavior for your project. Every graph is stored and used as a Unity asset and can be visually edited with the NodeCanvas editor window.

Nodes

The nodes are the central part of the graph, the building blocks that make up the graph. They represent the functionality and tasks of your system. There are unlimited types of nodes, such as nodes that represent actions for the graph to complete, conditions to be met, Unity events, and many more types, including the ability to create a custom type of node for a project. Each node can have its unique properties and settings.

Nodes are connected to make a graph. Connections can be made simply by dragging the output of one node to the input of another node in the editor. This links the nodes for the flow of your graph, allowing them to interact and share data. Execution Flow is an essential concept in NodeCanvas. Their connections determine the flow in which the nodes activate. The graph will start at the first node and flow through them in the order set up until completion.

Node Canvas graphs can call on other graphs allowing us to have nested graphs which allow us to split up functionality and have many simpler graphs of different types instead of trying to have one graph.

  • Behavior Tree: These graphs are very commonly used for AI as its nodes serve as a series of tasks, where each task represents a specific behavior or action that the AI can take. Each task can have its own set of conditions and actions and can be used to build complex AI behaviors.

  • State Machine: A State Machine is a graph representing states and the transitions between them. For this reason, it is commonly used for task flows, and game flows for applications. The State Machine is run by entering a state and then transitioning to other states based on predefined conditions.

  • Dialogue Tree: A Dialogue Tree is similar to a state machine, especially considering you can add the same type of nodes with the same actions and conditions, the main difference being the added dialogue features. The dialogue features can be used to present to the user any dialogue, from simple statements to questionnaires or even full-on stories and chats.

In summary, NodeCanvas provides several types of graphs to create and manage different types of behaviors. The Behavior Tree is used to create complex AI behaviors, the State Machine is used to control game mechanics, and the Dialogue Tree is used to create branching dialogue options. By using these three graphs together, you can create complex and engaging gameplay experiences.

Blackboards

A blackboard is a data container that can be shared between nodes in a graph. It can be used to store variables and other information that is used by the nodes in the graph. This is extremely useful for complex DLX projects that are going to be using many graphs in one scenario, as it makes it easy to share and use variables and objects from the scene among all the graphs.

Playback and debugging

NodeCanvas allows you to play, pause and step through the graph; it also allows you to check the execution flow of the graph and debug the graph.
