SCORM + LTI Implementation

If your project is already using SCORM, or is planning to use both it is best to add the SCORM object to the scene first and setup the SCORM integrator script. Further resources here: https://varlab-dev.atlassian.net/wiki/x/OYAKHg

Once you have the SCORM implementation completed you are ready to add LTI functionality to your project!

The first step will be to add the LTI “Learner Info” object to your scene. This can be done by right clicking the hierarchy and going to “VARLab” as shown below.

This will give you an object in the hierarchy you can place where ever you like. Typically a child of “Managers” is the place you will put it.

Now we need to edit the Scorm Integrator script that is usually attached to the Scorm manager object. The first addition will be in Awake() we need to tell the Scorm Manager to not initialize automatically on start.

Otherwise if it is deployed in webGL but “LearnerInfo.UserID” was not null means the LTI server provided information to the sim, so we set the LearnerID in the Scorm integrator to the value so it can be accessed via both “ScormIntegrator.LearnerID” or “LearnerInfo.UserID” and call the “OnInitialize” event which typically will start the cloud save check. The following simulations have LTI added to them so you can look through them for full implementation examples:

  1. https://bitbucket.org/VARLab/trades-electrical/src/main/

  2. https://bitbucket.org/VARLab/public-health-inspection/src/main/

  3. https://bitbucket.org/VARLab/paramedic-ambulance/src/main/
