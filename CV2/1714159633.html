Understanding the box model and flexbox is essential to understanding UI Toolkit in Unity, and front-end web development. They both use a very similar system with very similar behaviour. In CSS, you have more properties, display types, and alternative box models. In USS, it is made simple, but also more limited. There is only one box model, and everything is either DisplayStyle.Flex, or DisplayStyle.None.

Box Model

The box model refers to the layout of an element and how different properties affect it. Everything in both USS and CSS is a box. When hovering over an element in the UI Builder:

  • The blue area will be the space your content is placed in (Child elements)

  • The green will be your padding (Spacing INSIDE your current element)

  • The yellow will be the border

  • The orange will be the margin (Spacing OUTSIDE your element)

  • The element size you specify with width + height will be a combination of content + padding + border. Margin is excluded from this as it is considered outside of your element and used to space one element away from sibling elements

Ex. If an element has the USS below, then the real dimensions will be 260px by 160px because 20px will be taken away for padding + border.

.box-model {
  width: 300px;
  height: 200px;
  margin: 10px;
  padding: 15px;
  border-width: 5px;
  border-color: red;
}

When an Element Uses Space

An element will always take up space in the layout as long as it is set as DisplayStyle.Flex. Even if the opacity is set to 0, or the visibility is set to hidden.

  • Image 1 has DisplayStyle.Flex, Opacity 0, and Visibility.Hidden

  • Image 2 has DisplayStyle.None, Opacity 100, and Visibility.Visible

Flexbox

Flexbox is a layout system to arrange elements in rows and columns. The layout system is composed of a main axis, and a cross axis. The main axis is the direction of the flex layout (flex-row, flex-row-reversed, flex-column, or flex-column-reversed) and the direction the elements will be ordered in. The cross axis is the axis perpendicular to this.

Before you get to the explanation, play through https://flexboxfroggy.com/

Main and Cross Axis

By default, elements are organized in a column starting from main start and ending at main end.

  • Main start at the top

  • Main end at the bottom

  • Main start at the bottom

  • Main end at the top

  • Main start at the left

  • Main end at the right

  • Main start at the right

  • Main end at the left

Flex Properties

Below are the common USS properties that are used and will have changed behaviour depending on the state of the current main and cross axis. Property(s)

Description

All child elements will be ordered in a column from top to bottom

.flex-column-reverse

flex-direction: column-reverse

All child elements will be ordered in a column from bottom to top

.flex-row

flex-direction: row

All child elements will be ordered in a row from left to right

.flex-row-reverse

flex-direction: row-reverse

All child elements will be ordered in a row from right to left

Justification

Structural USS Classes

Class Name

Property(s)

Description

Example

.justify-start

justify-content: flex-start

All child elements will be positioned (Vertically by default) at the start position of the this element’s primary axis

.justify-center

justify-content: center

All child elements will be positioned (Vertically by default) at the center position of the this element’s primary axis

.justify-end

justify-content: flex-end

All child elements will be positioned (Vertically by default) at the end position of the this element’s primary axis

.justify-space-between

justify-content: space-between

All child elements will be distributed along this element’s primary axis with an equal amount of space between each other. The first and last element will be flush against flex-start and flex-end

.justify-space-around

justify-content: space-around

All child elements will be distributed along this element’s primary axis with an equal amount of space between each other. The first and last element are spaced half the amount from flex-start and flex-end

This element’s align-self property will match the parent element’s align-items property

Position

Structural USS Classes

Class Name

Property(s)

Description

Example

.relative

position: relative

This element’s positioning will depend on the parent element’s bounds and other sibling elements that are set to relative

.absolute

position: absolute

This element’s positioning will depend solely on the parent element’s bounds.

.grow

flex-grow: 1

Determines how the item will grow relative to the rest of the flexible items inside the same container. Flex-grow will make the element take up all available space in the parent container and share it evenly with other elements that have the same grow value

.shrink

flex-shrink: 0

Determines how the item will shrink relative to the rest of the flexible items inside the same container
