CORE Framework : WebGL Support

Created by Talon Ernst, last modified on Jul 30, 2024

This document goes over WebGL support with the packages being used for Multiplayer, discussing both what is possible as well as limitations.

What is supported

  • Multiplayer games in a WebGL build

  • Using Unity Relay, one of the clients can act as a host since it runs through Unity Services, allowing for a peer-to-peer architecture

  • Unity Relay fully supports encryption through the WSS protocol

  • Using UniTask, we can use Unity Services, and potentially even use Tasks more in general (UniTask has only been tested to a small degree so far).

What is not supported

  • We cannot directly have a WebGL build be a host/server.

  • Cannot use the Quality of Service feature of Unity Relay.
