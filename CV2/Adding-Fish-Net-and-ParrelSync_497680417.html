CORE Framework : Adding Fish-Net and ParrelSync

Created by Lisa Nguyen, last modified by Talon Ernst on Oct 25, 2023

Fish-Net is free to use from the Unity Asset Store (Fish-Net: Networking Evolved | Network | Unity Asset Store).

Once added to your assets you can open your Unity project and under the Package Manager go to ‘My Assets’ and you will be able to download and import Fish-Net to your project.

Using With ParrelSync

https://github.com/VeriorPies/ParrelSync

ParrelSync can be used to run multiple instances of/test multiplayer without having to build the project. You can go to the ParrelSync GitHub page and download the code ZIP file and add it to your project from disk or add it via UPM (more info can be found on the GitHub page’s README).

Creating Clones

Go to the ParrelSync tab and open the Clone Manager, if you cannot see this tab, restart Unity.

Click on ‘Add New Clone’ to create a clone of your current editor. Opening a clone in a new editor will allow you to test Client-Server interactions in real-time.

As you make changes in your main project the project clones will automatically be updated as well. If a clone is open and you made changes in the editor you will be prompted to reload the project clone.

Deleting a clone is easy too, just click on ‘Delete’ under the clone you want to delete and ParrelSync will handle the rest.

If you want to remove ParrelSync entirely make sure all clones have been deleted then go to the package manager to remove it from your project.


The clones added by ParrelSync do not need to be pushed onto repositories as they are purely local and meant for testing. Add the following line to your .gitignore to prevent this.

*_clone_[0-9]*/
