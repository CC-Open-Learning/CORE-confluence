CORE Framework : PlayFlow- WebAPI

Created by Lisa Nguyen, last modified on Jul 25, 2022

Using the WebAPI you can start, stop, restart, and get information from your PlayFlow server without having to open it in the Unity Editor. Your PlayFlow server must be uploaded in order to use this.

Link: https://api.cloud.playflow.app/docs

Using the WebAPI

Click ‘Try it out’ on the top right to be able to fill out the fields. Use the token given to you on the PlayFlow Dashboard. Your serverid will also be found there.

SSL is needed for WebGL builds, if it’s not running for WebGL keep this field false. You can get the SSL port from the first time you start your PlayFlow server.

Keep the default ‘medium’ for the type parameter and keep the arguments field blank.

Using VSCode and REST Client Extension

Install the REST Client extension (REST Client - Visual Studio Marketplace).

Using either HTTP or plaintext you can send requests to the WebAPI.

Your request will look something like this:

**These are meant to be used as examples, replace the fields with your own information, all parameters are found on the PlayFlow WebAPI site.

Starting Server

POST https://api.cloud.playflow.app/start_game_server
Content-Type: application/json
token: eb23bcbdb01040047f4b95b8a342d23e
version: 2
region: us-east
ssl: true
sslport: 49598
type: medium

Stopping Server

POST https://api.cloud.playflow.app/stop_game_server
Content-Type: application/json
token: eb23bcbdb01040047f4b95b8a342d23e
version: 2
region: us-east
serverid: 21027

Restarting Server

POST https://api.cloud.playflow.app/restart_game_server
Content-Type: application/json
token: eb23bcbdb01040047f4b95b8a342d23e
version: 2
region: us-east
serverid: 21027
ssl: true
