CORE Framework : Adding Voice-Over and Subtitles

Created by Leo Bunting on Aug 18, 2022

“Say” Action Task

The best option for a single actor voice-over, or subtitles paired with Action Tasks (in sequence or in parallel), is to add the Say Action Task to an Action State. This is preferable over using nodeCanvas' Dialogue Tree.

To do so, an Action State node should be added to the graph’s Canvas. Then, a Say Action Task should be assigned to it.

To use the Say Action Task, add an empty GameObject to the scene. Then, add the Dialogue Actor script, where we can set the name of the actor, the colour of the text, and an icon to be displayed alongside the subtitles. All these fields are optional.

Next, return to the Canvas and click your Action State. Then, set up the dialogue by selecting the Actor you just created, and adding the subtitle text and the audio clip file.

You’ll need to add the @DialogueUGUI prefab or a custom UI script to the scene to display the subtitles and play the voice-over files. The @DialogueUGUI prefab, which comes with NodeCanvas, has some options that may be customized.

If the Say action is used in a sub-tree (which must be saved as assets), the actor must be added as a variable to the blackboard. Sub-trees cannot reference scene objects.

Dialogue Tree

The nodeCanvas Dialogue Tree is used to create dialogue between two or more actors in a game. It also allows for dynamic dialogue text by using variable names in square brackets ([variableName]), yes or no questions, or multiple-choice questions.

In the Dialogue Controller settings, you can add Actors and assign a GameObject that contains a DialogueActor component.

More information on the Dialogue Tree and how to create dialogue with multiple actors is available in the nodeCanvas documentation.
