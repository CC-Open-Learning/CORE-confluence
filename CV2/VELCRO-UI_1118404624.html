What is VELCRO?

VELCRO UI is VARLab’s own Bootstrap/Tailwind UI library of USS classes, best practices, component templates, and ready to use UI prefabs. The goal is to standardize UI in all DLX as much as possible. This will likely cover the how UI is implemented through the USS class library and best practices more than visuals as individual DLX can override the default dark/light theme and will always need custom UI that is unique to that DLX.

Adding VELCRO to a DLX

  1. Ensure you have completed the Setting Up Bitbucket & Sourcetree and Accessing the Package Registry for Developers steps

  2. Navigate to Window > Package Manager

  3. Switch the package source drop down to “Packages: My Registries”

  1. Install the package

    1. All USS, TSS, UXML, and some setting assets files will be located in the, “/Assets/VELCRO UI” directory

    2. All Scripts, prefabs, tests, etc. will be located in the, “/Packages/VELCRO UI”

    3. The samples folder is an optional import that will import example scenes that demonstrate implementations of many UI included in the package

  2. Add VELCRO to the list of references in the DLX assembly definition file. It will be named similar to VARLab.[NameOfDLX]

    1. Press the + button to add another entry in the list

    2. Search for VARLab.Velcro by pressing the button

All USS, TSS, and UXML files are not in the default package directory of “/Packages/X” because the DLX UI builder does not have visibility files outside of the “/Assets” directory. These files are zipped into a .unitypackage when VELCRO is updated, and automatically unpackaged when VELCRO is installed or updated in a DLX.

  1. Create a [NameOfDLX]Dark.tss and [NameOfDLX]Light.tss files by Right click/Assets > Create > UI Toolkit > TSS Theme File

    1. Reference VELCRO’s Dark.tss in the “Inherited Themes” field for the DLX’s dark theme

    2. Reference VELCRO’s Light.tss in the “Inherited Themes” field for the DLX’s light theme

    3. All UI created for your DLX should have colour variables in [NameOfDLX]Dark.tss, and [NameOfDLX]Light.tss.

    4. All style sheets for your DLX should be referenced in the “Style Sheets” list of both theme files

All theme files must inherit from UnityRuntimeDefaultTheme or another theme file that inherits from UnityRuntimeDefaultTheme. VELCRO’s Dark/Light already inherit from Unity’s.

  1. Create a [NameOfDLX]PanelSettings.asset by Right click/Assets > Create > UI Toolkit > Panel Settings Asset

    1. Copy VELCRO’s PanelSettings values in “/Assets/VELCRO UI/Settings/PanelSettings” but reference your default DLX theme file. This will likely be [NameOfDLX]Dark.tss

    2. Reference VELCRO’s Text Settings asset

    3. Ensure the rest of the settings match. “Expand” for screen match mode is important

Any changes made to VELCRO assets will be discarded next time you update the package as the .unitypackage will unpack again. Creating your own DLX’s theme files, and panel settings will avoid panel setting sort order issues, and allow you to override VELCRO themes. When using any VELCRO UI, ensure the DLX PanelSettings is used instead of VELCRO’s

  1. This step is optional. USS is almost identical to CSS, so I find it helpful to edit USS/TSS files in VSCode and take advantage of the CSS IntelliSense and colouring through file assocications.

    1. Open VSCode, and navigate to File > Preferences > Settings

    2. Look for the “Files: Associations” section

    3. Add one file association for USS files, and the other for TSS files. The item should be the file extension, and the value should be “css”

    4. Now you will have more helpful IntelliSense and colouring when working with USS

Package Directory Overview

├─ Assets
│   ├─ Resources (Fonts & Icons)
│   ├─ Settings (Panel & Text Setting Assets)
│   ├─ Style Sheets (All USS files including velcro-structure and velcro-style)
│   ├─ Templates (UXML files for controls such as buttons, sliders, toggles, etc.)
│   ├─ UI (UXML files for complete UI such as notifications, toasts, etc.)
│   └─ UnityThemes
└─ Packages
    ├─ Editor
    ├─ Kamgam (UI Toolkit package for glow. Used for box shadow)
    ├─ Prefabs (Prefabs set up using completed UI and script functionality)
    ├─ Runtime (Scripts for completed UI and custom controls such as sliders, etc.)
    └─ Tests
