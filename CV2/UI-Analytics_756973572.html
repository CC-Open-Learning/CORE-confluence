Overview

Attention: To successfully examine the CORE Sandbox implementation, the installation of PlayFab and the VARLab analytic package must have been completed.

If installation is not complete, refer here for more instructions: https://varlab-dev.atlassian.net/l/cp/U7EyYxuJ

This page will provide a simple implementation of the analytics package and how to capture analytic events using UI Elements on Unity. The following example assumes that the VARLab analytic package is installed on your Unity project and that you have installed and signed into the PlayFab SDK.

Note: in the PlayFab SDK select which game title you want to use, or you won’t be raising any events.

UI Analytic Tutorial / Example

To find an example of the implementation of the analytic package for UI elements, check out the VARLab CORE Sandbox Development Environment under the branch CORE-89-Analytics-Tutorial-Updated-Package.

Fork the repository to have a local version of the example, and navigate to the scene called UI Analytics.

All of the resources that are not in the analytic package will be found in the scene hierarchy and in the Scripts folder. The specific scripts used for this scene will be in the UI Analytics folder.

Developer Note: Scripts are decoupled from Unity in this example

  • More information can be found in the drop down below.

Interfaces and Wrappers

The UI analytic tutorial contains a lot of wrappers and interfaces to decouple the mono behaviour of the scripts from Unity. This was done to conduct unit tests. There are some interfaces and wrappers for Buttons, Components, and GameObjects exist but may not be needed in future projects, depending on the future packages available. However, the wrappers and interfaces are just abstractions, and if you prefer a functional approach, check out the Interactable and State Machine implementation since the logic overlaps.

LoginHandler

Note: the analytic package provides a prefab of an Analytics gameObject, more information can be found here: https://varlab-dev.atlassian.net/wiki/spaces/CV2/pages/782565419/Setting+Up+Analytics#Analytics-Prefab

  • The CORE Sandbox examples did not use this prefab, however the implementation is the same as the prefab. For an explanation of the black box implementation of the analytics prefab object, please refer to here: https://varlab-dev.atlassian.net/l/cp/dg1yshzm

In the tutorial, the LoginLearner method from the analytic package will be handled by the LoginHandler script on the Canvas gameObject (Ex. 1). The LoginHandler script has 5 serialized components:

  • Input Field (IdInput)

  • Input FIeld (NameInput)

  • Quiz 1 Object (Quiz1)

  • Quiz 2 Object (Quiz2)

  • CoreAnalytics (Analytics)

The first four components are just the logic of the tutorial and have no impact on the functionality of the LoginLearner method from the analytic package. The LoginLearner method only accepts a string as a parameter for the Learner ID, which in this tutorial is the text from the IdInput field. The Quiz 1 and Quiz 2 objects are just SetActive() depending on if the user has signed in.

Note: The LoginLearner method is only captured event that will not be listening for analytic events. The LoginLearner method is the only method that relies on the CORE Analytics class from the analytic package, but will still need to be serialized onto a gameObject due to the mono behaviour.

Developer Note: LoginHandler needs to be on a gameObject

  • More information can be found in the drop down below.

LoginHandler

The implementation of the UI Analytics is a simple example of ‘how’ analytics can be used in a Unity project. The Login Handler script can be placed on any gameObject but does need to be on a gameObject since the analytic events class from the analytics package is a mono behaviour.

In addition, the logic used for determining the information passed to the LoginLearner method depends on the design of the project. The LoginLearner might be a user input or might need an API call from SCORM to get the LearnerID.

Quiz Components

The quiz component uses the scriptable object called the Analytics Channel. More information can be found by navigating to the “Sending Default Events” section on the page: https://varlab-dev.atlassian.net/l/cp/kyEdsk6X

QuizAnswer Script

With the use of the Analytics Channel, information can be collected and stored in objects. In this example, the quiz component for the correct or incorrect button is a gameObject with the defined quiz properties:

  • isCorrect

  • QuestionName

  • SectionNumber

The components of a quiz object are defined in the editor with the Quiz Answer script (Ex. 3), then extracted in the QuizHandler script to determine if the button object has a quiz component.

QuizHandler Script

The QuizHandler script is attached to the Canvas gameObject and will need the Analytics Event class from the analytics package. The Analytics Event is the same scriptable object from earlier. The Quiz Handler has four serialized components (Ex. 4):

  • Quiz Answer

  • Quiz Complete

  • Num of Sections

  • Analytics Event

The Quiz Answer is left unserialized as this will be populated by the Component of the Button object passed to the script. The Quiz Complete is the class that handles the DLXComplete analytic event. The Num of Sections is the tutorial logic to determine how many and how long each section is in the project. In this example, section 1 has 2 questions while section 2 has 1 question. Based on the logic of the tutorial, a user cannot progress to quiz 2 without completing quiz 1.

When the Quiz Handler is passed a button that has a component of QuizAnswer (CurrentClickedAnswer()), then a method is called to listen to this analytic event and raise it PlayFab. After the button that has a QuizAnswer component will raise a ProgressEvent, then the state will check if the Section is complete. If the 2nd question of Quiz 1 is passed to QuizHandler, then QuizHandler will raise two events; one for the ProgressEvent and the other for the SegmentCompleteEvent.

QuizComplete Script

With a SegmentCompleteEvent raise, the method from QuizComplete is called to validate if the DLX is complete or not. Once the DLX is complete, a DLXComplete event will be raised to PlayFab. The Quiz Complete script is attached to the Analytics gameObject and has the Analytic Event listener as a serialized component, where the Analytics Event is the same scriptable object as before (Ex. 5).

Developer Note: QuizHandler and QuizComplete need to be on a gameObject

  • More information can be found in the drop down below

QuizHandler and QuizComplete

The implementation of the UI Analytics is a simple example of ‘how’ analytics can be used in a Unity project. The Quiz Handler and Quiz Complete script can be placed on any gameObject, but does need to be on a gameObject since the analytic events class from the analytics package is a mono behaviour.

In addition, the logic used for determining when and how a segment is completed is based on the design of the Unity project. The tutorial is really meant to show how to raise these analytic events in a Unity project.

Custom Analytics Event

The EventContainer class in the analytic package allows custom event classes to be created by inheritance. For more information about creating custom events and inheriting from the EventContainer in the analytic package, navigate to “Creating Custom Events” on the page: https://varlab-dev.atlassian.net/l/cp/kyEdsk6X

ChecklistContainer

In the tutorial, a ChecklistContainer was made as an example of creating a custom analytic event. The ChecklistContainer can be found in the Scripts folder in UI Analytics, then Custom Events (Ex. 6).

The logic of the ChecklistContainer tracks the number of the totalGuesses and the numberCorrectGuesses, then raises this analytic event to PlayFab. The purpose of this custom analytic was to show that not all projects have to complete all of the objectives of the task, sometimes a user will miss some of the objectives that want to be captured.

The ChecklistContainer is raised in the QuizComplete script and is raised at the same time when the DLXCompleteContainer is raised to PlayFab.

Summary

These analytic tutorials are a tool to help provide a framework or guideline of how to use the core analytics and analytic events from the analytic package. The implementation is a general use case but hopes to be beneficial in understanding and easy to reverse engineer.

In this UI element implementation, there are interfaces and wrappers implemented to help reduce the dependency of Unity on these scripts to help with unit testing and was an effort by CORE Code Coverage and CORE TDD to help with their project work. The other two examples have a lighter and more functional approach with a similar implementation if needed.
