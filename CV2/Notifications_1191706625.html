Notifications

Notifications are simple toast messages used to provide feedback to the learner about a recent action. Notifications come in success, info, error, action, and custom versions.

https://www.figma.com/design/om9GwMhkfeJpcADbKz4ov8/CVRI-Design-System?node-id=1704-6750&t=yuEJnmZ31oDVSyQi-4

Setting up your scene

To get started with implementing notifications in your scene

  1. Drag in the notification prefab into the scene hierarchy at “/Packages/Velcro UI/Prefabs”

  2. Replace the PanelSettings reference from VELCRO to your project’s PanelSettings asset

  3. Change default SerializedFields if desired. A more complete summary of SerializedFields can be seen below

  4. Hook up UnityEvents as required

    1. You will need to supply HandleDisplayUI() with a NotificationSO, or list of arguments, or SetContent() with a list of arguments to populate content

    2. You can also change the custom notification by calling SetCustomNotification()

Properties & Methods

Property/Method

Description

Root

Root VisualElement of the UI

HandleDisplayUI()

Definition:

public void HandleDisplayUI(NotificationType notificationType, string message, FontSize fontSize = FontSize.Medium, Align alignment = Align.FlexStart)

Comments:

  • This method is intended to be a single access method to populate the notification type, icon, message, and display it

  • Invokes OnNotificationShown since Show() is called during the FadeIn() method after content is populated

HandleDisplayUI()

Definition:

public void HandleDisplayUI(NotificationSO notificationSO)

Comments:

  • This method is intended to be a single access method to populate the notification type, icon, message, and display it

  • Invokes OnNotificationShown since Show() is called during the FadeIn() method after content is populated

SetContent()

Definition:

public void SetContent(NotificationType notificationType, string message)

Comments:

  • Populates the notification type, icon, and message, but does not call Show()

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

SetCustomNotification()

Definition:

public void SetCustomNotification(Color backgroundColour, Color textColour, Sprite icon = null)

Comments:

  • Sets the notification icon, background colour, and text colour for the next time HandleDisplayUI() is called with a custom notification type

  • If no icon is provided, then the text will be centered like PHI’s action type notification

ClearClasses()

Definition:

public void ClearClasses()

Comments:

  • Removes all USS classes applied through SetContent() for the different notification types, and resets the notification to it’s base state

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

  • If DLX are controlling individual stages of notifications without HandleDisplayUI(), this method must be called before you can display another notification through Show(), or FadeIn()

FadeIn()

Definition:

public IEnumerator FadeIn()

Comments:

  • Coroutine to play the fade in and bump up animation on entry. On enter, the notification goes from 0 to 1 opacity and bumps up translateDirection units

  • If a notification is already visible, the animation will be skipped, and content will be updated immediately

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

  • Invokes OnNotificationShown since Show() is called during the FadeIn() method

FadeOut()

Definition:

public IEnumerator FadeOut()

Comments:

  • Coroutine to play the fade out and bump down animation on exit. On exit, the notification goes from 1 to 0 opacity and bumps down translateDirection units

  • If a notification is already visible, the animation will be skipped, and content will be updated immediately

  • Exposed if DLX would like to control individual stages of the UI instead of using HandleDisplayUI()

  • Invokes OnNotificationHidden since Hide() is called during the FadeOut() method

Show()

Definition:

public void Show()

Comments:

  • Shows the root of the notification

  • Invokes OnNotificationShown

Hide()

Definition:

public void Hide()

Comments:

  • Hides the root of the notification

  • Invokes OnNotificationHidden

Scriptable Object

To set up the notification, we will need to create a notification scriptable object asset.

  • Right Click/Assets > Create > ScriptableObjects > NotificationSO

  • Alternatively, you can create a SO through code and fill out properties from there using 

    NotificationSO notificationContent = ScriptableObject.CreateInstance<NotificationSO>();
notificationContent.NotificationType = NotificationType.Success;
notificationContent.Alignment = Align.FlexStart;
notificationContent.FontSize = FontSize.Medium;
notificationContent.Message = "Message";

Property

Description

NotificationType NotificationType

Notification type of the notification. Options are limited to enum NotificationType.cs

Align Alignment

Vertical position of the notification. Options are limited to FlexStart, Center, and FlexEnd

SerializedFields

Field

Description

float fadeDuration

Time in seconds for notification to go from 0->1 or 1->0 opacity

float solidDuration

Time in seconds for notification to stay at 1 opacity before fading out

float translateDirection

Represents the distance and direction which the notification will fade in and back out from its origin. Negative numbers represent up, positive numbers down. Origin is top left

Sprite successIcon

The background image of the [Icon] element on the left of the notification when the type is success. Set to the built-in success sprite by default

Sprite errorIcon

The background image of the [Icon] element on the left of the notification when the type is error. Set to the built-in error sprite by default

Sprite infoIcon

The background image of the [Icon] element on the left of the notification when the type is info. Set to the built-in info sprite by default

Sprite customIcon

Can be set to a sprite of your choice for making a customized notification. Replaces the default icons built into the 3 default notification types

Color customBackgroundColour

Background colour of the [Notification] element when the type is custom

Color customTextColour

Colour of the [Message] element when the type is custom
