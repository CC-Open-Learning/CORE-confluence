CORE Framework : Known Issues and Limitations

Created by Jacob Martens on Aug 14, 2024
  • PlatformController
  • FlexXR

PlatformController

  • Though runtime changing of platforms is possible, it can be unstable. It is recommended that desired platform is configured at startup, then left as-is throughout execution. Specifically, various XR components on game objects behave unexpectedly when enabled/disabled

FlexXR

  • FlexXR has some startup quirks

    • If it detects any components missing from the prefab on Awake(), it will disable that specific panel

    • It looks for Camera.main on start, disabling if it can't find one

    • It looks for an InputSystemUIInputModule. If it can't find one, it will attach one to the main camera

  • The best way to avoid startup issues is configuring the scene in a confirmed working way

    • Attach an EventSystem, InputSystemUIInputModule, and an XRUIInputModule to the MainCamera. If there are multiple MainCameras (i.e. an XR player and Flatscreen player), place these components on all cameras.

    • In a scene with multiple cameras, ensure only one is active when starting the game. It needs to be the camera associated with the Initial Platform.
