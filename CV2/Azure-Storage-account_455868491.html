Overview

Azure storage account provides a unique namespace for data objects like blobs, files, tables and disks to be uploaded and downloaded over the HTTPS.

There are different types of storage accounts and at VarLabs we are using the Standard general purpose account which has support for blobs, Queue, and Table Storage. The blobs contains the Data Lake Storage which is a set of capabilities dedicated to big data analytics built on the azure blob storage. For more information on big data analytics visit this link.

Every Object that is stored on the storage has a unique address assigned to it which is used to it which is used to interact with the data object.

The interaction are handled with a restful structure in mind and using a Web API as a middle man.

varlabsazurestorage is a part of the CoreUnity resource group in azure.

These are the properties of the Storage Account

The storage account houses containers which contains blob blocks (Data).

To make new storage accounts, access is needed to the Azure portal and for the time being only Jordan Robinson and myself (Ujjwal Prashar) have access to the storage. If a dev needs access to the storage please contact IT.

Container:

Containers act as different housing points for data that is being sent to Azure. Every Scenario has a separate container and once a scenario is loaded the value for the active container changes to that particular container. If no scenario is loaded that the files will be called from a container called “azureplayground“. The container are named after the unique identifiers of different scenarios.

Access Key:

Every Storage account has two 512 bit key assigned to it. These keys are used to authorize access in the storage accounts to send and retrieve data stored in the Simulations. These Access keys are stored on the Web API and to make GET or SET calls to the API authorization is required. Authorization is provided via the JWT token and all the calls made from Core have authorization pre applied on them so developers don’t have to worry about making authorization calls as well.
