Define the Problem


With the approach to building an analytical solution for the DLX projects, it is important to outline the details and considerations needed to encompass each application and simulation. Here are some key aspects to explore when defining the problem:

  1. Understand the Project Requirements

    1. Gather the specific needs and expectations of stakeholders (Ex. instructors and administrators).

    2. Clearly outline and define the purpose for building an analytical solution and the desired outcomes from the solution where an existing analytic tool is unable to accomplish the task.

    3. Identify any specific constraints or limitations for designing an analytical solution, such as time, budget, or technical requirements.

  2. Define the Metrics and Data Points

    1. Determine the metrics that will be needed to be tracked by the analytical solution. These may include:

      1. Total instructional hours per DLX

      2. Instructional hours per student

      3. Completion status

      4. Time spent in simulation

      5. Number of attempts

      6. Highest milestone achieved

    2. Determine the value and insight that these metrics can provide by the analytical solution.

  3. Scope of Solution

    1. Determine whether the designed analytic solution should be integrated with existing technology or be an independent system.

    2. Evaluate the advantages and disadvantages of having the analytic solution should be integrated versus an independent system.

    3. Consider the deployment of the analytic solution:

      1. Where the solution will be used (Ex. Unity)

      2. Ensure compatibility and scalability across different environments (Ex. VConestoga)

  4. Privacy and Data Security

    1. Understand and comply with any and all relevant privacy regulations for data collection

    2. Determine if any personally identifiable information is needed to be collected and stored, and ensure appropriate security measures are in place.

    3. Investigate techniques that protect student privacy while obtaining meaningful insights.

  5. End-User Needs and Use Cases

    1. Identify the target audience for the analytic solution ((Ex. instructors and administrators).

    2. Understand the specific use cases and workflows where the tracked metrics will be utilized.

    3. Determine the level of granularity required for data analysis and reporting based on user requirements.

      1. High-Level Granularity (no breakdown)
        - Ex. There have been a total of 500 instructional hours spent

      2. Medium-Level Granularity (some breakdown)
        - Ex. Student X has spent 15 instructional hours. Student Y has spent 24 instructional hours.

      3. Fine-Grained Granularity (more breakdown)
        - Ex. In Module 1, Student X has spent 15 instructional hours, while Student Y has spent 24 instructional hours.

By thoroughly understanding why developing an analytical solution is necessary to address the problem, an analytical solution can be effectively designed and developed to address the specific needs for applications and simulations of the DLX projects in Unity and Javascript.

Identify Data Sources

There are some considerations for identifying the data sources for tracking information for the analytical solution. From Unity to the Javascript application/simulation, there are different ways to process and handle and process data.

Unity

There are built-in features and components that can be utilized to gather data. The Unity Engine has an analytical package that can help capture events and components. Using the Unity scripts can help leverage and identify specific points in the application.

Javascript Application/Simulations

Employ libraries such as jQuery to capture event handling, which is an easy-to-use API that works across a multitude of browsers.

Using frameworks such as Reach or Angular will help simplify data collection and management.

Custom Data Collection Mechanisms

  • Explore the possibility of integrating with external services or APIs that help capture data.

  • Potential use of user input and forms that prompt users to provide the required information.

  • Integration of an existing database, or extending the current database schema for the new data.

Design Data Model

A data model is an abstract overview that organizes and manages the captured data that tells a story about how the events are unfolding in the environment. This helps highlight the values you need, where they come from, and how they’re structured.

Developing an effective data model helps provide insight into the functionality, performance, scalability, and usability of the environment and project. Most importantly, a data model centralizes the data visualization, which is important if there is data being provided from multiple sources.

With the design of the data model, Defining the Problem and Identifying Data Sources provide a great foundation for the direction of the model. The next steps are as follows:

Establish relationships between entities
With a well-established and technical view of the problem and data needing to be handled, defining the specific relationships between captured data can be distinguished. Here are the types of relationships:

  • One-to-one relationships

  • Many-to-one relationships

  • Many-to-many relationships

Depending on the database and the DBMS, the relationship will need to be represented by a column(s) in each table. For example, SQL databases need to define primary and foreign keys as unique values to reference values in other tables.


Physical Modeling

Provide specific detail on how the structure of the internal database(s) will be constructed, as well as connect any external data sources (Ex. Unity). The creation of the physical model will put all of the previous steps into practice. For the external data sources, think about how these will connect with the analytical solution and how these external data sources will be implemented into the data model.

Define Analytics Approach

With the model, determine the analysis techniques and algorithms that will provide insights from the modeled and collected data. The data provided doesn’t have any value by itself, so the analytic analysis provides value and meaning to the data collected and modeled. To accomplish the extraction of value from the data, identify the appropriate statistical and machine learning methods to process and interpret the data.

All visualization of the analysis representations should consider the end-users for presentation.

Develop Data Collection Mechanisms

With the process of the design for the analytical solution completed, the development of the mechanisms needs to be implemented. These mechanisms need to capture the required data points within Unity and other DLX projects. The integration of the tracking mechanisms needs to properly record the desired metrics, such as event logging or timers.

Ensure that the data collection process is efficient, accurate, and minimally intrusive to the user experience and the existing project.

Implement Data Storage and Processing

A data storage solution that meets the requirements to handle and store the collected data from the data sources (Ex. Unity). This solution should be able to handle the anticipated data volume and accommodate the defined data model from the design process.

Choose an appropriate data storage and database system technology that should align with the project requirements. If not provided by the chosen technology, develop data processing algorithms to aggregate, analyze, and generate meaningful insights from the collected data.

Create Reporting and Visualization

Create the system to design and develop the reporting mechanisms to present the tracked metrics and analytics results. The implementation of the system should generate visualizations, such as charts, graphs, or dashboards, to make the processed data more understandable and actionable for end-users.

Test and Validate

Before deploying the analytical solution, it is essential to test and validate it thoroughly. This phase ensures that the solution is generating reliable, accurate, and actionable insights. The following are some key steps in the testing and validation process:

  1. Conduct functional testing: Test the solution to ensure that it is working as intended and that all the features are functioning correctly.

  2. Validate data accuracy: Ensure that the data being collected and processed is accurate and consistent.

  3. Conduct performance testing: Evaluate the performance of the solution to ensure that it is efficient and scalable.

  4. Conduct user acceptance testing: Gather feedback from end-users to ensure that the solution meets their requirements and expectations.

  5. Conduct security testing: Ensure that the solution is secure and that the data being collected and processed is protected.

Document the Solution

Documentation is an essential aspect of any software solution. It provides a comprehensive overview of the solution, including its purpose, functionality, and technical details. The following are some key elements that should be included in the solution documentation:

  1. Introduction: Provide an overview of the solution, including its purpose and intended audience.

  2. Solution Architecture: Describe the technical architecture of the solution, including the data model, data sources, and analytics approach.

  3. User Guide: Provide detailed instructions on how to use the solution, including how to access the data, generate reports, and visualize the insights.

  4. Technical Guide: Provide technical detains on how the solution works, including the data collection and process mechanisms, the algorithms used for analytics, and the software and hardware requirements.

  5. Maintenance Guide: Provide instructions on how to maintain and support the solution over time, including how to troubleshoot issues and make updates and improvements.

Deploy and Maintain

Once the solution has been tested, validated, and documented, the final step is to deploy and maintain it. This involves ensuring that the solution is properly integrated into the existing technology stack and that it is supported and maintained over time. The following are some key steps in the deployment and maintenance process:

  1. Prepare for deployment: Ensure that the solution is properly configured and that all the necessary components are in place.

  2. Conduct user training: Provide training to end-users on how to use the solution and how to interpret the insights generated.

  3. Monitor performance: Continuously monitor the performance of the solution to ensure that it is functioning correctly and efficiently.

  4. Address issues: Address any issues or bugs that arise in a timely manner to ensure that the solution remains reliable and effective.

  5. Iterate and Improve: Continuously iterate and improve the solution based on user feedback and evolving requirements to ensure that it remains relevant and effective over time.

After defining the problem, identifying data sources, designing the data model, and implementing the data storage and processing, the next step is to test and validate the analytical solution. This entails ensuring that the data is being accurately collected, processed, and analyzed and that the results being generated are reliable, accurate, and actionable.

Once the solution has been thoroughly tested and validated, the next step is to document the solution. This documentation should provide a detailed overview of the solution, including the problem statement, the data sources used, the data model and schema, the analytics approach, and the reporting and visualization mechanisms.

After documenting the solution, the final step is to deploy and maintain the analytical solution. This involves ensuring that the solution is integrated into the existing technology stack and that it is properly maintained and supported over time. This includes monitoring the performance of the solution, addressing any issues that arise, and continuously iterating and improving the solution based on user feedback and evolving requirements.
