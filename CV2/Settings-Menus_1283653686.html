Settings Menus

The settings menu comes in two forms: a simple version which has controls for theme, volume, and mouse sensitivity, and a complex one which adds controls for sound effect, and dialogue volume.

https://www.figma.com/design/om9GwMhkfeJpcADbKz4ov8/CVRI-Design-System?node-id=4112-1915&t=yuEJnmZ31oDVSyQi-4

Setting up your scene

To get started with implementing a settings menu in your scene

  1. Drag in the simple or complex prefab into the scene hierarchy at “/Packages/Velcro UI/Prefabs”

  2. Replace the PanelSettings reference from VELCRO to your project’s PanelSettings asset

  3. Change default SerializedFields. A more complete summary of SerializedFields can be seen below

    1. Ensure starting values on the SettingsMenu script match your starting values on your AudioMixer and AudioSources

  4. Hook up UnityEvents as required

    1. You will need to connect volume events to an AudioManager

      1. Set up an AudioMixer with 3 exposed groups, “Volume”, a child “SoundEffects”, and a child “Dialogue”

      2. Convert the linear values of the Slider to logarithmic values. A tooltip is present on the Settings Menu Prefab slider fields to help understand how these values work.

    2. You will need to connect the camera sensitivity event to a CameraManager

      1. If you are using Cinemachine and PointAndClick, ensure both starting values are the same

    3. You will need to connect the theme toggle event to a ThemeManager

    4. You will need to call Show() from another script when you want the settings menu to be displayed

ConvertLinearVolumeToLog()

  • This method has been marked as [Obsolete] and moved into the DLX template

Definition:

public static float ConvertLinearVolumeToLog(float newValue)

Comments:

  • Converts a value from a linear slider (0 - 1) in the settings menu to a logarithmic value (0.0f to -80.0f dB) for use in an audio mixer

  • This is no longer called automatically by the Settings Menu when slider events are triggered. You must call this method manually, or perform the conversions yourself, as the slider events now return the linear float representation directly.

  • This is helpful for Cloud Save when loading volume levels

{
    if (!enabled)
    {
        audioMixer.GetFloat(MasterTag, out previousVolume);
        audioMixer.SetFloat(MasterTag, MinVolume);

Making Changes

If the settings menu does not fit your DLX needs out of the box, there are a few easy changes that can be made without having to create a new script.

Hiding Useless Controls & Adding Custom Controls

If the settings menu still has a control active that your DLX does not support such as a theme toggle,

  1. Create a new UXML file, [NameOfProject]SettingsMenu.uxml by Right Click/Assets > Create > UI Toolkit > UI Document

  2. Drag in the desired settings menu template from the library panel

  1. Unpack the TemplateContainer instance, and delete the wrapper parent element

  1. Change the desired container to DisplayStyle.None

This will allow you to continue to use SettingsMenuSimple.cs or SettingsMenuComplex.cs since the reference will continue to be there, just hidden from the user.

  1. You can now add your own DLX specific settings, and write a script specifically for those controls while letting VELCRO take care of the base settings menu.

Changing Slider Min & Max Values

If the default min and max values of sliders do not fit your DLX, you can change them both from a script or in the UXML file.

  1. Repeat steps 1 to 3 from above

  2. Navigate to the desired FillSlider control and change the Low Value, and High Value fields in the Attributes panel

Changing the volume sliders low and high values are not recommended since they are converted to audio mixer values ranging from 0.0dB to -80dB before being output by the UnityEvents
