CORE Framework : Analytics V2 PlayFab Connection to ADX

Created by Richard Colbourne, last modified on Aug 02, 2024

Important - access must be granted to Azure Data Explorer (ADX) before developers can create the data connection between PlayFab and ADX. Reach out to a member of the DNA team to have access created.

  • Configuring PlayFab
  • Configuring Azure Data Explorer (ADX)

Configuring PlayFab

  1. Log into your PlayFab account here: https://developer.playfab.com/en-us/my-games

  2. Click on the DLX you would like to create the data connection for:

  1. Under the Analyze heading, click Data.

  1. From the menu at the top, click Data Connections.

  1. Click the New Connection button that appears at the top right.

  1. Enter the required information in the Data Connection settings page and click Save. Note: Table will be named whatever you would like the table named in ADX (analytics-test in the screenshot below). The rest of the fields should be the same as the screenshot.

  1. Keep the page open as you will need the information listed below the fields you entered (screenshot below for reference although your information will be different). Also, do not hit the Save button at this time as it will give you an error.

Configuring Azure Data Explorer (ADX)

  1. Log into the PlayFab ADX cluster here: PlayFab Data Explorer

  2. Click on the Data tab to expand the options.

  1. From the expanded options, click on Query.

  1. Copy the first query from the information that appeared once you were done configuring PlayFab into the query area of ADX and click run.

  1. Copy the second query from the information that appeared once you were done configuring PlayFab into the query area of ADX and click run. Note: only the authorizedTitleId will be different. “BFC69” in the screenshot below is the titleId for the analytics-test PlayFab for example. Important Note: You may receive an error when running this command. If you receive an error, please reach out to a member of the DNA team to run the command as they will have higher access permissions. Click the permissions tab on the right of the ADX screen to see which VARLab members currently have database admin rights so you know who to reach out to.

  1. Once the queries have been successfully completed, close ADX and launch your DLX and create several pieces of test events. It takes approximately 15 minutes for data to flow from PlayFab into ADX (generally). Once the test data has been created and sufficient time has elapsed to allow for the data to flow between systems, log back into ADX.

  2. Click on the Query tab (as in step 3) and you should now see your table listed.

Return to PlayFab and hit the Save button to ensure the connection is set up on both ADX and PlayFab. Congratulations… the data connection is now complete.
