CORE Framework : UX/UI Onboarding for Developers

Created by Sana Javeed, last modified by Balazs Karner on Apr 15, 2025

The objective of this document is to onboard anyone new to the team about the design process in the project or use it as a refresher anytime.

Last Updated: August --, 2024

What documents can I access for any design related work?

Ans: The documents titled “Scenario Designs: < Project name> “under a specific DLX project will be the design related document. This document will contain the user workflows/scenarios, user tasks and the links to the figma files.

What documents can I access for any Content or Scripting related work?

Ans: Please contact your Instructional Designer (ID)/ Lead Developer to provide you access to the Sharepoint folder of the project that will house the information that you need.

How can I access the figma files for a project?

Ans: The documents titled “Scenario Designs: < Project name> “under a specific DLX project will be the design related document. This document will contain the links to the figma files.

The figma files are also linked in the Jira tickets created for that Sprint.

Do I need an account to access the figma file?

Ans: Yes. Please use your conestoga account to create a figma account and log in using the same credentials.

Where do I download the icons, assets/images of assets being referenced in the figma file?

Ans: For Icons, you have 2 options:

  • Option 1: Export the assets from the figma file

First select the asset in the Figma file, and click on the export button available on the properties panel view on the right. However, when you do import into unity please ensure the icon is not pixelated.

  • Option 2: Download from SVG Repo - Free SVG Vectors and Icons. All the icons are downloaded from this free source currently which means searching by name and downloading the format suitable for Unity.

Future scope

We plan on constructing an icon library with accurate naming conventions for easy findability and export for your projects. But for now, please refer to the above solutions, while we develop an optimized solution.

For Images of Assets or any information about assets, please contact your Lead Developer/ PI Dev/your team to provide you access to the team’s Sharepoint folder that houses this information that you need.

As a developer I am being invited to the “Design Syncs”. What are these scheduled for? And what to expect?

Ans: The Design Syncs are meetings scheduled on a weekly basis for the team to discuss the user workflows, storyboards and any design related conversations. Here the UX/ID facilitate the conversations with the team in order to brainstorm and discuss any potential solutions or concerns with the user workflows.

Here we also discuss about the technical feasibility or blockers in development with the proposed solution

If there are any changes in the design or workflow, how will I as a developer be informed?

Ans: Any modifications to the existing designs will have a Jira ticket where the figma designs will also be linked for clarity or a brief description will be provided if no design exists. This is to track any changes..

As a developer, how do I reach out for any design-related queries or figma-related access issues?

Ans: Tag in your project teams chats
