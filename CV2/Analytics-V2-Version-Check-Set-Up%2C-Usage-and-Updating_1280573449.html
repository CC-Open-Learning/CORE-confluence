CORE Framework : Analytics V2 Version Check Set Up, Usage and Updating

Created by Richard Colbourne on Aug 14, 2024

Your project must have PlayFab installed and the project set up on the PlayFab service before utilizing the Version Checking features of the Analytics Package. For more information on how to setup PlayFab in your project refer here: https://varlab-dev.atlassian.net/wiki/spaces/CV2/pages/1224572964/Analytics+V2+Installation#PlayFab-Account-&-Dependencies For more information on how to use the Version Checker in your scripts, refer here: https://varlab-dev.atlassian.net/wiki/spaces/CV2/pages/1236467761/Analytics+V2+Code+Implementation#Version-Check-Event

Overview

The Version Check feature of the analytics package allows developers to quickly assess whether the DLX version a learner is using is current while providing callbacks to allow for notifications or other version management options.

The Version Check must be setup in PlayFab and in the Analytics Config File. For more information about using the Analytics Config File, refer here: Using the Analytics Configuration File

Creating the Version Information on PlayFab

There are two ways to set up the version information on PlayFab, either directly in PlayFab or within the PlayFab Editor Extension in Unity. Only one method needs to be completed.

Method 1 - Create key pair in PlayFab directly

  1. Log into PlayFab here: PlayFab and click on the project you are creating the version information for.

  2. Click on the Content tab.

  1. Click on Title Data.

  1. Click on New Title Data.

  1. Click +Add.

  1. Set the name of the data (must be BuildNumber for the Version Check to function) and the version number and click Save.

Congrats, you’ve now set up the version number configuration on PlayFab!

Method 2 - Create key pair in the Unity Editor PlayFab Extension

  1. Open your project in Unity and log into PlayFab using the PlayFab Extension.

  2. Click on the Data tab and then the icon. Note: You can also click on the button to retrieve any Title Data that may have already been set up in PlayFab but not yet downloaded by the PlayFab Unity Extension.

  1. Set the name to BuildNumber and click the button.

  1. In the window that pops up, set NewValue to the build number you want then click Save.

  1. Once the window closes, you’ll see the value update in the Unity PlayFab Extension. Click Save.

Congrats, you’ve now set up the version number configuration in PlayFab using the Unity PlayFab Extension.

Updating the Build Number

There are two ways to change the version information on PlayFab, either directly in PlayFab or within the PlayFab Editor Extension in Unity. Only one method needs to be completed.

Method 1 - Changing the Version Information in PlayFab directly

  1. Log into PlayFab here: PlayFab and click on the project you are creating the version information for.

  2. Click on the Content tab.

  1. Click on Title Data.

  1. Under Primary title data, you’ll see BuildNumber. Click on the Edit icon.

  1. Change the Value to the new value and click Save.

Congrats, you’ve updated the version number configuration in PlayFab.

Method 2 - Update the key pair in the Unity Editor PlayFab Extension

  1. Open your project in Unity and log into PlayFab using the PlayFab Extension.

  2. Click on the Data tab. Note: You may need to click on the button to retrieve any Title Data that may have already been set up in PlayFab but not yet downloaded by the PlayFab Unity Extension.

  1. Click the button beside the item you want to update.

  1. In the window that pops up, update the value to the build number you want then click Save.

  1. Once the window closes, you’ll see the value update in the Unity PlayFab Extension. Click Save.

Congrats, you’ve updated the version number configuration in PlayFab using the Unity PlayFab Extension.

When to change the Build Number

Generally, you can change the build number at any time when working with a Development or Test version to ensure you are using the correct version. However, for Production versions, the build number should only be changed in PlayFab once the version is going live.

For example, when creating a new production build, the workflow should be completed as follows:

  1. Change the Build Version number in the Analytics Configuration File. For more information about the Analytics Configuration File, refer here: Using the Analytics Configuration File

  2. Create the Build.

  3. If the DLX has a build that is currently being used by students, update the PlayFab BuildNumber (either through the PlayFab extension, or in PlayFab directly, on the day the new build goes live. If the DLX doesn’t have a build being used by students, the PlayFab BuildNumber can be updated any time before the DLX goes live.
