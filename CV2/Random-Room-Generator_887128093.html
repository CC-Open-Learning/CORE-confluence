CORE Framework : Random Room Generator

Created by Richard Colbourne, last modified on Dec 14, 2023

Purpose:

Create a tool to generate random floor layouts of a defined number of rooms.

Status:

Currently, the generator allows the user to enter how many rooms wide by how many rooms deep they would like the floor to contain (in the picture above, 3 rooms wide by 2 rooms deep). The algorithm needs refinement as it is only guarantied one point of connection (the algorithm looks at the deepest created room in the first row and starts the second row generation using that point as the initial depth). As the planes are procedurally generated, the wall and stud creation tools should be able to integrate with this tool without a lot of re-work.

Usage:

In the Procedural Worlds project, double-click the Room Generator scene to launch to grid-based level editor section of the project.

While in scene mode, click on the Floor object in the hierarchy. The options for creating the layout are presented in the Inspector as follows:

Room Number: is the number of rooms wide the layout will be.

Depth: is the number of rooms deep the layout will be.

Max Size: is the maximum size a side of the room can be.

Min Size: is the minimum size a side of the room can be.

Once these properties are set to the desired settings by the user, going into play mode will generate the layout. Note: the larger the layout, the longer it will take to generate. The generator has been tested with rooms up to 250 x 250 without issue. Testing a 1000 x 1000 room caused the generator to become unresponsive as resources were likely exceeded.

The generation algorithm needs refining and expanding to create more realistic house layouts. As this was a completely experimental project, the overall structure of the project also needs updating and refining.
