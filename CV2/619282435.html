CORE Framework : Viewer & Player Clients

Created by Lisa Nguyen, last modified on Feb 09, 2023

This section will detail information about viewer and player client interactions.

Description

Players and Viewers are two types of clients.

Players can physically interact with the environment. Viewers are not visible to Players, they are able to move around the server in third-person view or watch what a Player is doing in real-time using their camera.

Scripts

  • Player controller

    • Used for both viewer and player

    • Using Unity’s new input system

  • Camera controller

    • Used for both viewer and player

    • Viewer will be able to lock/unlock camera panning when attached to a player

  • View switcher

    • Used for viewer-type clients to switch between their own camera and player cameras

  • Player interaction

    • Used for players to interact with the environment

Quick Start Guide

Player and viewer prefabs are included, at the moment you will have to switch them on manually in play mode (spawner script TBA).

Goals

  • Optimizing the viewer script for latency

  • Custom spawning so you can choose between being a Viewer or Player upon joining the server

  • 'Logout/Quit' from the server

  • Make the viewer script ‘easier’ to use

    • Scroll wheel to zoom in and out

    • Click and drag to pan around the environment

  • Viewer to be able to switch between more than one player

    • Using a ‘registering’ system where spawned in Players will auto-register their cameras to a List

      • Viewers will be able to access the list and choose which Player to watch

  • Testing multiple viewers in the same server
