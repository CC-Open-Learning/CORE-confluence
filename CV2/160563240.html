CORE Framework : Addressables & Cloud Content Delivery

Created by Jordan Robinson, last modified on May 06, 2021

Prerequisite article: Using Unity Cloud Content Delivery

Contents:

  • Profile Setup
  • Addressable Settings
  • Groups
  • Group Schemas
  • Final Steps

In this article we will cover how to set up our Unity project to access our Cloud Content Delivery (CCD) buckets to download assets remotely and access them via the Addressables system.

Profile Setup

Open the project in the Unity Editor and go to Window->Asset Management->Addressables->Profiles.

In the Addressables Profiles menu, we need to set the RemoteLoadPath to point to the CCD bucket we previously created and uploaded our Asset Bundles to. To get the exact address to paste here, view the bucket in CCD and open the ADDRESSABLE REMOTE PATH URLS tab. Here we can directly copy the remote path to our content.

For this example we will copy the release with the latest badge on it. In a production environment it is better to set this content to use a production badge so that users are not always on the newest assets and the latest releases can be used for testing first.

Once you have clicked the Copy to clipboard icon return to the Unity Editor and paste that into the RemoteLoadPath entry.

Addressable Settings

With the profile settings complete we need to set the Addressable Asset Settings to use our remote paths instead of the local ones. In the Unity editor, navigate to Window->Asset Management->Addressables->Settings. In this menu, enable Build Remote Catalog and point the Build and Load paths to their remote locations.

Groups

The final piece of the puzzle is to set all the Addressables Groups to use the Remote build and load paths. In the Unity editor, navigate to Window->Asset Management->Addressables->Groups. Within this menu, select each group and in the Inspector change them to use the Remote build and load paths.

Group Schemas

If you are using Unity Cloud Build to build your Addressable asset bundles, their path in the cloud will be your RemoteBuildPath. So in this case, if Cloud Build were to build these, they would be located at “/ServerData/WebGL.”

This can create an issue if your Addressable Groups are not set to use Internal Id Naming Mode setting Full Path. To be absolutely sure, check our the Asset Group Schema for your Addressables Group in “Assets/AddressableAssetsData/AssetGroups/Schemas” and make sure that setting is correct.

Final Steps

Once these settings have been finalized, your project will now load the asset catalog from CCD as well as each Addressable Group you set to be built and loaded remotely. You can have a combination of remote and local assets if you wish to distribute some of the assets with the build and others through CCD.
