CORE Framework : GameAnalytics Service

Created by Lisa Nguyen, last modified by Alistair Campbell on May 26, 2023

This article will outline the research put into the GameAnalytics service as a possible solution for gathering and visualizing DLX analytics.

Overview

GameAnalytics is a full suite for gathering and analyzing data. It's a pipeline consisting of data gathering tools you can build into your application based on events, processing it using a variety of granular options, then displaying it for easy human interpretation.

GameAnalytics has extremely easy integration into Unity projects having its own unity package that can be downloaded and installed using package manager (Instructions in the SDK section).
It also has integration into JS projects, having a JS library based on NodeJS.

Pros

  • No cost for regular feature set

    • Unlimited studios/games

  • Extremely easy setup in projects

Cons

  • The heavy lifting data analysis tools only get access to the data a day after collecting it

  • Cant export data without paying for Datasuite

    • Export seems to go to direct to AWS cloud storage, not to your chosen system.

  • Has to be built in order to receive metrics, harder to develop

Tools & Features

Dashboards [link]

GameAnalytics has a variety of real time dashboards for monitoring different values for your game, as well as the ability to create custom dashboards to display curated metrics for whatever your needs might be. Building custom dashboards uses a widget builder that allows you to design the element for the dashboard and select/filter the data for it.

Explore Tool [link]

The explore tool is a system that allows you to query data in pretty much any way you want. You can get your metrics and filter, aggregate them, and view them however you need to.

Funnels [link]

Funnels are a way of displaying data across a series of events. This is useful to see where players are getting stuck, which could help to expose software issues with shipped DLX projects, or discover difficult areas in educational content to help direct course focus.

Data Processing Tools [link]

Building on the regular funnels/explore/dasboards, GameAnalytics also has a premium data analysis suite called DataSuite.

The only information I can get without contacting them is that DataSuite gives you direct access to the user data instead of abstracting it through viewing tools. We may not need that level of control.

Other Features

Purchase and Advertisement tracking. Pretty self explanatory, hooks into app stores and provides metrics on player purchases. Probably not useful to us.

A/B Testing. Allows you to deploy 2 different versions of an application and directly compare them through your normal metrics collection. Might be useful, but I probably not used too much.

SDK/Integration tools [docs]

Unity [link]

GameAnalytics has a Unity package that can be added to your project. It can be downloaded from this link and import it in unity through Assets > Import Package > Custom Package.

After installing the package either sign up or log in through Window> GameAnalytics > Select Settings. Once logged in it will ask you to connect it to one of your game profiles using its key, which can be gotten from registering the game on GameAnalytics website.

From there, its just 2 things to get GameAnalytics running. Firstly, initializing the GameAnalytics SDK using GameAnalytics.Initialize(). Secondly, creating the GameAnalytics GameObject using Window > GameAnalytics > Create GameAnalytics Object.

Ensure only one GameAnalytics GameObject exists.

Only send events from the main thread.

No events are generated in editor, game must be built and run to generate events.

JavaScript [link]

The JS SDK is an NPM install, and depends on Node.js and Gulp.js.

It is initialized using script tags in HTML, and needs to be built using Gulp.js. The specific instructions are found here.

Events System [link]

There are 3 specific types of events that will likely be useful to us, these are Progression, Error, and Design events.

Custom dimensions are values that can be attached to events in order to have more values to group, sort, and compare by.

Progression Events [Link]

Useful for tracking a player’s progression through the game. Supposedly uses a 3 item hierarchy (ex. World, Level and Phase) to track where a player is.

ex. “A players starts Day 1 of the Wild West world in Plants vs. Zombies 2.”

There should only be ONE progression value, and any other values should be tracked using Design Events or custom fields.

The 3 item hierarchy is outlined as follows:

Not every field must be filled, as shown in the last row. You can either send all 3 columns, just columns 1 and 2, or only column 1.

Progression events also have statuses. The 3 possible statuses of a progression event are

  • Start

  • Complete

  • Fail

Progression events can also be given an optional score value along with these fields.

This is an example of a completed Progression Event call using Unity SDK.

GameAnalytics.NewProgressionEvent(GAProgressionStatus.Complete, "World_01", "Stage_01", "Level_Progress", 200);

Error Events

Can be used to log errors or issues caused by player behavior. They are sent with a type and a message.

ex. “A player has scored over 1 billion points on a level, which is not supposed to happen in your game.”

Error Events have a set of specific types, being the following

  • Info

  • Debug

  • Warning

  • Error

  • Critical

This is an example of an Error event call using Unity SDK.

GameAnalytics.NewErrorEvent (GAErrorSeverity.Info, "Could not find available server.");

Error events are shown in real time on the dashboard, unlike all other event types.

Design Events

Design events are essentially just a custom event based on a hierarchy of values. These values can be whatever you want, and are used to form a tree of events. These events can be triggered anywhere under any circumstance and have a pretty intuitive system for grouping/categorization. This seems to be a very powerful event system, and I think it would be quite useful for triggering events on success and fail conditions for specific tasks.

Here are some examples of events from the docs

These events have restrictions: There are limitations on the number of unique values that can be generated per day. There should not be more than 100,000 unique nodes per day. If this limit is exceeded they will contact you to resolve the issue (which is pretty nice of them). They talk about how to avoid this here.

Here is an example of a design event call using Unity SDK.

GameAnalytics.NewDesignEvent ("Achievement:Killing:Neutral:10_Kills", 123);

Design events use an event string to determine the type of event that is happening. These event strings are laid out in a hierarchy in descending order. This design will make a tree of events like this.

The other thing to note about these events is that to include a number, you can add it after the event string (as shown in the example).

Custom Event Fields [Link]

Custom Event Fields are not accessible through the dashboard or any free tools. They can only be accessed through the premium GameAnalytics Datasuite

GameAnalytics events can contain more information than what is passed in with the basic event calls. You are able to attach extra key/value pairs. The pairs can be attached to a dictionary, then attached to the event using code like this.

Dictionary<string, object> fields = new Dictionary<string, object>();
fields.put("test", 100);
fields.put("test_2", "hello_world");
GameAnalytics.NewDesignEvent (eventName, eventValue, fields);
