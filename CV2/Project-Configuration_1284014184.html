CORE Framework : Project Configuration

Created by Jacob Martens on Aug 14, 2024

This guide covers how to configure a project to support XR development using OpenXR and XR Interaction Toolkit

  • Installing Required Packages
  • Configuring the Project
  • Simple Project Validation

Installing Required Packages

From a Unity project, open Window->Package Manager, and locate Unity Registry. In Unity 6, this will be located along the left side of the window. In previous versions, it can be found by clicking Packages: In Project along the top. Using the search, find and install OpenXR Plugin (com.unity.xr.openxr) and the XR Interaction Toolkit (com.unity.xr.interaction.toolkit). Click Yes to any popup prompt that occurs when installing packages.

Known Issue

OpenXR needs to be version 1.10.0 for use with the Meta Quest 3 headset. There is an issue where one of the Project Validation checks incorrectly fails that is fixed with this version. To force update the package, open the project in Windows Explorer, and open [BASEFOLDER]\Packages\manifest.json. Locate "com.unity.xr.openxr" and change the version to "1.10.0". The full entry should be "com.unity.xr.openxr": "1.10.0". Upon saving and returning to the Unity Editor, it will re-import and install the updated version

Configuring the Project

In the Unity Editor, open Edit->Project Settings and select XR Plug-in Management. For the relevant build-platforms (the different menu tabs), select OpenXR under Plug-in Providers. In the left-side menu, select OpenXR. For each of the build-platforms you enabled OpenXR for in the previous step, an option needs to be added to the Enabled Interaction Profiles list. Click the plus icon, and select the controller you will be using for development. Additional side step: If you are working with a Meta Quest 3 Headset, ensure Meta Quest Support under OpenXR Feature Groups is enabled. Following this, select Project Validation from the left-side menu. For each previously selected build-platform, click Fix All in the top right corner. After clicking the button it will take some time to compile scripts and reload domain. When it has completed, ensure there are only green checks left. Continue clicking Fix All until there are no more issues left.

Simple Project Validation

To validate the project has been confiugred correctly, go back to the Package Manager and navigate back to Packages: In Project. Locate the XR Interaction Toolkit and select Samples. Install the Starter Assets sample. In the Project tab, navigate to Assets->Samples->XR Interaction Toolkit->2.5.4->Starter Assets, and open DemoScene. This scene is pre-configured for everything XR and should work with any configured headset.
