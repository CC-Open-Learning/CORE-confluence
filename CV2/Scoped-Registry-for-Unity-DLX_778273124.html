CORE Framework : Scoped Registry for Unity DLX

Created by David Inglis, last modified on Sept 27, 2024

Summary

This document describes the steps to configure the Unity Editor for a project in order to access the UPM Registry for CORE Framework packages.

This process will need to be done once per project, though it will likely be included in the https://bitbucket.org/VARLab/dlx-project-template/src repository.

Modifying the Unity Package Manifest

From within the Unity Editor for a project, navigate to Edit → Project Settings → Package Manager and enter the values below into the associated fields under Scoped Registries

Name

CORE

URL

http://upm.core.varlab.org:4873/

Scope(s)

com.varlab
com.paradoxnotion
com.pirateparrot
com.veriorpies
com.firstgeargames
com.microsoft.playfab

Alternatively, open the file Packages\manifest.json within the project folder and add the following lines underneath the dependencies block:

"scopedRegistries": [
  {
    "name": "CORE",
    "url": "http://upm.core.varlab.org:4873/",
    "scopes": [
      "com.varlab",
      "com.paradoxnotion",
      "com.pirateparrot",
      "com.veriorpies",
      "com.firstgeargames",
      "com.microsoft.playfab"
    ]
  }
]

After saving the file and opening the Editor, Unity will attempt to read from the new registry.

The article Accessing the Package Registry for Developers has been created to describe the workflow for accessing and configuring the UPM Package Registry for Unity from the command line.
