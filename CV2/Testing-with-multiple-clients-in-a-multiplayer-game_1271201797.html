CORE Framework : Testing with multiple clients in a multiplayer game

Created by Talon Ernst, last modified on Aug 09, 2024

In order to properly test a multiplayer game, you will need to have multiple instances of the project to act as different “players” inside of the game. There are multiple ways to achieve this, and they will be discussed here.

The are three main ways that this can be achieved. Two of these methods are through packages ParrelSync and Multiplayer Play Mode, and using one of these is the recommended way to do this, or you can make separate project builds to act as each player, this is not recommended.

ParrelSync

ParrelSync is a package that was made to allow developers to very quickly create a clone of their project in order to test multiple functionality.

When you make a clone using ParrelSync, it will create a clone of your project and store it locally, it can then be started and ran alongside the main project, allowing for multiplayer testing.

Once ParrelSync is in the project, a tab for it will appear at the top of the navigation bar, as seen in the following image

When this is clicked on, the following screen will appear

I already have two clones created, so they appear here. If ParrelSync was just downloaded only the “Add new clone” button will appear, and can be clicked to make a clone.

From this menu, a new clone can be created and loaded. If multiple clones are created, they will also all appear here and can be managed from here as well.

The following image is an example of the files that are created when a clone is created

The clones added by ParrelSync do not need to be pushed onto repositories as they are purely local and meant for testing. Add the following line to your .gitignore to prevent this.

*_clone_[0-9]*/

Multiplayer Play Mode

Multiplayer Play Mode, or MPM for short, is similar to ParrelSync in what it is designed to do, but it is made directly by Unity and is new to and only available in Unity 6. It still has its bugs and limitations, so be wary if this is what is used for testing.

Once MPM is in the project, you can find the MPM menu by going to Window in the top navigation bar and clicking on Multiplayer Play Mode, as seen in the following image

Once you click on this, the following menu will appear, you will see that you can create up to three more players alongside your main editor,

You can activate/deactivate a player by clicking the checkmark on the left side of the screen corresponding with the player you want to update. Other features include adding a tag to each player and seeing each respective player's logs, warnings and errors.

When you start your main editor, the other players will automatically start and look like the following image

If you want to show more things, such as an inspector or the hierarchy, you need to click on the button that says “Layout” on the top right of the screen, as seen in the following image

Once it is clicked on, the following menu will appear

Once you decide what you want to be shown/hidden, your settings will be saved, and you do not need to set them again.

Each player has their own settings for their layout, so if you change the settings of Player 2, it will only be there for Player 2, not Player 3 or Player 4

Project builds

While creating a build for each player you want to test with is an option and will work, it is not recommended, this will take an immense amount of time and will get really annoying to have to manage as you make more and more, please pick one of the other two options available.
