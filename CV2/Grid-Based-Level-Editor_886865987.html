CORE Framework : Grid-Based Level Editor

Created by Richard Colbourne on Dec 14, 2023

Purpose:

Create a tool to generate rooms, levels etc. using a grid-based editor.

Status:

Currently, the grid-based tool allows the placement of floor tiles on a grid. The framework has been designed for expandability and a preview system, a database to house items and different item types are all accounted for.

Usage:

In the Procedural Worlds project, double-click the GridScene scene to launch to grid-based level editor section of the project.

The editor is designed for play-mode with the expectation that the user can use the tool to design the level, save it, and reload it in either play-mode or scene-mode in order to further edit or utilize the level.

Controls:

In play-mode, pressing the red button on the UI bar (temporary implementation) allows the user to enter into the edit mode of the level editor. The mouse is used for selecting grid locations, placing and removing of tiles. Left-click will place a tile (if possible), Right-click will remove a tile (if possible). The ESC key turns off edit mode. The user can also pan around the scene using the WASD keys or the arrow keys. The - key will shrink the size of the grid (to a minimum size of 10 x 10) and the + key will expand the grid (to a maximum size of 100 x 100).

Next Steps:

There are several enhancements that should be added to this tool. The UI needs a redesign and further implementation. Other enhancements are as follows:

  • Box Selection - The user should be able to select multiple cells or items in the scene by clicking and dragging the mouse to create a box around the items or cells to be selected. Screenshot below from a game illustrating the concept.

  • Camera enhancements - Currently the user can pan the camera around the scene. The ability to zoom in and out as well as rotating around the scene should be added.

  • Additional items - The framework and database are already in place to add additional items to the editor. Prefabs needs to be created and implemented for items such as light fixtures, walls, doors, stairs etc.

  • Level save/load - The editor saves the information for each item in the scene in a dictionary ready for export/import. This could be built out into a full save and load system.
