Account Type

PlayFab Account

  • This account type only associate with PlayFab service, you get a PlayFab account if you sign up with PlayFab.

Microsoft Account (or Azure Active Directory “AAD”)

  • You can access PlayFab service by using Sign in with Microsoft on the PlayFab’s login page.

The two account types is separate from each other, you can have both account type with the same Microsoft email address, and your account would still be different. There is not a known way to switch account type once the account is created.

You can find out users' account type in All Users under the Users tab in Settings.

Azure Data Explorer

You can only use Microsoft Account to log in to Azure Data Explorer. For this reason, user that wish to access PlayFab data through Azure Data Explorer should create their account using the Microsoft Account type.

Sharing Dashboard

There are two methods for sharing a dashboard, both method requires you need to have granted permissions in PlayFab to access the underlying database for this to work.

  • Using the Share feature (it work sometimes?)

    • One requirement is you must be the owner of the data to share dashboard

Steps to share a dashboard directly

  1. Select the Share menu item in the top bar of the dashboard.

  2. Select Manage permissions from the drop-down.

  1. Enter the Azure AD user or Azure AD group (from the same organization, @conestogac.on.ca) in Add new members.

  2. In the Permission level, select one of the following values: Can view or Can edit.

  3. Select Add.

Steps to share a dashboard via export then import

  1. First you have to export the dashboard, in the dashboard, select File > Export to file.

  2. Save the JSON file.

Then, in the account you’re setting up the dashboard for, import the JSON file.

  1. In the main dashboard page, select New dashboard > Import from file.

  2. Select the file to import.

  3. Enter a dashboard name, and then select Create.

After performing the steps from above, you should be able to see the dashboard if the permissions is correctly set.
