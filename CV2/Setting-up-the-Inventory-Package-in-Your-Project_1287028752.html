CORE Framework : Setting up the Inventory Package in Your Project

Created by Liam Stanziani, last modified on Aug 15, 2024

This page will provide an example on the installation of the components that are required to use this package.

Step 1 - Import the Package into Unity.

Go to Window > Package Manager > My Registries, find the Inventory package and download it.

If you need to setup your unity registry for VARLab, please refer to this document: Accessing the Package Registry for Developers

Step 2 - Import an Inventory Game Object into your scene

Go into your scene that you want to use the Inventory system in, right-click the in the scene hierarchy, go to VARLab > Inventory and left-click

Step 3 - Setup the Inventory Game Object values

Go into the Inventory Game Object and set all of the inspector fields under the components:

Inventory UI

The header sections that you will need to modify are:

  • Visual Tree Assets

    • The UXML assets that will be used for the UI that is shown before the Inventory that will give access to the system and the Inventory UI which will include all of the functionality of the Inventory system and its values

  • UI Component Visual Element Names

    • The names of the visual elements relating to the parent of the items that will be inputted directly and the parent of the object viewer functionality

  • UI Component Button Names

    • The names of the buttons relating to the ability to open the inventory from the base UI, an item template that will be under the item row parent, the back button to close the object viewer functionality, the back button to close the current container that is opened and go to the previous one, and the inventory close button that will close the entire inventory UI

To Learn more about these what these inspector fields mean see this page: Inventory

Inventory Data

The inventory data component needs to have values inputted values to dynamically create the items/containers in the inventory UI.

Adding a new item

To add a new item, select the plus option at the bottom of the Inventory section, select the type as “Item”, then set a title, image and object for that item. The object can either be a game object or a prefab

Adding a new container

To add a new container, select the plus option at the bottom of the Inventory section, select the type as “Container”, then set a title, image, and create a inventory container under that. This can include more items that will be under the container when clicked, or other containers that can include more items under themselves

Prebuilt example of Inventory Data values

To Learn more about these what these inspector fields mean see this page: Inventory

Step 4 - Setup the Object Viewer Manager Game Object values

Go to the game object named “Object Viewer Manager” under the Inventory game object you setup in the last step, create the Keybindings scriptable object anywhere in your project files, select it under the “Keybindings” field, set the “Clone Game Object” field to true if you are using a prefabs for inventory data and set it to false if you are using active game objects for inventory data, and set the “Main Camera” field to be connected to your main camera.

Prebuilt example of the Object Viewer Controller values

To Learn more about these what these inspector fields mean see this page: Object Viewer

Step 5 - Run the scene
